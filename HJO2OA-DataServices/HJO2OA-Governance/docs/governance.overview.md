# Governance 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-Governance` |
| 模块定位 | `07-数据服务与集成` 的治理运行层，统一管理开放能力的版本治理、健康检查、告警规则、异常追踪和人工干预入口 |
| 真相源边界 | 拥有治理策略、健康检查规则、告警规则、服务版本记录；不拥有业务主数据，也不替代各子模块的定义真相源 |
| 一期目标 | 建立开放能力的可观测、可审计、可告警基础框架，优先支撑 API、连接器、同步和报表的运行治理 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 版本治理 | 管理服务与接口版本状态、发布兼容策略、废弃记录 | 支持登记、发布、废弃、查看 |
| 健康检查 | 聚合 API、连接器、同步、报表运行态，输出健康快照 | 支持基础状态检查与异常摘要 |
| 告警管理 | 维护阈值、规则、通知触发和升级策略 | 支持基础规则和告警留痕 |
| 异常追踪 | 汇总失败批次、错误分类、处理链路 | 支持查看与人工确认 |
| 人工干预入口 | 触发停用、重试、降级、备注等治理动作 | 支持人工介入，不做自动自愈 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `HJO2OA-OpenApi`、`HJO2OA-Connector`、`HJO2OA-DataSync`、`HJO2OA-Report`、`HJO2OA-DataService` | 消费这些模块的定义变化与运行结果，形成治理视图 |
| 上游 | `00` 安全、审计、调度 | 复用统一审计、安全与调度能力 |
| 下游 | `06` 消息与触达 | 告警事件交给 `06` 做通知分发，不在本模块内建设消息渠道 |
| 下游 | 平台管理端 | 为运维和管理员提供治理入口与运行状态 |
| 非本模块 | 业务域监控 | 业务域自己的业务规则监控仍归各业务域；本模块只治理 `07` 开放能力本身 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 版本治理 | `ServiceVersionRecord` | 发布/废弃记录 | 自动灰度发布编排 |
| 健康规则 | `HealthCheckRule` | 健康快照、异常分类 | 智能根因分析 |
| 告警策略 | `AlertRule` | 告警记录、升级策略 | 自动自愈闭环 |
| 干预入口 | 治理动作审计 | 停用、重试、确认 | 无审计直接执行底层修改 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 管理员配置健康检查和告警规则，绑定目标对象 | 治理策略 |
| 2 | 治理模块持续消费 API、连接器、同步、报表的运行态事件 | 健康视图 |
| 3 | 告警命中后生成告警记录并发布治理事件 | 告警记录 |
| 4 | 运维人员在治理页面查看异常链路并执行停用、重试或备注 | 人工干预结果 |
| 5 | 所有治理动作写入审计，并由 `06` 负责消息触达 | 审计记录与通知事件 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 不替代定义真相源 | 版本、状态、告警都只是治理视角，不直接改写其他模块定义数据 |
| 不承担消息触达 | 本模块只发布治理事件，触达渠道与通知模板归 `06` |
| 不做自动自愈 | 一期以人工确认和受控干预为主，不做自动修复链路 |
| 治理范围限定 | 仅聚焦 `07` 的开放能力，不扩展成全平台统一业务监控中心 |
| 治理动作可审计 | 停用、重试、降级、人工确认必须全量审计 |
