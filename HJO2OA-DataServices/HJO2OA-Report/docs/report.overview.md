# Report 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-Report` |
| 模块定位 | `07-数据服务与集成` 的分析输出层，统一管理统计口径、维度、聚合规则、报表快照和图卡数据源输出 |
| 真相源边界 | 拥有报表定义、指标定义、维度定义、刷新策略和报表快照；报表和快照是统计投影，不替代业务主数据真相源 |
| 一期目标 | 收敛基础统计口径与报表输出协议，支撑管理分析页和门户图卡数据源，但不接管门户页面编排 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 报表定义 | 管理报表编码、类型、来源范围、可见范围和刷新方式 | 支持新增、编辑、启停、查看 |
| 指标与维度 | 维护指标公式、聚合方式、维度切换规则 | 支持固定指标与基础维度 |
| 报表快照 | 生成并维护刷新结果、新鲜度、快照时间 | 支持调度与事件驱动刷新 |
| 图卡输出 | 对外输出图卡数据协议供 `04` 消费 | 只输出数据，不负责门户页面 |
| 统计治理输入 | 向治理模块提供刷新结果与新鲜度状态 | 支持运行状态事件与审计摘要 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `01/02/03/05/06` 业务域事件和查询能力 | 作为统计来源，不改变原始业务数据归属 |
| 上游 | `HJO2OA-DataService` | 报表可复用标准数据服务作为统计源之一 |
| 上游 | `00` 调度与事件总线 | 触发定时刷新和事件驱动刷新 |
| 下游 | `04` 门户与工作台 | `04` 仅消费图卡数据协议，页面聚合和编排仍归 `04` |
| 下游 | `HJO2OA-Governance` | 报表刷新成功率、新鲜度进入治理视图 |
| 非本模块 | 业务主数据与 BI 平台 | 不拥有业务主数据，一期也不建设复杂 BI 平台 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 报表定义 | `ReportDefinition` | 统计口径真相源 | 自定义复杂 BI 编排 |
| 指标模型 | `ReportMetricDefinition` | 指标聚合规则 | 指标血缘与成本治理 |
| 快照刷新 | `ReportSnapshot` | 报表结果与新鲜度 | 实时流式分析 |
| 图卡数据源 | 卡片输出协议 | 给 `04` 的可复用数据源 | 门户页面布局与用户交互编排 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 管理员定义报表类型、来源范围、指标和维度 | 报表草稿 |
| 2 | 配置刷新方式和可见范围，绑定数据服务或上游事件 | 可刷新报表定义 |
| 3 | 调度或事件触发刷新，生成快照和新鲜度状态 | 报表快照 |
| 4 | 管理分析页读取报表结果，门户图卡通过协议消费摘要数据 | 分析结果与图卡数据源 |
| 5 | 刷新结果进入治理模块做新鲜度和失败监控 | 刷新事件与审计记录 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 报表是投影 | 报表定义和快照是统计口径与结果投影，不替代业务主数据 |
| 门户聚合不归本模块 | 本模块只输出图卡数据协议，门户页面和用户交互归 `04` |
| 不做高级 BI | 一期不支持复杂拖拽建模、实时大屏、指标血缘分析 |
| 权限显式裁剪 | 报表可见范围必须按租户、组织或角色边界裁剪 |
| 刷新需标记新鲜度 | 所有快照必须带刷新时间和新鲜度状态，避免误用过期数据 |
