# DataSync 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-DataSync` |
| 模块定位 | `07-数据服务与集成` 的集成执行层，统一管理跨系统同步任务、映射规则、检查点、执行记录、对账与补偿 |
| 真相源边界 | 拥有同步任务定义、映射规则、检查点策略和执行记录；不同步业务主数据所有权，同步结果只是运行投影 |
| 一期目标 | 支撑全量、增量、事件驱动三类同步模式的基础建模与运行治理，重点覆盖任务配置、失败重试、对账和补偿入口 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 任务定义 | 管理同步任务编码、源端/目标端、同步模式、调度与状态 | 支持新增、编辑、启停、查看 |
| 映射规则 | 维护字段映射、转换规则、冲突策略 | 支持静态映射和冲突策略配置 |
| 检查点与执行记录 | 记录执行批次、检查点、耗时、差异摘要 | 支持全量/增量/事件驱动基础检查点 |
| 对账与补偿 | 发现差异、记录失败、提供重试和人工补偿入口 | 支持手工重试、人工确认，不做自动修复闭环 |
| 告警联动 | 将失败、积压、差异结果提供给治理模块 | 支持运行告警输入 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `HJO2OA-Connector` | 复用连接器访问源端与目标端系统 |
| 上游 | `01/02/03/05/06` 业务域与外围系统 | 只消费其数据或事件，不拥有其业务主模型 |
| 上游 | `00` 调度与事件总线 | 调度触发全量/增量执行，事件总线承载事件驱动同步 |
| 下游 | `HJO2OA-Governance` | 治理模块消费失败、延迟、差异等运行结果做告警和干预 |
| 下游 | 外部系统或目标模块 | 同步输出通过稳定协议写入，不直接跨边界改底表 |
| 非本模块 | 报表真相源与门户聚合 | 同步可形成投影，但不能替代 `report` 的统计口径，也不承担 `04` 门户聚合职责 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 同步任务定义 | `SyncExchangeTask` | 源目标关系、调度与模式 | 实时 CDC 与复杂流式编排 |
| 映射与冲突策略 | `SyncMappingRule` | 字段映射与冲突处理规则 | 智能自动映射 |
| 执行记录 | `SyncExecutionRecord` | 执行状态、检查点、差异摘要 | 自动修复引擎 |
| 对账补偿 | 差异记录、人工补偿入口 | 运行干预能力 | 全自动闭环补偿 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 管理员配置同步任务，选择源端、目标端、同步模式与调度策略 | 同步任务草稿 |
| 2 | 维护字段映射、冲突策略和检查点模式 | 可执行任务定义 |
| 3 | 调度或事件触发执行，生成执行记录和检查点 | 同步批次记录 |
| 4 | 失败或差异触发重试、对账或人工补偿 | 补偿处理结果 |
| 5 | 成功/失败结果推送给治理模块和相关下游 | 运行事件与审计记录 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 同步是投影 | 执行结果、差异快照、补偿记录都不是业务主数据真相源 |
| 写入必须受控 | 向目标系统写入必须通过稳定接口、命令或连接协议，不能跨模块直改底层表 |
| 显式检查点 | 增量与事件驱动同步必须声明检查点和幂等键策略 |
| 不做实时 CDC | 一期不实现复杂 CDC、流处理、跨云多活同步 |
| 补偿先人工可控 | 先支持失败重试、差异确认、人工补偿入口，不做全自动补偿编排 |
