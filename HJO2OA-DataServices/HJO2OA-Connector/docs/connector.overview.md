# Connector 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-Connector` |
| 模块定位 | `07-数据服务与集成` 的连接能力真相源，统一管理 HTTP、数据库、消息队列等基础连接器定义、参数和健康状态，为数据服务、开放接口、同步任务提供外部资源访问能力 |
| 真相源边界 | 拥有连接器定义、参数引用、连通性检查与健康快照；不拥有外部系统业务主数据，不解释外部业务语义 |
| 一期目标 | 收敛基础连接器建模、参数配置、连接测试、健康检查，优先支撑 HTTP/DB/MQ 三类连接器 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 连接器定义 | 维护连接器编码、类型、协议、认证方式、超时与重试策略 | 支持新增、编辑、启停、查看 |
| 参数管理 | 管理地址、库名、队列名、账号等参数，并对敏感项只保存密钥引用 | 支持参数模板、敏感参数标记 |
| 连通性测试 | 提供手工测试与结果留痕，验证连接器在当前环境可用 | 支持成功/失败结果记录，不做自动修复 |
| 健康快照 | 记录最近一次健康检查状态、耗时、异常摘要 | 支持快照展示与告警输入 |
| 能力复用 | 为 `data-service`、`open-api`、`data-sync`、`report` 复用 | 只提供连接能力，不替代这些模块的业务规则 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `00` 安全与配置能力 | 敏感参数必须引用统一密钥与配置能力，不在本模块存储明文 |
| 上游 | `01/02/03/05/06` 业务域与外围系统 | 只视为被连接资源，不接管其业务模型 |
| 下游 | `HJO2OA-DataService` | 数据服务可绑定连接器读取外部资源或执行受控写入 |
| 下游 | `HJO2OA-DataSync` | 同步任务复用连接器作为源端/目标端访问方式 |
| 下游 | `HJO2OA-Governance` | 治理模块消费健康快照与异常结果进行告警 |
| 非本模块 | 门户聚合与用户触达 | 门户聚合归 `04`，消息触达归 `06`，连接器不承担用户入口职责 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 基础连接器目录 | `ConnectorDefinition` | 连接器元数据与状态 | SaaS 市场化模板与可视化编排 |
| 参数模板与密钥引用 | `ConnectorParameter` | 连接参数清单 | 明文凭证托管、跨环境自动分发 |
| 连通性测试 | 测试记录、异常摘要 | 最近测试结果 | 自动巡检修复 |
| 健康快照 | `ConnectorHealthSnapshot` | 可供治理消费的运行态投影 | 智能诊断 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 管理员创建连接器定义并选择类型、协议、认证方式 | 连接器草稿 |
| 2 | 配置参数与密钥引用，标记敏感项 | 参数配置完成 |
| 3 | 发起连接测试，记录成功/失败与耗时 | 测试结果 |
| 4 | 启用连接器供数据服务或同步任务引用 | 可复用连接能力 |
| 5 | 健康检查结果进入治理模块，用于告警和停用决策 | 健康快照与审计记录 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 不拥有业务主数据 | 连接器只连接外部资源，不定义组织、流程、内容、会议等业务真相源 |
| 不替代开放能力 | 对外接口、同步逻辑、报表口径分别归 `open-api`、`data-sync`、`report`，连接器只提供访问通道 |
| 不建设市场化体系 | 一期不做连接器商店、插件市场、跨租户共享模板 |
| 密钥必须外置 | 账号、密码、Token 等敏感信息只能以密钥引用方式存在 |
| 运行态是投影 | 健康状态、测试结果只是运行投影，不能替代连接器定义本身 |
