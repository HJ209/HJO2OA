# OpenApi 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-OpenApi` |
| 模块定位 | `07-数据服务与集成` 的对外开放边界真相源，统一管理开放接口目录、版本、鉴权、凭证、限流、配额和调用审计规则 |
| 真相源边界 | 拥有开放接口定义、凭证授权、限流与配额策略；不拥有门户聚合页面、不拥有业务主数据、不承担消息触达 |
| 一期目标 | 建立标准开放 API 管理与授权能力，优先支撑接口目录、应用凭证、调用审计、版本治理和基础限流 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 接口目录 | 管理开放接口编码、路径、方法、版本与状态 | 支持注册、发布、废弃、下线 |
| 鉴权授权 | 管理应用凭证、签名方式、接口授权范围 | 支持基础凭证绑定与有效期管理 |
| 配额限流 | 维护限流和调用配额策略 | 支持基础阈值与限流策略 |
| 调用审计 | 记录接口调用摘要、错误分类、调用方信息 | 支持审计查询和治理输入 |
| 版本治理 | 与治理模块协作维护发布与废弃流程 | 支持版本登记和兼容说明 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `HJO2OA-DataService` | 开放接口优先复用数据服务协议，不重复定义业务查询语义 |
| 上游 | `00` 安全、审计、配置 | 复用统一密钥、签名、审计和配置能力 |
| 下游 | 第三方系统、外围平台 | 通过标准接口提供受控开放能力 |
| 下游 | `HJO2OA-Governance` | 调用状态、版本变更和异常结果纳入治理 |
| 非本模块 | 门户聚合与开发者门户 | 门户聚合归 `04`；一期不建设复杂开发者门户，仅收敛接口管理能力 |
| 非本模块 | 用户消息触达 | 对外告警通知归 `06`，本模块只发布接口状态事件 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 接口注册与发布 | `OpenApiEndpoint` | 对外开放接口目录 | 复杂开发者门户与 SDK 市场 |
| 凭证授权 | `ApiCredentialGrant` | 调用凭证与授权范围 | 跨租户复杂 OAuth 平台 |
| 限流配额 | `ApiRateLimitPolicy` | 调用控制规则 | 自适应弹性限流 |
| 调用审计 | 调用日志与错误摘要 | 治理输入与审计查询 | 全链路 APM 替代方案 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 基于数据服务定义创建开放接口，配置路径、方法、版本和鉴权方式 | 接口草稿 |
| 2 | 配置调用方凭证、配额和限流策略 | 可发布接口配置 |
| 3 | 接口发布后供第三方系统调用，调用过程写入审计和错误摘要 | 调用记录 |
| 4 | 接口废弃或版本升级时通知治理模块和相关调用方 | 版本治理记录 |
| 5 | 运行异常进入治理链路，由 `06` 负责后续通知触达 | 治理事件与审计留痕 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 先复用服务定义 | 对外接口应优先装配 `data-service`，避免重复实现数据协议 |
| 不承接门户聚合 | 用户门户首页、办公中心读模型不归本模块 |
| 不拥有业务主数据 | 开放接口只暴露受控协议，不改变业务主数据归属 |
| 版本显式治理 | 破坏性变更必须新版本发布，不允许原地覆盖 |
| 调用控制基础化 | 一期只做基础凭证、配额、限流和审计，不做开发者社区化能力 |
