# OpenApi 前端说明

## 页面清单

| 页面/区域 | 页面职责 | 权限要求 | 状态要点 | 移动端约束 |
|-----------|----------|----------|----------|------------|
| 接口目录页 | 查看接口清单、版本、鉴权方式和引用服务 | 开放接口管理员、审核人 | 草稿、已发布、已废弃、已下线 | 移动端仅建议查看目录与状态 |
| 接口发布页 | 配置路径、方法、版本、鉴权、配额和限流 | 开放接口管理员 | 草稿保存、待发布、发布成功、发布失败 | 表单复杂，移动端不做完整编辑 |
| 凭证管理页 | 管理调用方凭证、授权范围、到期状态 | 安全管理员、开放接口管理员 | 有效、将过期、已吊销、已过期 | 移动端仅查看凭证状态，不开放发放与吊销 |
| 调用审计页 | 查看调用量、错误分布、耗时和异常摘要 | 运维管理员、审核人 | 正常、告警、超限、失败集中 | 移动端只看摘要，不支持复杂筛选分析 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 说明 |
|-----------|----------|------|
| 接口目录页 | 筛选版本、查看详情、跳转生命周期记录 | 列表需突出版本、鉴权方式、下线计划 |
| 接口发布页 | 保存草稿、发布、废弃、复制新版本 | 破坏性变更必须走新版本，不允许直接覆盖已发布版本 |
| 凭证管理页 | 发放凭证、吊销凭证、调整授权范围 | 敏感信息只展示引用或掩码，不展示明文 |
| 调用审计页 | 按接口/调用方/时间筛选，查看错误聚类 | 审计视图只读原始调用结果，修订结论需单独留痕 |

## 前端约束

| 项 | 说明 |
|----|------|
| 权限 | 必须区分接口配置、凭证管理、审计查看三类权限 |
| 状态 | 前端需同时展示接口生命周期状态与运行风险状态，避免混淆 |
| 移动端 | 一期移动端只保留目录和审计摘要查看，不支持发布、凭证发放和复杂分析 |
| 边界 | 前端是平台管理后台，不承担开发者社区、业务门户首页或用户消息入口 |
