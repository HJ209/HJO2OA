# DataService 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-DataService` |
| 模块定位 | `07-数据服务与集成` 的服务定义真相源，统一管理可复用的数据查询、命令、导出、回调等服务协议，供开放接口、报表和同步任务复用 |
| 真相源边界 | 拥有服务定义、参数协议、字段映射、权限模式与缓存策略；不拥有组织、流程、内容、会议等业务主数据 |
| 一期目标 | 完成服务定义建模、参数和字段映射、测试验证与权限边界收敛，为后续开放 API 和报表复用打基础 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 服务协议管理 | 维护查询、命令、导出、回调类服务定义 | 支持定义、查看、启停、版本内维护 |
| 参数模型 | 管理输入参数、分页参数、校验规则、默认值 | 支持基础类型和分页参数 |
| 字段映射 | 维护来源字段、输出字段、转换与脱敏规则 | 支持静态映射与脱敏标记 |
| 权限边界 | 定义内部公开、应用级、主体级权限范围 | 供 `open-api` 与管理端复用 |
| 结果验证 | 提供测试执行、结果预览、缓存策略校验 | 支持人工测试，不做复杂编排 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `01/02/03/05/06` 业务域 | 这些模块提供真实业务数据与事件，数据服务只做协议封装与复用，不改写主数据归属 |
| 上游 | `HJO2OA-Connector` | 需要访问外部资源时复用连接器能力 |
| 下游 | `HJO2OA-OpenApi` | 开放接口优先复用数据服务定义，不重复定义查询/命令语义 |
| 下游 | `HJO2OA-Report` | 报表可复用数据服务作为统计源之一，但统计口径仍由报表模块持有 |
| 下游 | `HJO2OA-DataSync` | 同步任务可复用服务定义发起标准读取或受控写入 |
| 非本模块 | 门户聚合与消息触达 | 门户首页聚合归 `04`，消息触达归 `06`，数据服务不直接面向用户侧入口 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 服务定义 | `DataServiceDefinition` | 标准化服务协议 | 图形化编排与脚本市场 |
| 参数建模 | `ServiceParameterDefinition` | 参数结构、校验与分页约束 | 复杂 DSL |
| 字段映射 | `ServiceFieldMapping` | 输出字段、脱敏和转换规则 | 实时流式转换管道 |
| 服务测试 | 预览结果、错误定位 | 管理端验证能力 | 开发者自助沙箱 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 管理员创建数据服务并选择类型、数据来源与权限模式 | 服务草稿 |
| 2 | 配置参数模型、分页规则、字段映射和脱敏策略 | 服务协议完整 |
| 3 | 绑定内部查询或连接器能力，执行人工测试 | 测试结果与预览 |
| 4 | 服务启用后被开放接口、报表或同步任务复用 | 可复用服务定义 |
| 5 | 服务状态变化进入治理链路，供审计和版本管理使用 | 审计记录与状态事件 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 不拥有业务主数据 | 只定义访问协议，不定义业务域核心表与主数据 |
| 不替代开放接口 | 对外鉴权、凭证、配额、限流归 `open-api`，本模块只管服务协议 |
| 不做复杂编排 | 一期不做服务流编排、脚本引擎、跨环境联调 |
| 分页显式建模 | 面向列表查询的分页、筛选必须进入参数模型，不允许隐式约定 |
| 映射结果可审计 | 字段映射与脱敏规则变更必须留痕，避免对外协议漂移 |
