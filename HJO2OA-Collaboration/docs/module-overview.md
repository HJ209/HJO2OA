# 05-协同办公应用-模块功能

## 模块定位

该模块负责承载平台上的通用协同办公业务应用，是对 O2OA 内置应用能力的集中对齐与优化实现。范围包括公文、会议、日程、任务、考勤、合同、固定资产、信息公告、企业网盘、人事等高频办公场景。

## 业务目标

- 以平台底座能力快速组装高频业务应用
- 让公文、会议、任务、考勤等能力彼此协同，而不是孤立系统
- 统一消息提醒、附件管理、审批轨迹、组织权限和搜索入口
- 支持行业项目按需增删业务应用

## 边界约束

- 本模块是平台底座能力在业务层的装配与落地集合，不拥有统一内容主模型、统一消息主模型和统一附件底层存储的所有权。
- 公告、制度、知识展示、网盘与文件协同等内容型场景必须优先复用 `03-内容与知识` 与 `00-附件中心` 的底座能力。
- 本模块可以定义业务语义、业务流程和业务入口，但不重复建设平台级内容底座、通用数据开放能力和统一触达底座。

## 子模块划分

### 5.1 公文管理

负责收文、发文、拟稿、核稿、签发、传阅、归档和查询。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 收文管理 | 收文登记、收文流转、收文归档 | 收文列表页、收文详情页、收文登记弹窗 |
| 发文管理 | 拟稿、核稿、签发、编号、正文、附件、签章 | 发文列表页、发文编辑页、正文编辑区 |
| 传阅与归档 | 传阅记录、归档规则、归档查询 | 传阅页、归档列表页 |
| 公文模板 | 公文模板定义、模板复用 | 模板选择器、模板管理页 |
| 电子签章 | 签章验证、签章留痕 | 签章展示区、签章验证状态 |

### 5.2 会议管理

负责会议室资源、会议申请、会议通知和纪要沉淀。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 会议室管理 | 会议室 CRUD、资源冲突校验、设备配置 | 会议室列表页、会议室新增/编辑弹窗 |
| 会议申请与审批 | 会议申请、流程审批、会议通知 | 会议申请页、审批处理页 |
| 签到签退 | 签到/签退记录、签到统计 | 签到页、签到统计展示 |
| 会议纪要 | 纪要编写、纪要关联会议、纪要归档 | 纪要编辑页、纪要详情页 |
| 会议状态跟踪 | 会议状态流转、自动提醒 | 会议列表页、状态标签 |

### 5.3 日程与任务管理

负责个人日程、团队日程和任务跟踪。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 个人日程 | 日程 CRUD、日程提醒、日程冲突检测 | 日程列表页、日程新增/编辑弹窗、日历视图 |
| 团队日程 | 团队日程共享、日程权限控制 | 团队日历视图、日程查看页 |
| 任务分配与跟踪 | 任务 CRUD、任务分配、进度反馈、延期提醒 | 任务列表页、任务详情页、进度条 |
| 任务完成评价 | 完成标准、评价记录 | 评价弹窗、评价记录查看 |

### 5.4 考勤与请假

负责考勤记录、请假出差等流程型场景。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 请假/出差/加班/补卡/外出 | 流程型考勤申请、审批、结果记录 | 申请列表页、申请表单页 |
| 考勤记录汇总 | 打卡记录、班次规则、异常检测 | 考勤汇总页、异常标记 |
| 考勤统计分析 | 按组织/人员/时间段统计、异常率分析 | 统计图表页、异常排行 |
| 班次与规则配置 | 班次定义、考勤规则、组织关联 | 班次配置页、规则配置页 |

### 5.5 合同与资产

负责合同台账和固定资产台账型场景。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 合同台账 | 合同 CRUD、合同审批、履约节点 | 合同列表页、合同详情页 |
| 履约与到期提醒 | 履约进度跟踪、到期自动提醒 | 履约进度条、到期提醒卡片 |
| 资产台账 | 资产 CRUD、资产分类、资产状态 | 资产列表页、资产详情页 |
| 资产流转 | 领用/借用/归还/盘点/报废流程 | 资产流转申请页、流转记录 |

### 5.6 信息公告与文件协同

负责基于统一内容底座和附件底座装配公告发布与企业网盘文件协同场景。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 公告发布 | 复用内容主模型完成公告发布、审批、发布范围配置 | 公告列表页、公告编辑页 |
| 网盘文件管理 | 复用附件中心完成文件夹、上传/下载/删除、分类管理 | 网盘浏览页、文件上传区 |
| 共享空间 | 基于统一权限与附件版本能力完成共享配置和版本控制 | 共享空间页、权限配置弹窗 |
| 文件预览 | 复用附件中心在线预览与格式支持能力 | 文件预览组件 |

### 5.7 跨应用联动与统一体验

负责各业务应用间的联动和统一入口。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 统一业务入口 | 应用菜单聚合、应用状态统计 | 业务应用入口页、应用图标区 |
| 跨应用跳转链路 | 待办->公文审批、会议->纪要、考勤->请假等跳转支持 | 跳转链接、关联信息展示 |
| 移动端高频场景 | 会议/日程/审批/任务/公告/文件移动端适配 | 移动端各业务页面 |

## 优化方向

- 不把每个业务应用做成孤岛，而是建立共享底座与统一体验
- 公文、会议、任务、考勤等能力全部接入统一待办、消息和搜索中心
- 统一附件和文档预览能力，减少业务模块重复建设
- 通过模板化和配置化能力提高行业化扩展效率
- 用平台能力驱动业务复用，而不是堆砌多个独立系统

## 典型业务场景

- 发文审批通过后自动归档并推送相关人员阅知
- 会议申请通过后自动写入会议安排并提醒参会人员
- 员工任务延期后同步在工作台和消息中心提示负责人
- 请假流程审批完成后自动影响考勤统计结果
- 合同到期前自动提醒经办人和管理者处理续签事项

## 交付边界

该模块是平台能力在业务层的落地集合，核心依赖组织权限、流程表单、内容发布、消息中心、附件中心与工作台能力。其设计应坚持业务可扩展、底座可复用的原则，不得绕过平台底座重复建设内容、消息、附件和数据开放能力。
