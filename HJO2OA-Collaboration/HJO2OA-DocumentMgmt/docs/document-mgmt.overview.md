# document-mgmt 模块概览

## 模块定位

| 项目 | 说明 |
|------|------|
| 模块定位 | 承载 `OfficialDocument` 聚合，统一管理收文、发文、拟稿、签发、传阅、归档等公文业务台账与状态机。 |
| 领域对象 | `OfficialDocument`、`DocumentCirculationRecord`、`DocumentSealRecord`。 |
| 一期范围 | 收发文台账、审批接入、文号生成、传阅记录、签章留痕、归档查询。 |
| 不负责内容 | 不自建审批引擎、不自建待办中心、不持有附件二进制、不建设独立内容底座与全文检索。 |

## 核心职责

| 职责 | 说明 | 边界口径 |
|------|------|----------|
| 公文主档管理 | 维护公文标题、文种、密级、紧急程度、主办组织、拟稿人、文号和业务状态。 | 公文业务语义与状态归 `05`。 |
| 审批接入与结果回写 | 按需发起 `02` 流程审批，并将审批结果回写为 `IN_APPROVAL`、`EFFECTIVE`、`VOIDED` 等业务状态。 | 审批实例与待办归 `02`。 |
| 传阅与签章留痕 | 记录传阅对象、阅读状态、签章类型、签章人和时间。 | 正文与签章证据只保存引用。 |
| 归档与事件输出 | 归档后冻结主档关键字段，并对外发布公文提交、生效、归档事件。 | 消息触达归 `06`，门户展示归 `04`。 |

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `01` 身份与权限 | 提供拟稿人、主办组织、可见范围、数据权限校验。 |
| 上游 | `02` 流程审批 | 提供流程发起、审批处理、流程完成/终止回调。 |
| 上游 | `00` 附件中心 | 提供正文附件、签章证据、归档附件的元数据与存储引用。 |
| 下游 | `06` 消息与触达 | 消费 `biz.document.submitted`、`biz.document.approved`、`biz.document.archived` 生成提醒。 |
| 下游 | `04` 门户与办公中心 | 消费公文读模型展示待处理、公文卡片和快捷入口。 |
| 下游 | `03` 内容与知识 | 仅按事件消费公文已生效/已归档事实，不回写公文主档。 |

## 核心能力

| 能力 | 说明 | 一期口径 |
|------|------|----------|
| 收文登记 | 登记来文基础信息、来源单位、收文时间、处理意见和附件引用。 | 支持列表、详情、登记和归档。 |
| 发文拟稿与提交 | 拟稿、保存草稿、提交审批、审批通过后生成文号并生效。 | 不扩展复杂模板中心。 |
| 传阅追踪 | 发起传阅、记录已读/未读、保留传阅轨迹。 | 只做追加式记录，不覆盖历史轨迹。 |
| 签章与归档 | 记录签章信息并在归档后冻结主档。 | 已归档公文只读。 |

## 典型业务流

| 业务流 | 步骤 |
|--------|------|
| 发文生效 | 拟稿保存 -> 提交 `02` 审批 -> `process.instance.completed` 回写生效 -> 生成文号/签章记录 -> 发布 `biz.document.approved`。 |
| 收文传阅 | 收文登记 -> 指定传阅对象 -> 接收人查看并更新阅读状态 -> 形成传阅台账 -> 按规则归档。 |
| 归档闭环 | 生效公文满足归档条件 -> 归档操作冻结关键字段 -> 发布 `biz.document.archived` -> 供 `03/04/06` 消费。 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 审批统一复用 `02` | 公文模块不得自建流程定义、审批节点和待办箱。 |
| 正文与附件统一复用 `00` | 仅保存 `attachmentId` 或证据引用，不复制附件内容。 |
| 内容型扩展统一复用 `03` | 不在本模块自建正文发布、全文检索和知识归档能力。 |
| 状态以业务主档为准 | 流程完成不是唯一真相，必须显式回写公文业务状态。 |
