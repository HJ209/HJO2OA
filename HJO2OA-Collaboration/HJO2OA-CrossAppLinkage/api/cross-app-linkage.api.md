# cross-app-linkage API 说明

## 接口分组

| 分组 | 关键接口 | 关键查询/写入 | 分页与筛选 | 幂等 | 权限与审计 | 错误处理 |
|------|----------|---------------|------------|------|------------|----------|
| 统一入口查询 | `GET /linkage/entries` | 查询当前用户可见的业务入口、角标、快捷操作和跳转元数据。 | 支持按 `scene`、`deviceType`、`keyword` 过滤。 | 查询天然幂等。 | 可见范围完全复用 `01` 与源模块授权；访问写入口审计。 | `403` 无访问权限，`404` 入口未注册。 |
| 状态聚合查询 | `GET /linkage/status-summary` | 查询最近会议、待反馈任务、临期合同、异常考勤等聚合状态。 | 支持按 `summaryType`、`priority`、`updatedAt` 筛选。 | 查询天然幂等。 | 结果只返回用户有权限查看的业务摘要；访问写审计。 | `422` 聚合源不可用，`500` 投影刷新失败。 |
| 深链跳转解析 | `POST /linkage/resolve` | 根据 `bizType + bizId + sourceScene` 解析目标页面、参数和回跳上下文。 | 不分页。 | 按 `bizType + bizId + sourceScene + requestId` 幂等。 | 解析前再次校验源模块权限；记录跳转来源、目标和操作者。 | `403` 源对象不可见，`404` 路由不存在，`409` 跳转目标已失效。 |
| 聚合索引刷新 | `POST /linkage/index/refresh` `POST /linkage/routes/reload` | 供系统内部刷新入口索引、重建路由和状态缓存。 | 不分页。 | 按 `refreshScope + requestId` 幂等。 | 仅系统内部或平台管理员调用；写系统审计。 | `500` 刷新失败需重试，`422` 刷新范围非法。 |

## 通用约束

| 主题 | 说明 |
|------|------|
| 业务写入边界 | 不提供业务主档写接口，所有业务修改仍回到源模块。 |
| 权限边界 | 权限与数据范围完全复用 `01` 和源模块，不维护第二套授权规则。 |
| 缓存边界 | 聚合索引和状态摘要属于可重建缓存，不可作为业务真相源。 |
| 审计要求 | 入口访问、深链跳转、缓存刷新都必须记录来源场景和操作上下文。 |
| 开放边界 | 对外开放的统一入口或超级链接协议统一经 `07` 暴露，本模块只负责域内编排。 |
