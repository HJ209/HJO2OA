# cross-app-linkage 模块概览

## 模块定位

| 项目 | 说明 |
|------|------|
| 模块定位 | 作为统一入口与体验编排层，消费各业务模块读模型和事件，提供跨应用跳转、状态标签和移动端高频入口。 |
| 持有对象 | 不形成独立业务主数据，只维护轻量路由、聚合索引和缓存型编排结果。 |
| 一期范围 | 统一入口、深链跳转解析、业务状态聚合、移动端高频场景入口。 |
| 不负责内容 | 不直接写业务主表、不自建审批/内容/附件模型、不承接复杂流程编排引擎。 |

## 核心职责

| 职责 | 说明 | 边界口径 |
|------|------|----------|
| 统一入口聚合 | 将公文、会议、日程、任务、考勤、合同、共享空间等入口统一汇总。 | 门户首页编排归 `04`，本模块只提供入口读模型。 |
| 跨应用跳转解析 | 根据业务对象类型、来源和状态解析跳转目标与回跳上下文。 | 业务详情仍由源模块提供。 |
| 状态标签与时间线聚合 | 消费业务事件，汇总展示状态标签、最近动作和高频待处理入口。 | 业务真相源仍归源模块。 |
| 移动端高频体验 | 收敛移动端会议、日程、任务、考勤、公告等高频访问入口。 | 不复制第二套权限和状态规则。 |

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `05` 各业务子模块 | 提供业务事件、读模型、路由目标和状态事实。 |
| 上游 | `01` 身份与权限 | 提供用户身份、组织范围、角色权限和可见应用集合。 |
| 下游 | `04` 门户与办公中心 | 消费统一入口与状态聚合结果进行首页展示。 |
| 下游 | `06` 消息与触达 | 基于消息或提醒中的业务对象定位跳转页面。 |
| 下游 | 移动端容器 | 消费移动端高频入口和深链解析结果。 |

## 核心能力

| 能力 | 说明 | 一期口径 |
|------|------|----------|
| 统一入口 | 聚合业务入口、状态角标、快捷访问。 | 按权限显示，不自定义业务规则。 |
| 深链跳转 | 从门户、消息、待处理卡片跳转到源模块详情。 | 只解析路由，不写业务数据。 |
| 状态聚合 | 聚合最近会议、待反馈任务、临期合同等状态摘要。 | 基于事件驱动刷新，不直接查询全量主表。 |
| 移动端高频场景 | 提供会议、日程、任务、考勤、公告等移动入口。 | 先覆盖高频场景，不做复杂工作台编排。 |

## 典型业务流

| 业务流 | 步骤 |
|--------|------|
| 门户入口访问 | `04` 请求统一入口 -> 本模块按用户权限返回可见业务入口与状态角标 -> 点击后深链到源模块详情。 |
| 消息跳转 | `06` 消息携带业务对象标识 -> 本模块解析路由与回跳上下文 -> 打开源模块页面。 |
| 事件驱动刷新 | 消费 `biz.*` 事件 -> 更新入口聚合缓存与状态标签 -> 门户和移动端读取最新投影。 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 只做编排不做持有 | 不形成独立主数据，不复制业务台账和状态机。 |
| 门户统一复用 `04` | 本模块不直接编排首页，只输出入口和状态读模型。 |
| 触达统一复用 `06` | 本模块只提供跳转能力，不管理送达渠道和消息模板。 |
| 编排能力收敛 | 一期仅支持入口、跳转、状态聚合和移动高频入口，不做复杂流程编排。 |
