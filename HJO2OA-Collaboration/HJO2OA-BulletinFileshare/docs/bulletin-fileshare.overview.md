# bulletin-fileshare 模块概览

## 模块定位

| 项目 | 说明 |
|------|------|
| 模块定位 | 作为公告与共享文件场景装配层，承载 `SharedSpace` 聚合并聚合已发布内容、附件和外部链接。 |
| 领域对象 | `SharedSpace`、`SpaceMember`、`SpaceResourceBinding`。 |
| 一期范围 | 共享空间、成员权限、资源绑定、公告场景聚合展示、文件入口装配。 |
| 不负责内容 | 不拥有公告正文、不自建内容发布与检索模型、不持有附件二进制、不建设复杂网盘目录树和多人协作版本。 |

## 核心职责

| 职责 | 说明 | 边界口径 |
|------|------|----------|
| 公告场景装配 | 聚合 `03` 已发布内容形成公告入口与列表。 | 公告正文与发布流程归 `03`。 |
| 共享空间管理 | 维护共享空间编码、类型、状态、可见范围和管理人。 | 共享空间主档归 `05 bulletin-fileshare`。 |
| 成员与权限绑定 | 管理组织、角色、岗位、人员级别的空间成员与权限。 | 统一复用 `01` 主体标识。 |
| 资源绑定与失效治理 | 绑定内容、附件或外部链接，并处理失效引用。 | 附件与正文主数据分别归 `00/03`。 |

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `03` 内容与知识 | 提供公告正文、制度文章、知识内容的发布事实与内容引用。 |
| 上游 | `00` 附件中心 | 提供共享文件入口所需的附件元数据与预览/下载能力。 |
| 上游 | `01` 身份与权限 | 提供空间成员主体、角色、组织和数据权限范围。 |
| 下游 | `04` 门户与办公中心 | 展示公告卡片、共享空间入口和资源快捷访问。 |
| 下游 | `06` 消息与触达 | 消费空间变更事件，按需发送空间提醒。 |

## 核心能力

| 能力 | 说明 | 一期口径 |
|------|------|----------|
| 公告列表聚合 | 聚合 `03` 已发布内容，形成公告浏览入口。 | 只读展示，不提供正文编辑。 |
| 共享空间主档 | 创建团队/部门/项目/公共空间，控制状态与可见范围。 | 先做基础空间与权限。 |
| 成员权限管理 | 维护 `VIEWER/EDITOR/MANAGER` 成员权限。 | 权限主体支持组织、岗位、角色、人员。 |
| 资源绑定 | 绑定内容、附件、链接三类资源。 | 仅保存轻量引用与排序，不复制资源实体。 |

## 典型业务流

| 业务流 | 步骤 |
|--------|------|
| 公告场景展示 | `03` 发布文章 -> 本模块消费 `content.article.published` -> 刷新公告场景入口 -> 用户跳转阅读正文。 |
| 共享空间创建 | 新建空间 -> 配置成员权限 -> 绑定附件或链接 -> 发布 `biz.space.updated`。 |
| 资源失效治理 | 接收 `infra.attachment.deleted` -> 标记绑定失效 -> 管理员重新绑定资源。 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 不拥有内容底层模型 | 公告正文、制度文章、知识内容统一归 `03`，本模块只做场景装配。 |
| 不拥有附件底层存储 | 文件预览、下载、二进制存储统一复用 `00`。 |
| 共享空间轻量化 | 先支持成员、权限、资源绑定，不做复杂目录树、版本协作和多人编辑。 |
| 门户展示统一复用 `04` | 本模块只提供场景读模型与入口，不编排门户首页模板。 |
