# schedule-task 模块概览

## 模块定位

| 项目 | 说明 |
|------|------|
| 模块定位 | 承载 `ScheduleEntry` 与 `TaskAssignment` 两个聚合，负责时间安排、提醒规则、任务分配和执行反馈。 |
| 领域对象 | `ScheduleEntry`、`ReminderRule`、`TaskAssignment`、`TaskFeedback`。 |
| 一期范围 | 个人/团队日程、会议联动系统日程、任务分配、反馈、延期、逾期提醒。 |
| 不负责内容 | 不替代 `02` 待办中心，不做复杂项目排程、甘特图和资源计划引擎。 |

## 核心职责

| 职责 | 说明 | 边界口径 |
|------|------|----------|
| 日程台账管理 | 管理个人与团队日程、可见范围、时间占用和提醒规则。 | 日程业务状态与提醒规则归 `05`。 |
| 任务执行闭环 | 管理任务创建、分配、反馈、延期、完成和逾期。 | 任务是业务执行台账，不是流程待办。 |
| 跨模块时间联动 | 消费会议等事件生成系统日程，保持来源对象与本模块日程关联。 | 来源业务主档仍归源模块持有。 |
| 事件与提醒输出 | 发布日程创建/更新、任务分配/完成/逾期事件供门户和消息消费。 | 送达策略由 `06` 实现。 |

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `01` 身份与权限 | 提供个人、团队、组织和数据权限范围。 |
| 上游 | `05 meeting-mgmt` | 消费会议创建/取消事件生成或撤销系统日程。 |
| 上游 | `02` 流程审批 | 仅消费流程结果形成来源型系统日程，不复制待办和审批节点。 |
| 下游 | `06` 消息与触达 | 基于提醒规则和任务状态发送站内提醒或移动提醒。 |
| 下游 | `04` 门户与办公中心 | 展示我的日程、我的任务、逾期事项和快捷入口。 |

## 核心能力

| 能力 | 说明 | 一期口径 |
|------|------|----------|
| 个人/团队日程 | 创建个人或团队日程，支持冲突提示和可见范围控制。 | 冲突提示不强制拦截。 |
| 提醒规则 | 为日程维护提前提醒分钟数和渠道偏好。 | 渠道实际送达复用 `06`。 |
| 任务分配 | 为个人任务指定负责人、截止时间、优先级和来源业务。 | 支持来自会议/公文/合同等业务来源。 |
| 反馈与延期 | 记录开始、评论、延期、完成等反馈动作。 | 反馈采用追加式记录。 |

## 典型业务流

| 业务流 | 步骤 |
|--------|------|
| 个人日程创建 | 新建日程 -> 校验时间段并提示冲突 -> 保存提醒规则 -> 发布 `biz.schedule.created`。 |
| 会议联动日程 | 接收 `biz.meeting.created` -> 生成只读系统日程 -> 会议取消时接收 `biz.meeting.cancelled` 并同步关闭日程。 |
| 任务闭环 | 创建任务 -> 指派负责人 -> 负责人反馈进度/延期 -> 到期未完成触发 `biz.task.overdue` -> 完成后发布 `biz.task.completed`。 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 不替代待办中心 | 流程待办、审批动作统一留在 `02`，任务仅表达业务执行事项。 |
| 不建设复杂排程 | 暂不支持复杂资源排程、依赖链、甘特图和跨项目计划引擎。 |
| 提醒统一复用 `06` | 本模块只维护提醒规则与触发时点，不负责送达通道。 |
| 来源型日程只读 | 来源于会议或流程的系统日程不可随意编辑，只允许源业务驱动变更。 |
