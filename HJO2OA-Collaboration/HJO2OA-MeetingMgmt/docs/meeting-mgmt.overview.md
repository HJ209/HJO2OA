# meeting-mgmt 模块概览

## 模块定位

| 项目 | 说明 |
|------|------|
| 模块定位 | 承载 `MeetingPlan` 聚合，负责会议申请、排期、参会人管理、签到和纪要沉淀。 |
| 领域对象 | `MeetingPlan`、`MeetingParticipant`、`MeetingSignRecord`、`MeetingMinutes`。 |
| 一期范围 | 会议申请、会议室冲突校验、审批接入、签到签退、纪要发布、取消闭环。 |
| 不负责内容 | 不自建消息送达中心、不自建门户首页、不做视频会议平台接入和复杂会务资源运维。 |

## 核心职责

| 职责 | 说明 | 边界口径 |
|------|------|----------|
| 会议主档与状态机 | 维护会议主题、时间、会议室、组织者、状态和审批引用。 | 会务状态归 `05`。 |
| 排期与冲突校验 | 校验会议室时间冲突、会务资源占用、参会人名单。 | 资源冲突规则由会议模块维护。 |
| 签到签退留痕 | 记录签到/签退事实和统计信息。 | 只记录事实，不以签到反向改写审批流。 |
| 纪要沉淀与事件输出 | 会议完成后产出纪要并发布会议创建、取消、纪要发布事件。 | 内容引用与附件存储分别复用 `03/00`。 |

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `01` 身份与权限 | 提供组织者、参会人、会议室管理员的身份和数据权限。 |
| 上游 | `02` 流程审批 | 会议申请审批、流程回调和审批摘要复用 `02`。 |
| 上游 | `00` 附件中心 | 会议通知附件、纪要附件、签到佐证均走附件引用。 |
| 下游 | `05 schedule-task` | 消费 `biz.meeting.created`、`biz.meeting.cancelled` 生成或取消系统日程。 |
| 下游 | `06` 消息与触达 | 基于会议创建、取消、纪要发布事件发送提醒。 |
| 下游 | `04` 门户与办公中心 | 展示会议卡片、我的会议、会议纪要快捷入口。 |

## 核心能力

| 能力 | 说明 | 一期口径 |
|------|------|----------|
| 会议申请与排期 | 填报会议主题、时间、地点、参会范围并发起审批。 | 覆盖线上/线下/混合三种类型。 |
| 会议室冲突校验 | 会议排期时校验同一会议室时段不可重复占用。 | 以会议室和时间段为核心约束。 |
| 签到签退 | 记录参会人的签到/签退事实和统计。 | 不做复杂定位设备接入。 |
| 纪要发布 | 会议完成后沉淀纪要正文引用和附件。 | 纪要仅对已完成会议开放。 |

## 典型业务流

| 业务流 | 步骤 |
|--------|------|
| 会议排期 | 发起会议申请 -> 冲突校验 -> 提交 `02` 审批 -> 回写 `SCHEDULED` -> 发布 `biz.meeting.created`。 |
| 会中签到 | 会议进入 `ONGOING` -> 参会人签到/签退 -> 形成签到统计 -> 保留事实记录供纪要引用。 |
| 纪要闭环 | 会议 `COMPLETED` -> 录入纪要正文引用/附件 -> 发布 `biz.meeting.minutes-published` -> 门户和消息更新。 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 审批统一复用 `02` | 会议模块只维护业务状态，不内嵌审批节点模型。 |
| 消息统一复用 `06` | 不在本模块自建通知模板和送达渠道。 |
| 内容与附件分层复用 | 纪要正文引用复用内容能力，附件与佐证复用 `00`。 |
| 高阶会务能力暂缓 | 不做座次管理、视频会议平台深集成、复杂设备资源运维。 |
