# contract-asset 模块概览

## 模块定位

| 项目 | 说明 |
|------|------|
| 模块定位 | 承载 `ContractLedger` 与 `AssetLedger` 两类台账聚合，负责合同履约提醒和固定资产状态流转。 |
| 领域对象 | `ContractLedger`、`ContractMilestone`、`ContractReminder`、`AssetLedger`、`AssetTransferRecord`。 |
| 一期范围 | 合同台账、履约节点、到期提醒、资产台账、领用/借用/归还/盘点/报废记录。 |
| 不负责内容 | 不实现复杂财务核算、折旧、采购链路、设备维护体系和对外开放接口治理。 |

## 核心职责

| 职责 | 说明 | 边界口径 |
|------|------|----------|
| 合同主档管理 | 维护合同编号、类型、相对方、生效区间、金额和状态。 | 合同业务台账归 `05`。 |
| 履约节点与提醒 | 管理履约节点、提醒规则和到期预警。 | 具体提醒送达归 `06`。 |
| 资产主档与流转 | 维护资产编码、分类、保管人、状态和流转记录。 | 资产状态变更必须留痕。 |
| 审批接入与结果回写 | 需要审批的合同激活或资产流转动作统一接入 `02` 并显式回写主档。 | 审批实例与待办不在本模块持有。 |

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `01` 身份与权限 | 提供合同归属组织、经办人、资产保管人和数据权限。 |
| 上游 | `02` 流程审批 | 提供合同审批、报废/特殊流转审批和结果回写。 |
| 上游 | `00` 附件中心 | 提供合同扫描件、补充协议、资产佐证等附件引用。 |
| 下游 | `06` 消息与触达 | 消费合同临期和资产状态变化事件发送提醒。 |
| 下游 | `04` 门户与办公中心 | 展示到期合同、资产待处理事项和台账入口。 |
| 下游 | `07` 开放集成 | 对外同步台账、报表和第三方接口统一复用 `07`。 |

## 核心能力

| 能力 | 说明 | 一期口径 |
|------|------|----------|
| 合同台账 | 维护合同主档、相对方、金额和生效区间。 | 先做基础台账，不做财务核算。 |
| 履约节点与提醒 | 维护履约节点、到期规则、临期事件。 | 由调度或规则引擎触发提醒，不依赖人工轮询。 |
| 资产台账 | 维护资产编码、分类、保管人与状态。 | 先做基础台账与状态流转。 |
| 资产流转留痕 | 记录领用、借用、归还、盘点、报废。 | 所有状态变化必须产生流转记录。 |

## 典型业务流

| 业务流 | 步骤 |
|--------|------|
| 合同生效 | 新建合同草稿 -> 提交 `02` 审批 -> 回写 `ACTIVE` -> 生成履约节点与提醒规则。 |
| 合同临期提醒 | 调度扫描到期时间 -> 命中提醒规则 -> 发布 `biz.contract.expiring` -> `06/04` 刷新提醒视图。 |
| 资产流转 | 发起领用/借用/归还/报废 -> 记录流转动作 -> 更新资产状态 -> 发布 `biz.asset.status-changed`。 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 台账优先 | 先实现基础合同/资产台账，不做复杂采购、预算、折旧和维护体系。 |
| 审批统一复用 `02` | 合同生效、特殊流转等审批统一接入 `02`，本模块只做结果回写。 |
| 附件统一复用 `00` | 不复制合同扫描件、资产照片等附件实体，只保留引用。 |
| 对外开放统一复用 `07` | 外部系统同步、报表开放和 API 治理不在本模块内建设。 |
