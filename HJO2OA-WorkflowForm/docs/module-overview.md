# 02-流程与表单-模块功能

## 模块定位

该模块是对标 O2OA 的核心能力之一，负责承载流程建模、流程发起、流程流转、审批处理、待办中心、表单设计、表单渲染、流程监控和流程归档等关键能力。它既是平台中最核心的事务引擎，也是大部分协同业务应用的承载基础。

## 业务目标

- 支持通用审批流、复杂会签流、分支流、并行流和条件路由
- 支持表单与流程绑定、字段权限控制、流程节点操作控制
- 支持草稿、提交、撤回、退回、转办、加签、阅办、抄送等常见能力
- 支持统一待办、已办、我发起、抄送我、关注流程等视图
- 支持公文、合同、费用、请假、采购、会议等业务流程接入

## 子模块划分

### 2.1 流程定义与建模

负责流程模型的设计与版本管理，是流程引擎的配置入口。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 流程模型管理 | 流程定义 CRUD、流程分类、启停用 | 流程列表页、流程新增/编辑入口 |
| 节点与路由配置 | 节点类型（审批/抄送/条件/并行）、路由规则、参与者规则 | 流程设计器画布、节点属性面板、路由连线配置 |
| 审批动作配置 | 节点可用动作（通过/驳回/退回/转办/加签/撤回/催办）、动作条件 | 节点动作勾选、动作条件编辑器 |
| 流程版本管理 | 版本发布、历史版本保留、版本回滚、变更影响提示 | 版本列表页、版本对比、发布确认弹窗 |
| 参与者规则 | 组织匹配、岗位匹配、角色匹配、表单字段匹配、动态计算 | 参与者规则配置面板、组织/岗位/角色选择器 |

### 2.2 流程实例与任务管理

负责流程发起后的实例和任务生命周期管理。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 流程发起 | 创建流程实例、首节点任务生成、表单数据绑定 | 流程发起入口页、流程分类选择、表单填写页 |
| 任务流转 | 任务状态机、提交/通过/驳回/退回/转办/加签/撤回 | 审批处理页、动作按钮区、意见填写区 |
| 审批记录与轨迹 | 审批记录持久化、操作轨迹、抄送记录 | 审批轨迹时间线、节点状态展示 |
| 草稿管理 | 草稿保存、草稿恢复、草稿提交 | 草稿保存按钮、草稿列表、草稿恢复 |
| 流程归档 | 流程结束后归档、归档数据查询 | 归档列表页、归档详情查看 |

### 2.3 审批动作与规则引擎

负责审批操作的统一抽象和规则判定。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 统一审批动作 | 提交/通过/驳回/退回/转办/加签/撤回/催办/阅知 | 动作按钮区、动作权限控制 |
| 动作规则判定 | 节点规则、表单状态、操作者身份、权限校验 | 动作按钮可用/禁用状态 |
| 动态参与者计算 | 条件判断、组织关系匹配、岗位匹配、角色匹配 | 参与者预览、计算结果展示 |
| 异常补偿 | 流程异常中断、人工干预、强制终止 | 异常处理入口、干预操作确认 |

### 2.4 表单元数据与渲染协议

负责表单定义和动态渲染的数据协议。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 表单字段定义 | 字段类型、校验规则、联动规则、默认值、字典引用 | 表单设计器字段配置面板 |
| 表单布局定义 | 行列布局、分组、Tab 页、子表布局 | 表单设计器布局画布 |
| 字段权限映射 | 流程节点与字段权限（可编辑/只读/隐藏）映射 | 节点权限预览、字段权限矩阵 |
| 表单渲染协议 | 标准化元数据输出、版本管理、发布机制 | 表单渲染器输入协议 |
| 表单版本管理 | 定义版本、发布、回滚 | 版本切换、版本对比 |

### 2.5 表单渲染器

负责根据元数据协议动态渲染表单页面。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 基础字段渲染 | 文本/数字/日期/字典/人员/组织/附件/富文本/图片 | 表单渲染器组件库 |
| 子表与明细表 | 子表增删行、明细表嵌套 | 子表渲染组件、行操作按钮 |
| 字段联动 | 值变化触发显隐/必填/可选/默认值联动 | 联动规则引擎、实时响应 |
| 多态渲染 | 创建态/编辑态/查看态/审批态差异化渲染 | 状态切换、字段权限控制 |
| 移动端表单 | 移动端适配渲染、简化布局 | 移动端表单渲染器 |

### 2.6 待办中心与聚合视图

负责统一聚合所有流程相关任务。待办是事务处理视图，消息是通知触达视图，两者联动但不混同。待办中心负责待办/已办/我发起/抄送/草稿等任务列表的聚合与流转；消息提醒由 `06-消息移动与生态` 统一负责，待办事件到消息提醒的映射规则在两模块间约定。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 待办列表 | 待办任务聚合、按类型/紧急程度/组织/时间筛选 | 待办列表页、筛选区、快捷跳转 |
| 已办列表 | 已办任务聚合、处理结果查看 | 已办列表页、处理结果标签 |
| 我发起 | 发起记录聚合、流程状态跟踪 | 我发起列表页、状态标签 |
| 抄送我 | 抄送记录聚合、阅知标记 | 抄送列表页、批量阅知 |
| 草稿箱 | 草稿聚合、继续编辑/删除 | 草稿列表页、编辑入口 |
| 归档查询 | 归档数据聚合、全文检索 | 归档查询页、高级筛选 |

### 2.7 流程监控与分析

负责流程效率分析和异常预警。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 流程耗时分析 | 平均耗时、节点耗时、超时统计 | 耗时分析图表页、趋势图 |
| 节点停滞分析 | 停滞节点、停滞时长、停滞原因 | 停滞分析页、节点排行 |
| 审批拥塞分析 | 待办堆积量、审批人负载、退回率 | 拥塞分析图表页、人员排行 |
| 超时预警与催办 | 超时规则配置、自动催办、催办记录 | 预警规则配置页、催办记录查看 |

## 优化方向

- 流程引擎与业务单据解耦，降低后续业务扩展成本
- 表单协议统一，避免不同业务重复建设表单页面
- 待办中心统一，避免用户在多个业务系统之间切换
- 增强流程监控、性能分析和异常处理能力
- 支持移动端审批与消息直达，提高处理效率
- 流程设计器与表单设计器共享设计器框架，避免重复建设

## 典型业务场景

- 员工发起请假申请并按组织层级流转审批
- 公文发文流程中支持会签、核稿、签发与归档
- 合同审批流中根据金额、部门、风险等级走不同审批路径
- 管理员分析某类流程平均耗时并优化审批节点
- 用户在移动端直接处理待办并查看附件与审批记录

## 边界约束

- 待办是事务处理视图，消息是通知触达视图，两者联动但不混同。
- 待办中心负责待办/已办/我发起/抄送/草稿等任务列表的聚合与流转；消息提醒统一由 `06-消息移动与生态` 负责路由与触达。
- 待办事件到消息提醒的映射规则在本模块与 `06` 之间约定，本模块发布待办事件，`06` 消费并决定触达渠道。
- 身份上下文切换后，待办列表、待办数量和门户待办卡片都必须同步刷新。

## 交付边界

该模块负责事务流转能力和表单底座，但具体业务语义仍由公文、合同、会议、人事等业务模块定义。流程模块必须提供统一且稳定的能力支撑。
