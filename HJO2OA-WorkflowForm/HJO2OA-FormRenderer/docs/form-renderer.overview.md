# form-renderer 概览

## 模块定位

`form-renderer` 是 `02-流程与表单` 的前端渲染层子模块，负责消费 `form-metadata` 输出的标准渲染协议，并结合实例/任务上下文将表单渲染为创建态、编辑态、查看态和审批态页面。它不拥有独立领域模型，也不负责业务数据落库。

## 核心职责

- 将字段、布局、校验、联动和字段权限映射协议渲染为统一组件树。
- 根据页面模式、当前节点和身份上下文计算字段最终状态。
- 收集用户输入、执行本地校验并向上游提交 `formDataPatch`。
- 提供附件、组织/人员选择、子表编辑等标准组件行为。

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `form-metadata` | 只消费标准渲染协议，不解析设计器草稿内部结构。 |
| 上游 | `process-instance` | 读取实例/任务上下文、当前节点、已有表单值和模式。 |
| 上游 | `action-engine` | 将当前表单补丁交给动作接口提交，不自己执行业务动作。 |
| 上游 | `00-基础设施` / `01-组织与权限` | 复用附件、字典、人员/组织选择器能力。 |
| 边界外 | 草稿、审批、归档 | 发起草稿、审批动作、归档查询都由其他子模块负责。 |

## 核心能力表

| 能力 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| 基础字段渲染 | 字段协议、当前值 | 组件树 | 只支持已定义字段类型。 |
| 子表与附件 | 子表定义、附件协议 | 明细行编辑、上传/预览 | 上传走基础设施附件接口。 |
| 联动与校验 | 联动规则、校验规则 | 本地状态刷新、错误提示 | 一期仅支持简单联动。 |
| 多态渲染 | `create/edit/view/approve` 模式 | 不同交互态表单 | 字段最终状态由模式和权限叠加决定。 |

## 典型业务流

1. 页面进入后读取渲染协议、实例/任务上下文和已有表单值。
2. 渲染器根据当前模式和节点字段权限映射计算每个字段的最终状态。
3. 用户输入触发联动和本地校验，必要时更新显隐、必填和默认值。
4. 用户保存草稿或提交动作时，渲染器将 `formDataPatch` 交给上游接口。
5. 服务端返回最新上下文后，页面重新渲染，不在前端维护持久业务状态。

## 一期实现约束

- 一期仅提供 PC 端完整渲染能力，不单独建设移动端渲染器；如移动端入口复用现有协议，仅限基础字段与只读详情场景。
- 联动规则仅支持显隐、必填、默认值和只读切换，不支持脚本执行。
- 渲染器不自行计算业务权限，只消费服务端返回的字段状态协议。
- 不发布领域事件，不建设独立后端持久化。
