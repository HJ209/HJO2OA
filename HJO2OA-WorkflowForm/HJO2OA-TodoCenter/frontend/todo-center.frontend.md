# todo-center 前端

## 页面清单

| 页面 | 适用端 | 页面职责 |
|------|--------|----------|
| 待办列表页 | PC/移动端 | 展示当前身份下可处理待办及快捷跳转入口。 |
| 已办列表页 | PC/移动端 | 展示已完成处理记录和处理结果摘要。 |
| 我发起列表页 | PC/移动端 | 展示自己发起的流程及当前状态。 |
| 抄送列表页 | PC/移动端 | 展示抄送/阅知项并支持标记已阅。 |
| 草稿列表页 | PC | 展示待继续编辑的发起草稿。 |
| 归档查询页 | PC | 提供归档检索和详情入口。 |

## 页面职责

- 待办、已办和抄送页是用户事务入口，不负责审批动作执行本身。
- 我发起页关注实例状态追踪，不展示审批动作按钮。
- 草稿页只负责恢复编辑或删除草稿，不负责草稿落库逻辑。
- 归档页只读展示实例结果，禁止再次处理或修改。

## 关键交互

1. 页面切换身份上下文后，所有列表和数量角标必须同时刷新。
2. 待办列表点击后跳转审批页或实例详情页，不在列表内直接执行动作。
3. 抄送页支持单条或批量标记已阅。
4. 草稿页点击继续编辑时跳转到发起表单页并恢复草稿。
5. 归档页支持标题、流程分类、发起人、结束时间等高级筛选。

## 字段状态、节点状态、权限状态约束

- 列表字段状态：
  - 待办显示紧急程度、到期状态、当前节点。
  - 抄送显示 `UNREAD/READ`。
  - 已办显示处理结果和完成时间。
- 节点状态：
  - 待办只展示当前活动节点。
  - 已办和归档展示历史节点摘要，不允许误认为仍可处理。
- 权限状态：
  - 所有列表均按当前身份裁剪。
  - 管理员如需查看全量运行态，应走实例管理页，不复用普通待办列表。

## 端差异

- 移动端重点提供待办、已办、我发起和抄送四类基础视图，筛选条件做收敛。
- 草稿批量管理、归档高级检索和复杂筛选保留在 PC。
- 消息提醒入口不在本子模块页面内呈现，仍由 `06` 的消息视图承接。
