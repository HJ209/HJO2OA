# todo-center 概览

## 模块定位

`todo-center` 是 `02-流程与表单` 的读模型与聚合查询子模块，负责 `TodoItem` 投影及待办相关列表聚合。它统一提供待办、已办、我发起、抄送我、草稿和归档入口，但不承担审批动作执行。待办是事务处理视图，消息触达由 `06-消息移动与生态` 负责。

## 核心职责

- 基于 `process.task.*` 事件维护 `todo_item` 投影。
- 聚合待办、已办、我发起、抄送我、草稿、归档和数量统计视图。
- 提供统一筛选、分页、排序和快捷跳转入口。
- 处理抄送阅知状态、待办逾期展示和身份切换后的列表刷新。

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `process-instance` | 实例、任务是权威运行时数据；`todo-center` 只维护投影和聚合查询。 |
| 上游 | `06-消息移动与生态` | 消息只负责通知触达；待办中心不管理站内信、短信、邮件渠道。 |
| 上游 | `01-组织与权限` | 列表和数量必须受当前身份上下文约束，身份切换后立即刷新。 |
| 下游 | `04-门户与工作台` / 业务应用 | 通过聚合查询接口展示待办卡片、列表和快捷入口。 |
| 边界外 | 审批动作、业务表单数据 | 不负责任务处理、不保存业务单据正文。 |

## 核心能力表

| 能力 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| 待办投影维护 | `process.task.*` 事件 | `todo_item` | 事件消费必须幂等。 |
| 列表聚合查询 | 当前身份、筛选条件 | 待办/已办/我发起/抄送列表 | 只返回当前身份可见数据。 |
| 草稿与归档聚合 | 草稿、归档权威表查询结果 | 草稿箱、归档列表 | 草稿/归档不在 `todo_item` 独立建模。 |
| 数量统计 | 当前身份上下文 | 待办数量、抄送未读数量等 | 身份切换后必须同步刷新。 |

## 典型业务流

1. `process.task.created` 到达后，子模块按任务信息创建或更新待办投影。
2. 用户在待办页按分类、紧急程度、时间和流程名称筛选并打开详情。
3. 任务完成、终止、转办或超时后，投影异步更新，列表状态和数量同步变化。
4. “我发起”“草稿”“归档”视图通过聚合查询组合实例、草稿和归档权威数据。
5. 抄送项支持阅知标记，但不转化为审批动作。

## 一期实现约束

- 一期重点覆盖待办、已办、我发起、抄送我、草稿、归档和数量统计。
- 待办与消息严格分离；待办事件只表达事务状态，通知渠道由 `06` 决定。
- `todo_item` 只投影任务视图；草稿与归档通过聚合查询获得，不重复建表。
- 不提供任务审批写接口，用户点击待办后跳转到实例/审批页面处理。
- 移动端提供基础待办和已办查询，复杂筛选和归档高级检索保留在 PC。
