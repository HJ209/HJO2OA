# process-definition 前端

## 页面清单

| 页面/区域 | 适用端 | 页面职责 |
|-----------|--------|----------|
| 流程列表页 | 管理端 | 展示定义清单、状态、版本数、最近发布时间，承接新建与进入设计器。 |
| 流程设计器页 | 管理端 | 维护节点、路由、动作编码、绑定表单和发布前校验。 |
| 节点属性侧栏 | 管理端 | 编辑节点类型、参与者规则、节点动作、超时规则和多实例参数。 |
| 版本链抽屉/对比页 | 管理端 | 查看同编码历史版本、差异和废弃状态。 |
| 发布确认弹窗 | 管理端 | 展示校验结果、影响范围和绑定表单版本后执行发布。 |

## 页面职责

- 列表页只做管理入口，不承担流程发起。
- 设计器页负责建模输入和本地校验提示，不直接执行任何运行时动作。
- 节点属性侧栏负责录入参与者规则与动作白名单，不负责动态权限判定。
- 版本链页只支持查看、派生新版本、废弃旧版，不支持“覆盖式回滚”。

## 关键交互

1. 新建流程时先选择分类并绑定已发布表单版本，再进入设计器。
2. 设计器保存草稿时只落草稿版，不改变外部可用版本。
3. 用户点击发布时，前端必须先触发服务端校验接口，再允许确认发布。
4. 切换历史版本时页面进入只读态，若需修改只能“基于该版本新建草稿”。
5. 参与者规则配置支持组织/岗位/角色/表单字段/表达式五类输入控件。

## 字段状态、节点状态、权限状态约束

- 定义状态：
  - `DRAFT` 可编辑、可校验、可发布。
  - `PUBLISHED` 只读，可派生新版本或废弃。
  - `DEPRECATED` 只读，不允许再作为默认发起版本。
- 节点状态：
  - `START` 仅允许一个。
  - `END` 至少一个。
  - `USER_TASK` 必须配置参与者规则和至少一个动作编码。
- 字段状态：
  - 设计器中的字段权限矩阵仅展示来自 `form-metadata` 的配置结果，不在本模块直接维护字段定义。
- 权限状态：
  - 设计管理员可编辑全部草稿。
  - 发布权限与编辑权限分离时，未具备发布权限的用户只可保存草稿。
  - 观察者只可查看版本和配置快照。

## 端差异

- 一期仅提供管理端页面，不提供移动端流程设计器。
- 若后续开放移动端查看，也仅允许只读查看定义详情，不允许编辑或发布。
