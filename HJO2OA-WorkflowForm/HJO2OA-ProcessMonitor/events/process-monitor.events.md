# process-monitor 领域事件

## 发布事件

一期不发布领域事件。

- 父模块已明确 `process-monitor` 一期暂缓，因此当前不引入 `process.monitor.*` 事件族。
- 即使后续启用，监控结果优先作为管理读模型，不直接驱动主链路状态变化。

## 消费事件

一期也不启用事件消费。

- 后续若建设实时分析缓存，优先消费 `process.instance.*`、`process.task.*` 和 `todo.item.*` 事件构建分析视图。
- 在一期之前，不应创建分析型订阅器占用事件总线资源。

## 投影、幂等与补偿

- 一期无独立投影表，无事件幂等和补偿链路。
- 后续若启用实时监控，应将监控缓存视为可重建投影，允许通过重放实例/任务事件恢复。
- 监控数据永远不得成为流程运行态的权威来源。
