# process-monitor 概览

## 模块定位

`process-monitor` 是 `02-流程与表单` 预留的管理端监控与分析子模块，定位为读模型分析层，面向流程管理员提供耗时、停滞、拥塞和超时观察能力。根据父模块 `module-design.md`，该子模块一期暂缓，不纳入当前交付实现。

## 核心职责

- 汇总实例、任务和待办运行态数据，形成管理分析视图。
- 识别长耗时流程、停滞节点、审批拥塞和超时任务。
- 为后续催办、运维排查和流程优化提供管理入口。

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `process-instance` / `todo-center` | 分析数据来源于实例、任务和待办权威数据或其事件流。 |
| 上游 | `06-消息移动与生态` | 消息模块负责触达渠道；监控模块只给出分析结果，不直接管理消息发送。 |
| 下游 | 管理后台 | 仅服务管理端监控视图，不作为业务用户处理入口。 |
| 边界外 | 审批动作、草稿、归档 | 不负责执行动作、不负责保存草稿、不管理归档内容。 |

## 核心能力表

| 能力 | 目标输出 | 一期状态 | 说明 |
|------|----------|----------|------|
| 流程耗时分析 | 平均耗时、分定义耗时趋势、节点耗时排行 | 暂缓 | 当前仅保留契约。 |
| 节点停滞分析 | 停滞节点、停滞时长、停滞实例列表 | 暂缓 | 当前仅保留契约。 |
| 审批拥塞分析 | 审批人负载、待办堆积、退回率 | 暂缓 | 当前仅保留契约。 |
| 超时观察与催办线索 | 超时任务清单、催办建议 | 暂缓 | 一期不开放自动催办。 |

## 典型业务流

1. 管理员选择流程分类、定义、时间范围进入监控页。
2. 系统汇总实例、任务和待办运行数据，返回耗时、停滞、拥塞等读模型结果。
3. 管理员从图表跳转到具体实例或任务详情，定位瓶颈节点与处理人。
4. 如需催办，由后续版本或其他模块承接，不在一期监控模块内直接执行。

## 一期实现约束

- 一期暂缓，不新增独立持久化、领域事件、管理页面和生产接口。
- 当前文档仅冻结后续实现输入边界，避免后续监控建设越界到流程主链。
- 若提前开放试验性接口，必须以管理端只读查询为限，不得直接驱动流程状态变化。
