# form-metadata 前端

## 页面清单

| 页面/区域 | 适用端 | 页面职责 |
|-----------|--------|----------|
| 表单列表页 | 管理端 | 展示表单清单、状态、引用关系和版本数。 |
| 表单设计器页 | 管理端 | 编辑字段、布局、校验、联动和组件属性。 |
| 节点字段权限矩阵 | 管理端 | 配置并预览节点对字段的可见、可编辑、必填状态。 |
| 版本链抽屉/对比页 | 管理端 | 查看版本差异、派生新版本、废弃旧版。 |
| 发布确认弹窗 | 管理端 | 展示校验问题和影响范围后执行发布。 |

## 页面职责

- 列表页只负责导航与筛选，不承担渲染协议预览。
- 设计器页负责输入表单结构，不负责业务数据填写。
- 字段权限矩阵页只配置字段状态，不负责审批动作按钮权限。
- 版本链页仅支持查看与派生，不允许直接改写历史版本。

## 关键交互

1. 设计器新增字段后即时校验 `fieldCode` 唯一性和字段类型约束。
2. 切换布局模式时保留字段定义，但必须重新校验空布局、无归属字段等问题。
3. 权限矩阵选择流程节点后，页面展示对应字段状态预览。
4. 点击发布前必须先执行服务端校验，并阻断未通过校验的版本发布。

## 字段状态、节点状态、权限状态约束

- 字段状态：
  - 基础状态定义在字段级，节点级只能覆盖 `visible/editable/required`。
  - 子表字段权限跟随子表容器和明细字段同时校验。
- 节点状态：
  - 仅显示来自流程定义的节点引用，不在本页面创建流程节点。
- 权限状态：
  - 表单设计管理员可编辑草稿。
  - 仅具备发布权限的角色可执行发布和废弃。
  - 查看者只读浏览，不可修改字段和布局。

## 端差异

- 一期仅提供管理端设计器，不提供移动端设计入口。
- 后续如提供移动端，仅允许只读预览渲染效果，不允许编辑或发布。
