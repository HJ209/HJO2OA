# form-metadata 概览

## 模块定位

`form-metadata` 是 `02-流程与表单` 的设计时表单协议子模块，负责维护 `FormMetadata` 聚合及版本链。它产出字段、布局、校验、联动和字段权限映射协议，供 `process-definition` 绑定，供 `form-renderer` 在运行时渲染。

## 核心职责

- 维护表单字段定义、布局定义、字段校验和联动规则。
- 维护节点级字段权限映射 `{nodeId -> fieldCode -> state}`。
- 输出标准化渲染协议，保证不同业务流程可以复用统一表单渲染器。
- 管理表单元数据版本发布、废弃和历史版本只读查询。

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | 字典、附件、组织选择器协议 | 只引用基础设施和组织权限模块的选择器/字典能力，不复制底层数据。 |
| 协同 | `process-definition` | 流程定义只能绑定已发布表单版本；节点字段权限映射由本模块维护、流程设计器只做预览。 |
| 下游 | `form-renderer` | 渲染器只消费本模块输出的渲染协议，不直接解析设计器内部草稿结构。 |
| 下游 | `process-instance` | 流程发起时冻结表单元数据快照；运行中实例不受后续表单版本变更影响。 |
| 边界外 | 业务表单数据 | 本模块不保存业务单据正文，不负责字段值持久化。 |

## 核心能力表

| 能力 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| 字段定义 | 字段编码、类型、默认值、校验规则 | `fields` JSON | 字段编码在单表单内唯一。 |
| 布局定义 | 分组、栅格、Tab、子表布局 | `layout` JSON | 仅维护结构，不绑定业务数据。 |
| 联动与校验 | 显隐、必填、默认值、长度/范围规则 | `validations`、`linkageRules` | 一期不支持脚本联动。 |
| 字段权限映射 | 节点 ID、字段状态 | `fieldPermissionMap` | 只支持 `visible/editable/required` 三类状态。 |
| 版本发布/废弃 | 草稿定义、校验结果 | 发布版表单协议 | 发布后不可原地修改。 |

## 典型业务流

1. 设计人员新建表单元数据草稿，配置字段、布局、校验与联动规则。
2. 在权限映射面板中配置各流程节点对字段的可见、可编辑、必填状态。
3. 调用校验接口检查字段编码唯一性、布局合法性、子表嵌套和引用字段是否有效。
4. 发布后，流程定义绑定该版本；流程实例在启动时冻结该版本快照。
5. 后续调整通过派生新版本完成，历史实例继续使用旧版快照。

## 一期实现约束

- 一期只支持基础字段类型、子表、附件、人员/组织选择和简单联动。
- 发布后不可编辑，只能派生新版本；不提供覆盖式回滚。
- 联动规则仅支持显隐、必填、默认值、只读切换，不支持自定义脚本。
- 节点字段权限仅支持 `visible/editable/required`，不引入独立权限体系。
- 表单设计器仅提供管理端页面，不提供移动端设计能力。
