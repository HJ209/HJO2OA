# action-engine 概览

## 模块定位

`action-engine` 是流程运行时的动作判定与留痕子模块，负责统一解析“当前任务能做什么、是否允许做、做完后如何路由”。它拥有 `TaskAction` 不可变审计记录，但不拥有流程实例和任务主状态。

## 核心职责

- 计算任务可用动作集合和按钮显示状态。
- 校验动作前置条件：节点配置、表单状态、当前身份、目标节点/目标参与人是否完整。
- 执行动作命令，产出不可修改的 `TaskAction` 记录。
- 解析下一节点参与者、条件路由和目标节点约束，向 `process-instance` 返回状态推进结果。

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `process-definition` | 读取节点 `actionCodes`、路由和参与者规则，不维护设计时配置。 |
| 上游 | `form-renderer` | 接收前端提交的 `formDataPatch` 和本地校验结果，不直接渲染表单。 |
| 上游 | `01-组织与权限` | 复用身份上下文和组织关系解析动态参与者。 |
| 协同 | `process-instance` | 动作执行成功后由实例模块写入任务/实例状态并发布 `process.task.*` 事件。 |
| 边界外 | 草稿/归档 | 不拥有发起草稿、归档查询和实例挂起/恢复/终止接口。 |

## 核心能力表

| 能力 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| 动作可用性判定 | 任务上下文、节点配置、当前身份 | 可用动作列表 | 不可用动作必须返回明确禁用原因。 |
| 动作执行 | `actionCode`、意见、目标节点/参与人、表单变更 | `TaskAction`、下一步推进指令 | 每次执行都必须落审计记录。 |
| 路由与目标校验 | 当前节点、动作配置、条件表达式 | 目标节点集合 | 仅支持简单表达式。 |
| 动态参与者解析 | 参与者规则、身份上下文、表单字段值 | 候选人集合 | 候选人为空时不得推进流程。 |

## 典型业务流

1. 审批页面加载任务时，先查询当前任务可用动作和每个动作的附加要求。
2. 用户选择动作并补充意见、目标节点或目标参与人后提交执行命令。
3. 动作引擎校验节点配置、当前身份、表单补丁、目标节点/参与人和路由规则。
4. 校验通过后落 `TaskAction` 记录，并将推进结果返回给 `process-instance` 写入任务与实例状态。
5. 失败时返回明确错误码，不单独发布动作事件，由上游界面决定是否重试。

## 一期实现约束

- 一期重点覆盖 `approve`、`reject`、`return`、`transfer`、`add_sign` 等常用动作，实际可用动作以节点 `actionCodes` 为准。
- 条件判断只支持简单表达式和表单字段值比较，不支持脚本引擎。
- 实例挂起、恢复、终止属于 `process-instance` 管控接口，不通过本模块普通动作入口执行。
- 动作记录不可修改、不可删除，作为统一审计来源。
- 一期不开放强制跳节点、改写历史动作等高风险人工干预能力。
