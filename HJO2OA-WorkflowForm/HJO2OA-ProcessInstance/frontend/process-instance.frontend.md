# process-instance 前端

## 页面清单

| 页面/区域 | 适用端 | 页面职责 |
|-----------|--------|----------|
| 流程发起入口页 | PC/移动端入口 | 选择流程定义、展示分类和最近使用项。 |
| 发起与草稿编辑页 | PC 为主 | 承载表单渲染、草稿保存、正式提交。 |
| 审批处理页壳层 | PC/移动端 | 承载实例标题、轨迹、表单容器和动作区插槽。 |
| 实例详情页 | PC/移动端只读 | 展示实例状态、当前节点、审批轨迹和业务单据引用。 |
| 草稿列表页 | PC | 查看、继续编辑或删除发起草稿。 |
| 归档列表/详情页 | PC | 查询已结束实例并查看归档摘要。 |

## 页面职责

- 发起入口页只负责选择流程和跳转，不直接保存业务数据。
- 发起与草稿编辑页负责和 `form-renderer` 协同采集输入，草稿/提交命令由 `process-instance` 发起。
- 审批处理页壳层负责实例上下文和轨迹区，动作按钮与意见输入由 `action-engine` 组件承载。
- 归档页只读展示，不允许在归档态再次发起动作。

## 关键交互

1. 用户从入口页选择流程后，页面先拉取定义快照和表单渲染协议，再进入填写态。
2. 保存草稿时仅调用草稿接口，不触发实例创建。
3. 提交发起后页面跳转到实例详情或待办详情，并刷新“我发起”与待办数量。
4. 审批处理中，轨迹区和当前节点状态需在动作成功后即时刷新。
5. 归档列表支持标题、流程分类、发起人和结束时间筛选。

## 字段状态、节点状态、权限状态约束

- 字段状态由 `form-renderer` 根据模式和节点权限决定，实例页本身不重复计算字段权限。
- 节点状态至少区分 `CURRENT`、`COMPLETED`、`PENDING`、`TERMINATED`，轨迹时间线按实例实际顺序展示。
- 页面权限：
  - 发起人可编辑自己的草稿并查看自己发起的实例。
  - 当前处理人或候选人可进入审批壳层。
  - 管理员可查看全部实例并执行挂起/恢复/终止。
- 挂起实例进入只读壳层，禁止再次提交动作。

## 端差异

- 移动端仅复用发起入口、基础详情和审批壳层，不提供草稿批量管理、归档高级筛选和实例控制入口。
- 管理端额外显示实例控制按钮、更多筛选条件和完整轨迹调试信息。
