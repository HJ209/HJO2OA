# aggregation-api 前端合同

## 页面清单

| 页面/容器 | 页面职责 | 接入卡片 |
|-----------|----------|----------|
| 门户首页卡片区 | 承载首页聚合卡片的批量装配与局部刷新。 | 待办、消息、公告、快捷入口、基础统计 |
| 办公中心卡片区 | 承载办公中心摘要卡片和列表型卡片。 | 待办、消息、流程统计 |
| 移动工作台卡片区 | 承载轻量卡片集合和高频入口。 | 待办、消息、快捷入口 |
| 列表型卡片查看更多面板 | 承载列表卡片的分页加载和筛选。 | 待办列表、消息列表、公告列表 |

## 页面职责与关键交互

| 交互 | 说明 | 依赖模块 |
|------|------|----------|
| 批量加载卡片 | 根据页面装配上下文一次性获取所需卡片快照。 | `aggregation-api` |
| 单卡刷新 | 针对某张卡片执行定向刷新或重试。 | `aggregation-api` |
| 列表分页与筛选 | 对待办、消息、公告等列表型卡片执行分页和筛选。 | `aggregation-api` |
| 降级展示 | 快照失败时展示空态、旧数据或错误提示。 | `aggregation-api` |
| 卡片跳转 | 点击卡片或列表项后进入对应业务模块页面。 | `portal-home` + 原业务模块 |

## 模板、卡片、个性化、移动端约束

- 模板约束：哪些卡片出现、顺序如何、位于哪个区域，由模板与个性化共同决定，卡片组件本身不得自行改位。
- 卡片约束：前端组件只能消费 `aggregation-api` 返回的统一协议，不直接拼装多源业务接口。
- 个性化约束：个性化只影响卡片顺序、显隐和快捷入口，不改变卡片返回的数据结构。
- 移动端约束：移动端复用同一聚合协议，但只展示轻量字段和高频交互，不复制第二套接口契约。

## 身份切换影响

- 身份切换后必须清空当前页面上的所有快照状态和分页游标，并用新的 `snapshotKey` 重新拉取数据。
- 切换过程中不得继续显示旧身份的未读数、待办数或快捷入口。
- 列表型卡片的筛选条件可保留 UI 形态，但实际数据必须按新身份重新查询。
