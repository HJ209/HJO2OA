# 4.4 聚合数据接口

## 模块定位

`aggregation-api` 是门户首页、办公中心和移动工作台的读模型层，负责按身份上下文聚合卡片数据并输出用户侧快照。它服务入口体验，不承担通用开放 API、设计器配置或模板发布职责。

## 核心职责

- 管理 `AggregationCardSnapshot`，缓存不同身份、场景和卡片类型下的聚合结果。
- 聚合待办、消息、公告、快捷入口、基础统计等门户卡片读模型。
- 支持批量获取、单卡刷新、失效重算和卡片级降级。
- 响应业务事件、模板事件和个性化事件，刷新受影响快照。

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `portal-model` | 只消费模板解析结果和卡片布置，不参与模板真相源维护。 |
| 上游 | `widget-config` | 按卡片协议聚合数据，不复制卡片定义。 |
| 上游 | `personalization` | 仅消费个性化顺序、显隐和快捷入口结果。 |
| 上游 | `02/03/05/06` | 消费待办、消息、公告、统计等上游读模型，不拥有其主数据。 |
| 下游 | `portal-home` | 以卡片读模型形式供首页、办公中心和移动端渲染。 |
| 外部边界 | `07-数据服务与集成` | 不提供第三方通用数据查询接口，不承担跨系统开放能力。 |

## 核心能力表

| 能力 | 说明 | 一期实现口径 |
|------|------|--------------|
| 批量聚合 | 按场景批量返回页面所需卡片快照。 | 支持首页、办公中心、移动工作台的批量卡片装配。 |
| 单卡刷新 | 对指定卡片执行定向刷新或回源。 | 用于操作后定向刷新，避免整页全量失效。 |
| 快照缓存 | 维护 `snapshotKey`、过期时间、源水位和缓存状态。 | 一期默认 Redis 级缓存，不强制 L1 本地缓存。 |
| 卡片降级 | 对 `STALE`、`FAILED` 状态卡片执行容错返回。 | 快照失败不能阻塞整页渲染。 |
| 事件驱动失效 | 根据业务和配置事件刷新受影响快照。 | 以待办、消息、身份切换、模板/个性化变更为一期主链路。 |

## 典型业务流

1. `portal-home` 按当前 `sceneType` 请求聚合读模型。
2. `aggregation-api` 结合生效模板、卡片布置和个性化结果计算本次所需卡片集合。
3. 先按 `tenantId + personId + assignmentId/positionId + sceneType + cardType` 查询快照缓存。
4. 命中 `READY` 直接返回；命中 `STALE` 返回旧值并异步刷新；未命中或 `FAILED` 时回源重算。
5. 当业务事件、模板发布或个性化变更发生时，定向失效受影响快照并在下一次读取时重建。

## 一期实现约束

- 只服务门户首页、办公中心和移动工作台场景，不承担通用开放查询职责。
- 一期重点支持待办、消息、快捷入口、基础公告和基础流程统计卡片。
- 日程、会议和复杂业务看板能力可保留扩展位，但不作为一期闭环前提。
- 一期不依赖 WebSocket 实时推送，采用缓存失效 + 前端轮询/定向刷新。
