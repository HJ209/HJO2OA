# aggregation-api API 合同

## API 定位

`aggregation-api` API 负责门户首页、办公中心和移动工作台所需的用户侧聚合读模型输出。接口按场景、卡片类型和身份上下文返回快照，不承担模板设计、模板发布或通用数据开放。

## 接口分组

| 分组 | 资源/对象 | 合同要求 |
|------|-----------|----------|
| 场景批量聚合 | 页面卡片集合 | 按 `sceneType`、`clientType`、`pageCode` 批量返回当前页面所需卡片快照。 |
| 办公中心聚合 | 办公中心工作区数据 | 返回办公中心左侧分类导航、右侧列表摘要和相关卡片数据。 |
| 单卡查询与刷新 | `AggregationCardSnapshot` | 支持指定 `cardType` 的读取、失效和定向刷新。 |
| 列表型卡片分页查询 | 待办/消息/公告等列表卡片 | 提供列表项分页、筛选和排序能力，避免首页一次性加载全部明细。 |

## 边界口径

- 本模块是读模型层，不保存模板草稿、发布状态或设计器配置。
- 模板发布归 `portal-model`，个性化归 `personalization`，卡片定义归 `widget-config`。
- 上游待办、消息、公告、统计等主数据仍归对应业务域，本模块只输出聚合结果。
- `aggregation-api` 仅服务门户与办公中心场景，不承担平台级开放查询接口职责。

## 分页与筛选约定

| 查询对象 | 分页参数 | 主要筛选条件 |
|----------|----------|--------------|
| 场景批量聚合 | 不分页 | `sceneType`、`clientType`、`pageCode`、`cardTypes` |
| 待办列表卡片 | `pageNo`、`pageSize` | `todoCategory`、`urgentOnly`、`keyword`、`sortBy` |
| 消息列表卡片 | `pageNo`、`pageSize` | `messageCategory`、`readStatus`、`keyword`、`sortBy` |
| 公告/内容卡片 | `pageNo`、`pageSize` | `contentCategory`、`publishRange`、`keyword` |

## 缓存、新鲜度、权限与审计约束

| 主题 | 合同要求 |
|------|----------|
| 缓存 | 快照键至少包含 `tenantId + personId + assignmentId/positionId + sceneType + cardType`。 |
| 新鲜度 | `READY` 直接返回，`STALE` 允许返回旧值并异步刷新，`FAILED` 必须降级而不能阻塞整页。 |
| 权限 | 所有聚合结果必须按当前身份上下文裁剪，禁止返回当前身份不可见的数据。 |
| 审计 | 默认只读查询不做高频审计；手动刷新、失败重试和异常降级需要记录运维/诊断日志。 |

## 错误场景

| 场景 | 处理要求 |
|------|----------|
| 缺少身份上下文字段 | 拒绝查询，返回身份上下文缺失错误。 |
| 请求的卡片类型不在当前模板中 | 返回卡片不可用或忽略该卡片，不允许越权查询。 |
| 上游业务域超时或失败 | 返回降级卡片或旧快照，并触发异步重试。 |
| 快照刷新失败 | 返回 `FAILED`/降级语义，同时记录失败原因。 |
| 试图将本接口作为通用开放查询使用 | 拒绝受理超出门户场景的调用。 |
