# portal-model 前端合同

## 页面清单

| 页面/入口 | 页面职责 | 使用人群 |
|-----------|----------|----------|
| 模板列表页 | 浏览、筛选、创建门户模板，管理模板状态。 | 门户管理员、运营管理员 |
| 页面结构管理页 | 维护页面树、默认页、页面类型和路由。 | 门户管理员、设计管理员 |
| 区域与卡片布置页 | 维护区域树、区域属性和卡片布置清单。 | 门户管理员、设计管理员 |
| 版本管理页 | 查看草稿/已发布/已废弃版本链，执行复制和切换。 | 门户管理员 |
| 发布配置页 | 配置 `clientType`、时间窗、范围、优先级和下线操作。 | 门户管理员、运营管理员 |

## 页面职责与关键交互

| 交互 | 说明 | 依赖模块 |
|------|------|----------|
| 创建模板 | 录入模板编码、场景、分类、默认页等基础信息。 | `portal-model` |
| 管理页面结构 | 新增/删除页面，设置默认页和页面排序。 | `portal-model` |
| 配置区域与卡片布置 | 在区域内选择卡片并维护默认显隐、排序和覆写参数。 | `portal-model` + `widget-config` |
| 版本复制与发布 | 基于草稿复制版本，发布前执行完整性校验。 | `portal-model` |
| 范围选择与冲突校验 | 配置人/岗位/角色/部门/组织范围，并检查发布时间窗冲突。 | `portal-model` |

## 模板、卡片、个性化、移动端约束

- 模板约束：同一版本只能有一个默认页；已发布版本只读；布局模式必须与页面场景匹配。
- 卡片约束：前端只能选择已启用的卡片定义，卡片协议和展示策略不可在此页直接修改。
- 个性化约束：`required=TRUE` 的区域和卡片必须在模板层固定，不得标记为可被用户删除。
- 移动端约束：移动工作台必须复用同一模板体系，仅通过 `pageType` 和 `layoutMode` 区分移动端页面。

## 身份切换影响

- 管理后台当前身份切换只影响可操作权限和可查询的发布范围，不改变模板真相源。
- 涉及生效模板解析或预览的页面，必须按选定目标身份上下文重新加载范围命中结果。
- 身份切换后应清空当前发布预览缓存，避免把旧 `assignmentId` 下的可见范围误用于新身份。
