# personalization API 合同

## API 定位

`personalization` API 负责当前生效模板之上的个人/任职级个性化读写接口，包括排序、显隐、快捷入口、主题偏好和重置回退。它不承担模板发布、角色范围裁决或聚合读模型输出。

## 接口分组

| 分组 | 资源/对象 | 合同要求 |
|------|-----------|----------|
| 基础模板上下文查询 | 生效发布模板上下文 | 返回当前 `sceneType + assignmentId` 命中的基础模板、允许覆写范围和必要卡片约束。 |
| 当前个性化配置查询 | `PersonalizationProfile` | 返回当前身份下的个性化视图，支持任职级优先、用户全局回退。 |
| 个性化保存 | 顺序/显隐/布局/主题 | 保存模板授权范围内的有限覆写，不允许越权覆盖模板真相源。 |
| 快捷入口维护 | `QuickAccessEntry` | 维护常用流程、应用、链接和内容入口的排序与置顶。 |
| 重置与回退 | `PersonalizationProfile.status` | 提供重置为基础模板默认值的能力。 |

## 边界口径

- 角色化、组织化和全局模板的发布真相源在 `portal-model`，本模块只读取其解析结果。
- 读模型归 `aggregation-api`，本模块不直接返回待办、消息、统计卡片数据。
- 个性化是有限覆写，不是复制模板或生成新模板分支。
- 个性化配置必须按 `personId + assignmentId + sceneType` 隔离，不得跨身份复用。

## 分页与筛选约定

| 查询对象 | 分页参数 | 主要筛选条件 |
|----------|----------|--------------|
| 当前个性化配置查询 | 不分页 | `sceneType`、`assignmentId` |
| 快捷入口列表 | `pageNo`、`pageSize`、`sortBy`、`sortOrder` | `entryType`、`pinned`、`targetCode` |

## 缓存、新鲜度、权限与审计约束

| 主题 | 合同要求 |
|------|----------|
| 缓存 | 保存或重置后必须失效对应个性化视图缓存和受影响聚合快照。 |
| 新鲜度 | 当前用户保存成功后，下一次读取必须读到最新个性化结果，不能继续返回旧顺序或旧显隐状态。 |
| 权限 | 默认仅当前用户可维护自身个性化；管理端代操作需具备明确管理员权限。 |
| 审计 | 个性化保存、重置、快捷入口增删改必须记录审计日志，保留操作身份和场景。 |

## 错误场景

| 场景 | 处理要求 |
|------|----------|
| 基础发布模板不存在或已下线 | 拒绝保存，要求重新解析当前基础模板。 |
| 试图隐藏必保留卡片或区域 | 拒绝保存。 |
| 使用错误的 `assignmentId` 保存当前身份配置 | 拒绝保存，避免跨身份串用。 |
| 快捷入口目标非法或重复 | 拒绝保存，要求修正目标编码/链接。 |
| 超出模板授权边界的布局覆写 | 拒绝保存，不在渲染时静默丢弃。 |
