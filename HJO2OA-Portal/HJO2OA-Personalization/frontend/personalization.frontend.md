# personalization 前端合同

## 页面清单

| 页面/入口 | 页面职责 | 使用人群 |
|-----------|----------|----------|
| 当前场景个性化设置页 | 查看当前基础模板上下文，维护卡片顺序、显隐和布局微调。 | 终端用户 |
| 快捷入口编辑器 | 维护常用流程、应用、链接和内容入口。 | 终端用户 |
| 主题与显示偏好面板 | 选择主题和基础显示偏好。 | 终端用户 |
| 重置确认弹窗 | 将当前场景配置回退到基础模板默认值。 | 终端用户 |

## 页面职责与关键交互

| 交互 | 说明 | 依赖模块 |
|------|------|----------|
| 查看基础模板上下文 | 展示当前身份命中的基础发布模板和可覆写范围。 | `portal-model` + `personalization` |
| 拖拽调整卡片顺序 | 在模板允许范围内重排卡片顺序。 | `personalization` |
| 隐藏/恢复卡片 | 对允许隐藏的卡片做显隐切换。 | `personalization` + `widget-config` |
| 维护快捷入口 | 新增、删除、排序和置顶快捷入口。 | `personalization` |
| 重置个性化 | 清空当前场景覆写并回退默认模板视图。 | `personalization` |

## 模板、卡片、个性化、移动端约束

- 模板约束：角色化、组织化模板命中结果来自 `portal-model`，本页面不提供模板发布或范围配置入口。
- 卡片约束：必保留卡片、必保留区域和 `allowHide=FALSE` 卡片不能在这里被隐藏。
- 个性化约束：保存的是有限覆写，不允许复制模板、修改卡片协议或创建新模板分支。
- 移动端约束：移动工作台个性化只支持高频入口、顺序和基础显隐，不单独维护一套模板。

## 身份切换影响

- `assignmentId` 切换后必须重新加载对应场景的个性化配置；任职级配置优先于用户全局配置。
- 未保存的本地修改在身份切换时必须提示用户处理，避免写入到错误身份。
- 身份切换后快捷入口和主题偏好也必须按新上下文重新解析，不能复用旧身份缓存。
