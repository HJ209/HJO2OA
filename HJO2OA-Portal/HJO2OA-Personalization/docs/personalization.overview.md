# 4.3 个性化与角色化配置

## 模块定位

`personalization` 负责在已生效门户发布模板之上叠加用户级、任职级的有限覆写，输出最终个性化视图。角色化和组织化模板的真相源仍在 `portal-model` 的发布范围控制中，本模块不承担模板发布真相源职责。

## 核心职责

- 管理 `PersonalizationProfile` 与 `QuickAccessEntry`。
- 在当前 `sceneType + assignmentId` 上解析用户是否存在任职级个性化，并回退到用户全局个性化。
- 保存卡片顺序、隐藏规则、布局微调、快捷入口和主题偏好。
- 在保存或重置后通知 `portal-home` 与 `aggregation-api` 重新解析视图和刷新快照。

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `portal-model` | 仅基于已生效发布模板做有限覆写，不管理模板发布、优先级和范围裁决。 |
| 上游 | `widget-config` | 遵守卡片 `allowHide`、`required` 等约束，不改写卡片协议。 |
| 上游 | `01-组织与权限` | 依赖 `assignmentId`、岗位、角色等身份上下文做个性化隔离。 |
| 下游 | `portal-home` | 输出渲染前的个性化视图，不承担页面渲染。 |
| 下游 | `aggregation-api` | 变更后驱动聚合快照失效或重算，但不直接维护快照。 |
| 外部边界 | 应用目录/模板中心 | 快捷入口和主题偏好只是用户偏好数据，不构成应用目录或模板分叉真相源。 |

## 核心能力表

| 能力 | 说明 | 一期实现口径 |
|------|------|--------------|
| 个性化视图解析 | 优先读取任职级个性化，未命中时回退到用户全局个性化。 | 关键隔离键为 `personId + assignmentId + sceneType`。 |
| 卡片顺序与显隐 | 保存排序、折叠和隐藏偏好。 | `required=TRUE` 或 `allowHide=FALSE` 的卡片不可隐藏。 |
| 快捷入口维护 | 维护常用流程、应用、链接和内容入口。 | 作为个性化配置的子集，不形成独立目录模型。 |
| 主题与布局微调 | 保存主题编码和允许范围内的布局微调。 | 一期仅支持基础主题和轻量布局覆盖。 |
| 重置与回退 | 用户清空个性化后回退到基础发布模板。 | `status=RESET` 时渲染回退默认模板，不删除历史审计。 |

## 典型业务流

1. 用户登录或切换身份后，系统先由 `portal-model` 解析当前生效发布模板。
2. `personalization` 按 `assignmentId` 查找任职级个性化，未命中则回退到用户全局个性化。
3. 在模板允许范围内叠加卡片顺序、隐藏、快捷入口和主题偏好，形成最终视图。
4. 用户保存个性化后发布保存事件，驱动聚合快照失效和页面定向刷新。
5. 用户执行重置时回退到基础模板默认值，不创建新的模板副本。

## 一期实现约束

- 角色化、组织化差异由 `portal-model` 的发布范围实现，本模块只做解析后的个人/任职级覆写。
- 一期只支持排序、隐藏、快捷入口和主题偏好，不做复杂模板分叉或可视化模板复制。
- 个性化配置不得跨身份透传，身份切换后必须按新 `assignmentId` 重新解析。
- 超出模板授权边界的配置在保存时直接拒绝，不在渲染时隐式忽略。
