# widget-config 前端合同

## 页面清单

| 页面/入口 | 页面职责 | 使用人群 |
|-----------|----------|----------|
| 卡片定义列表页 | 浏览、筛选、创建卡片定义，维护状态。 | 门户管理员、组件管理员 |
| 数据源配置面板 | 配置查询编码、请求/响应协议、字段映射和回退策略。 | 组件管理员 |
| 展示策略配置面板 | 配置样式、最大条数、隐藏/折叠能力和展示限制。 | 组件管理员 |
| 可见规则配置页 | 配置权限模式和自定义作用范围。 | 门户管理员、组件管理员 |
| 引用影响侧栏 | 查看当前卡片被哪些模板和页面引用。 | 门户管理员、组件管理员 |

## 页面职责与关键交互

| 交互 | 说明 | 依赖模块 |
|------|------|----------|
| 创建卡片定义 | 选择卡片类别、渲染类型、来源模块和刷新模式。 | `widget-config` |
| 维护数据协议 | 编辑请求/响应 schema、字段映射和 fallback 策略。 | `widget-config` |
| 调整展示策略 | 控制默认样式、最大条目数、是否允许隐藏/折叠。 | `widget-config` |
| 查看引用影响 | 评估某个卡片变更会影响哪些模板和设计器草稿。 | `widget-config` + `portal-model` |
| 停用卡片 | 在停用前展示影响提示，避免线上模板直接失效。 | `widget-config` |

## 模板、卡片、个性化、移动端约束

- 模板约束：卡片定义是多个模板共享的公共资源，模板差异必须通过布置级覆写完成。
- 卡片约束：聚合型卡片只能声明协议，不能在前端配置绕过聚合层的直连接口。
- 个性化约束：是否允许用户隐藏/折叠由卡片策略控制，个性化侧不能突破这里的限制。
- 移动端约束：移动端优先复用同一卡片定义，通过展示策略和布置级覆写控制差异，不维护第二套卡片体系。

## 身份切换影响

- 管理页本身不因终端用户身份切换而改变卡片定义，但预览可见性时必须使用目标身份上下文。
- 若当前管理员身份变化导致权限收缩，应立即重新校验可编辑卡片范围。
- 切换身份后不得继续沿用旧身份下缓存的可见规则预览结果。
