# widget-config API 合同

## API 定位

`widget-config` API 负责门户卡片定义、数据源协议、展示策略和可见规则的管理接口。它输出的是设计时卡片契约，不直接输出用户侧聚合数据。

## 接口分组

| 分组 | 资源/对象 | 合同要求 |
|------|-----------|----------|
| 卡片定义管理 | `WidgetDefinition` | 提供卡片创建、编辑、启停、按类别查询和状态管理。 |
| 数据源协议管理 | `WidgetDataSourceBinding` | 提供查询编码、请求协议、响应协议、字段映射、回退策略维护。 |
| 展示策略管理 | `WidgetDisplayPolicy` | 提供样式、最大条数、折叠/隐藏能力和展示约束配置。 |
| 可见规则管理 | 权限模式与作用范围 | 提供 `INHERIT_PORTAL`、`CUSTOM_SCOPE` 等可见策略配置。 |
| 引用状态查询 | 卡片引用概况 | 提供给 `portal-model`/`portal-designer` 的只读查询，用于保存与发布前校验。 |

## 边界口径

- 读模型归 `aggregation-api`，本模块不返回待办、消息、公告等聚合结果。
- 模板发布归 `portal-model`，本模块不保存模板范围、优先级和生效时间窗。
- 设计器只是本模块的消费者，卡片定义真相源始终在 `widget-config`。
- `dataSourceType=AGGREGATION_QUERY` 的卡片必须经由 `aggregation-api` 输出，不允许前端直连业务数据源。

## 分页与筛选约定

| 查询对象 | 分页参数 | 主要筛选条件 |
|----------|----------|--------------|
| 卡片定义列表 | `pageNo`、`pageSize`、`sortBy`、`sortOrder` | `category`、`renderType`、`dataSourceType`、`status`、`sourceModule` |
| 数据源绑定列表 | `pageNo`、`pageSize` | `widgetDefinitionId`、`queryCode` |
| 展示策略列表 | `pageNo`、`pageSize` | `widgetDefinitionId`、`allowHide`、`allowCollapse` |

## 缓存、新鲜度、权限与审计约束

| 主题 | 合同要求 |
|------|----------|
| 缓存 | 卡片定义可被下游短时缓存，但任何配置写入后必须触发下游模板校验与聚合缓存失效。 |
| 新鲜度 | 卡片定义变更后，后续模板预览和聚合刷新必须使用最新协议，不允许继续使用已停用定义。 |
| 权限 | 仅门户管理员/运营管理员可修改卡片定义；普通用户无直接写入口。 |
| 审计 | 卡片定义、协议、展示策略、可见规则的变更必须记录审计，并保留 `changedFields`。 |

## 错误场景

| 场景 | 处理要求 |
|------|----------|
| 卡片编码重复 | 拒绝保存，返回唯一性冲突。 |
| 数据源协议不完整或字段映射非法 | 拒绝保存，提示协议校验失败。 |
| 卡片被活动模板引用时直接停用 | 拒绝或要求显式确认影响范围，避免线上模板失效。 |
| 试图为聚合卡片配置绕过聚合层的直连数据源 | 拒绝保存。 |
| 展示策略与权限规则冲突 | 拒绝保存，例如必显卡片被配置为允许隐藏。 |
