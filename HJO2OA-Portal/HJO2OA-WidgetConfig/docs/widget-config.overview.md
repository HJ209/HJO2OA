# 4.2 工作台组件配置中心

## 模块定位

`widget-config` 是门户卡片定义、数据协议和展示策略的设计时真相源，负责统一工作台卡片的类别、数据源绑定、刷新模式和可见规则。它不拥有待办、消息、公告等源数据，也不负责聚合输出。

## 核心职责

- 管理 `WidgetDefinition`、`WidgetDataSourceBinding`、`WidgetDisplayPolicy` 等卡片定义对象。
- 定义卡片类别、渲染类型、数据源协议、字段映射、缓存 TTL 和刷新模式。
- 管理卡片的默认展示策略、隐藏/折叠能力和权限模式。
- 向 `portal-model`、`portal-designer`、`aggregation-api` 提供统一卡片协议，避免前端绕过聚合层拼装数据。

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | 业务域读模型 | 只消费上游模块已经提供的读模型或查询能力，不改造业务主模型。 |
| 上游 | `07-数据服务与集成` | 仅可复用既有数据服务能力，不将门户卡片定义升级为通用开放 API。 |
| 下游 | `portal-model` | 模板只引用卡片定义，不复制数据源协议和展示策略。 |
| 下游 | `aggregation-api` | `dataSourceType=AGGREGATION_QUERY` 的卡片必须经由聚合层输出。 |
| 下游 | `portal-home` | 前端仅按卡片协议渲染，不直接访问业务数据库或多源接口。 |
| 外部边界 | `personalization` | 个性化只能调整模板授权范围内的显隐和顺序，不能改写卡片定义。 |

## 核心能力表

| 能力 | 说明 | 一期实现口径 |
|------|------|--------------|
| 卡片定义管理 | 定义卡片编码、类别、渲染类型、状态和来源模块。 | 支持待办、消息、公告、快捷入口、基础统计等标准卡片。 |
| 数据源绑定 | 维护查询编码、请求/响应协议、字段映射和回退策略。 | 优先支持 `AGGREGATION_QUERY`、`STATIC_CONFIG`、`SHORTCUT`。 |
| 展示策略 | 管理最大条数、宽度、样式和是否允许折叠/隐藏。 | 一期仅支持标准卡片协议，不支持复杂脚本卡片。 |
| 可见规则 | 声明继承门户权限还是自定义作用范围。 | 卡片可见性仍受身份上下文和模板发布范围双重约束。 |
| 变更通知 | 卡片协议变更后通知下游刷新模板校验与聚合缓存。 | 通过 `portal.widget.updated`、`portal.widget.disabled` 驱动。 |

## 典型业务流

1. 管理员创建卡片定义，指定类别、渲染类型和来源模块。
2. 配置数据源绑定、请求/响应协议、字段映射和回退策略。
3. 配置展示策略和可见规则，明确是否允许隐藏、折叠和缓存建议。
4. `portal-model` 在模板布置中引用该卡片，`portal-designer` 将其暴露到组件面板。
5. 卡片协议变更后发布事件，驱动模板校验、聚合缓存失效和前端卡片刷新。

## 一期实现约束

- 卡片定义只拥有展示协议和数据协议，不拥有待办、消息、公告等业务主数据。
- `AGGREGATION_QUERY` 卡片必须通过 `aggregation-api` 输出，不允许前端绕过聚合层直连多源接口。
- 一期不支持复杂脚本卡片、自定义资源包和可执行代码型组件。
- 卡片定义可以被多个模板复用，模板差异必须落在 `WidgetPlacement.overrideProps` 而不是复制卡片。
