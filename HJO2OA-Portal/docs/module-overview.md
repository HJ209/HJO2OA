# 04-门户与工作台-模块功能

## 模块定位

该模块用于对标 O2OA 的门户管理平台能力，负责构建统一工作入口、个性化首页、角色化门户、工作台卡片、快捷入口、统计视图和多场景信息聚合能力。它承担系统第一入口的角色，直接影响用户对整个平台的使用效率与感受。

## 业务目标

- 支持企业门户、部门门户、专题门户和个人工作台
- 支持待办、消息、公告、日程、统计图卡、快捷应用、常用流程等卡片化聚合
- 支持不同角色、岗位、组织的差异化首页配置
- 支持门户模板、页面布局、组件拖拽和个性化设置
- 支持 PC 与移动端门户联动
- 前端布局参考 O2OA 的首页和办公中心模式，形成统一门户首页、左导航工作区和设计器式配置界面

## 边界约束

- 本模块负责门户首页、办公中心和工作台场景下的聚合读模型，不拥有通用业务主数据所有权。
- `aggregation-api` 仅服务门户首页和办公中心场景，不承担通用数据开放职责。
- 通用数据服务、开放 API、连接器和跨系统数据交换统一由 `07-数据服务与集成` 承担。

## 子模块划分

### 4.1 门户页面模型管理

负责门户、页面、子页面和组件区域的定义与版本管理。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 门户管理 | 门户 CRUD、门户分类、门户状态 | 门户列表页、门户新增/编辑弹窗 |
| 页面与子页面管理 | 页面 CRUD、子页面 CRUD、布局结构定义 | 页面管理页、子页面配置 |
| 组件区域定义 | 区域类型、区域配置、区域与卡片绑定 | 区域配置面板 |
| 版本与模板管理 | 版本发布/回滚、模板复用、草稿编辑 | 版本列表页、模板选择器 |
| 门户发布与权限 | 对组织/角色/身份/租户可见范围控制、灰度/定时发布 | 发布配置页、范围选择器 |

### 4.2 工作台组件配置中心

负责工作台卡片的类型、数据源和展示策略管理。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 卡片类型管理 | 待办/消息/统计/内容/业务快捷等卡片类型定义 | 卡片类型列表页、类型配置 |
| 卡片数据源配置 | 数据来源绑定、刷新策略、缓存策略 | 数据源配置面板 |
| 卡片展示策略 | 展示样式、排序规则、数量限制 | 展示策略配置面板 |
| 卡片权限规则 | 按角色/岗位/组织控制卡片可见性 | 卡片权限配置页 |

### 4.3 个性化与角色化配置

负责不同角色的差异化首页和用户个性化设置。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 全局模板配置 | 管理员配置默认门户模板 | 全局模板管理页 |
| 组织/角色模板 | 按组织或角色配置差异化首页模板 | 组织模板管理页、角色模板管理页 |
| 用户个性化 | 用户自定义卡片排序、常用应用收藏、布局微调 | 个性化设置页、拖拽排序 |
| 个性化范围控制 | 限制用户可调整的范围、重置为默认 | 范围配置、重置按钮 |

### 4.4 聚合数据接口

负责跨模块数据聚合，减少首页接口调用复杂度。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 待办聚合 | 待办数量、待办分类统计、紧急待办的门户读模型输出 | 待办卡片组件 |
| 消息聚合 | 未读消息数量、消息分类统计 | 消息卡片组件 |
| 内容聚合 | 最新公告、热门内容、订阅更新 | 公告卡片组件、内容卡片组件 |
| 日程与会议聚合 | 日程安排、会议提醒 | 日程卡片组件、会议卡片组件 |
| 业务看板聚合 | 流程统计、考勤统计、任务进度 | 看板卡片组件 |
| 常用流程与应用 | 常用流程列表、常用应用入口 | 快捷入口卡片组件 |

### 4.5 门户首页与办公中心

负责门户首页展示和办公中心工作区布局。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 门户首页渲染 | 根据门户配置渲染首页、三段式布局（顶部品牌/导航、中部聚合区、底部信息区） | 门户首页页面 |
| 办公中心工作区 | 左侧分类导航（待办/已办/待阅/已阅/草稿）、右侧列表与处理区 | 办公中心页面 |
| 卡片跳转链路 | 卡片点击跳转对应业务详情 | 卡片跳转交互 |
| 移动端首页 | 轻量化移动端工作台、高频卡片优先 | 移动端工作台页面 |

### 4.6 门户设计器

负责管理员维护门户页面布局和组件配置。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 设计器画布 | 布局编辑、区域划分、组件拖入 | 设计器画布区 |
| 组件区 | 左侧组件列表（卡片/导航/轮播/快捷菜单/专题） | 左侧组件面板 |
| 属性区 | 右侧属性编辑、样式配置、数据源绑定 | 右侧属性面板 |
| 预览与发布 | 多终端预览、草稿保存、发布确认 | 预览按钮、发布弹窗 |

## 优化方向

- 门户配置与工作台配置统一底座，避免建设两套首页系统
- 聚合接口统一输出，降低首页接口调用复杂度
- 提高个性化与角色化能力，满足不同岗位办公需求
- 页面布局参考 O2OA 的典型形态，但以现代组件化方式实现，避免静态页面拼接式开发
- 加强移动端工作台体验，突出高频任务处理效率
- 建设模板机制，提高多项目实施复用率

## 典型业务场景

- 普通员工登录后首先看到待办、公告、日程与常用流程
- 领导角色看到审批统计、经营看板、重点通知与会议提醒
- 行政部门配置专题门户展示制度、通知和常用表单入口
- 分公司拥有独立门户模板，但共享集团平台底座
- 用户按个人习惯调整首页卡片顺序与常用应用入口

## 交付边界

该模块负责系统入口和聚合体验，不直接承载具体业务规则。它输出的是面向门户与办公中心的用户侧聚合读模型，而不是平台级通用数据开放能力；需要与消息、流程、内容、日程、会议、搜索、权限等多个模块紧密协作。
