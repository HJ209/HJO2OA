# 4.5 门户首页与办公中心

## 模块定位

`portal-home` 是门户首页、办公中心和移动工作台的渲染层，负责把模板解析结果、个性化视图和聚合卡片快照装配成最终页面。它不拥有模板真相源、卡片协议真相源或业务主数据。

## 核心职责

- 渲染门户首页三段式布局、办公中心左右分栏布局和移动端轻量工作台。
- 根据模板布置和聚合结果加载对应卡片组件，并处理卡片跳转链路。
- 在身份切换、个性化保存、卡片刷新后触发页面级或卡片级重装配。
- 对上游异常执行空态、降级、重试和局部刷新处理。

## 上下游边界

| 方向 | 模块 | 边界说明 |
|------|------|----------|
| 上游 | `portal-model` | 只消费模板解析结果，不管理模板草稿、发布或范围裁决。 |
| 上游 | `personalization` | 只消费个性化覆写结果，不保存用户偏好。 |
| 上游 | `aggregation-api` | 只消费卡片读模型，不绕过聚合层直连业务域接口。 |
| 上游 | `widget-config` | 仅按卡片协议渲染，不管理卡片定义。 |
| 外部边界 | 业务域页面 | 卡片点击后跳转到原业务模块处理，不在门户层重写业务流程。 |
| 外部边界 | 实时推送 | 一期不依赖 WebSocket 推送，刷新以轮询和定向刷新为主。 |

## 核心能力表

| 能力 | 说明 | 一期实现口径 |
|------|------|--------------|
| 门户首页渲染 | 渲染品牌区、导航区、聚合区和底部说明区。 | 使用 `THREE_SECTION` 模板结构。 |
| 办公中心工作区 | 渲染左侧分类导航、右侧列表与处理区。 | 使用 `OFFICE_SPLIT` 模板结构。 |
| 移动端工作台 | 渲染轻量卡片集合和高频入口。 | 使用 `MOBILE_LIGHT` 模板结构。 |
| 卡片交互路由 | 处理卡片点击、快捷入口跳转和列表项深链。 | 跳转目标仍归待办、消息、流程等原业务模块。 |
| 页面降级与刷新 | 处理快照失败、身份切换和定向刷新。 | 支持局部卡片刷新，不要求整页重载。 |

## 典型业务流

1. 用户登录后进入门户首页或办公中心，请求当前身份下的模板解析结果。
2. 读取个性化覆写，得到最终页面结构、卡片顺序和快捷入口。
3. 调用 `aggregation-api` 批量获取当前页面所需卡片快照。
4. 根据布局模式装配卡片组件并渲染页面。
5. 用户点击卡片后跳转到对应业务模块；身份切换或卡片刷新后仅重装配受影响区域。

## 一期实现约束

- `portal-home` 不持久化模板、个性化或聚合快照数据。
- 首页布局、办公中心布局和移动工作台必须复用同一门户模板体系，不得拆成三套独立配置模型。
- 一期不构建独立门户业务 API，页面数据由 `portal-model`、`personalization`、`aggregation-api` 提供。
- 页面刷新采用轮询 + 操作后定向刷新，不引入强依赖实时推送。
