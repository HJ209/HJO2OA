# portal-home 前端合同

## 页面清单

| 页面 | 页面职责 | 主要使用场景 |
|------|----------|--------------|
| 门户首页 | 渲染品牌区、导航区、聚合区和底部说明区。 | PC 首页入口 |
| 办公中心 | 渲染左侧导航、右侧工作区和列表型卡片。 | 事务处理入口 |
| 移动工作台 | 渲染轻量卡片、高频入口和移动端导航。 | 移动办公入口 |
| 空态/无发布页 | 在无可用发布模板或无权限时给出引导。 | 异常或初始化场景 |

## 页面职责与关键交互

| 交互 | 说明 | 依赖模块 |
|------|------|----------|
| 首屏装配 | 按当前身份拉取模板壳、个性化视图和卡片批量数据。 | `portal-model` + `personalization` + `aggregation-api` |
| 卡片局部刷新 | 对单张卡片或单个区域进行刷新，不重载整页。 | `aggregation-api` |
| 办公中心分类切换 | 切换左侧分类后刷新右侧列表或卡片区域。 | `aggregation-api` |
| 卡片跳转 | 卡片点击后进入待办、消息、流程等原业务模块。 | 原业务模块 |
| 异常降级 | 快照失败、无发布模板时展示空态/降级态。 | `portal-home` |

## 模板、卡片、个性化、移动端约束

- 模板约束：首页、办公中心、移动工作台必须共用同一套模板体系，不得形成三套独立配置。
- 卡片约束：页面不得越过 `aggregation-api` 直接拼装业务接口；卡片是否展示受模板和个性化共同约束。
- 个性化约束：页面只消费个性化结果，不提供模板级结构编辑能力。
- 移动端约束：移动端聚焦高频场景，允许裁剪低频卡片，但契约来源仍是同一模板和聚合协议。

## 身份切换影响

- 身份切换后必须立即丢弃旧身份的模板壳、个性化视图和卡片数据，并重新执行首屏装配。
- 切换期间允许展示加载态，但不得继续展示旧身份的待办数、未读数或组织相关入口。
- 若新身份没有命中任何发布模板，应展示空态页而不是沿用旧身份页面。
