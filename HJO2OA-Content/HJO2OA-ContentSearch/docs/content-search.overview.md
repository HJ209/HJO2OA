# content-search 子模块总览

## 模块定位

`content-search` 位于发布权限与检索投影层，负责全文索引、分类筛选、收藏订阅、相关推荐和面向门户/搜索页的读模型输出。

## 核心职责

| 核心职责 | 说明 | 输出结果 |
|----------|------|----------|
| 全文索引投影 | 将已发布内容投影为可检索索引。 | 支撑关键字搜索和过滤查询 |
| 收藏/订阅偏好 | 维护用户收藏、订阅目标与基础推荐。 | 支撑个人关注内容入口 |
| 搜索读模型输出 | 输出列表结果、筛选项和相关推荐。 | 为门户、搜索页和内容中心提供统一查询接口 |

## 上下游边界

| 方向 | 模块/能力 | 边界说明 |
|------|-----------|----------|
| 上游 | `content-lifecycle`、`content-permission` | 只消费已发布且可见的内容，不维护主模型状态 |
| 上游 | `category-management` | 引用栏目、分类、标签与专题语义作为检索维度 |
| 下游 | 内容搜索页、`04-Portal`、订阅场景 | 输出读模型，不负责发布、审批或通知送达 |
| 协同 | `06-Messaging` | 可把订阅变更作为提醒意图来源，但不直接发送消息 |

## 核心能力

| 能力 | 说明 | 关键约束 |
|------|------|----------|
| 全文检索 | 支持关键字、栏目、分类、标签、专题和时间过滤。 | 仅索引已发布且有效内容 |
| 收藏与订阅 | 维护个人收藏、关注栏目/专题/标签等目标。 | 用户偏好不能突破可见性边界 |
| 相关推荐 | 基于栏目、标签和热度输出基础相关内容。 | 一期只做规则型推荐，不做复杂画像算法 |

## 典型业务流

1. 模块消费发布事件，将内容与分类标签投影为全文索引。
2. 用户发起搜索时先做关键字与筛选查询，再通过可见性结果裁剪。
3. 用户收藏或订阅内容/栏目后，系统记录偏好并用于个人入口展示。
4. 内容下线或权限变化时，索引与个人入口结果同步刷新。

## 一期实现约束

- 搜索索引和推荐结果都是投影，不得反向替代内容主模型。
- 只有通过 `content-permission` 裁决的结果才能进入最终搜索返回集。
- 一期不实现复杂语义搜索、画像推荐和跨语言检索，只提供基础全文与条件筛选。
