# category-management 子模块总览

## 模块定位

`category-management` 位于内容域的语义组织层，负责栏目树、分类树、标签组、专题骨架以及栏目权限规则的统一治理，为稿件挂载、检索筛选和门户展示提供稳定语义骨架。

## 核心职责

| 核心职责 | 说明 | 输出结果 |
|----------|------|----------|
| 栏目树治理 | 维护栏目层级、排序、启停和编码规则。 | 为内容挂载、导航和聚合提供统一入口骨架 |
| 分类/标签/专题定义 | 统一管理分类树、标签组、专题骨架与归属关系。 | 为筛选、专题推荐和运营编排提供语义对象 |
| 栏目权限规则 | 绑定组织、岗位、角色和显式拒绝规则。 | 为发布可见性提供第一道边界 |

## 上下游边界

| 方向 | 模块/能力 | 边界说明 |
|------|-----------|----------|
| 上游 | `01-OrgPerm` 身份上下文、组织与角色 | 仅消费主体与范围解析结果，不持有组织主数据 |
| 下游 | `content-lifecycle`、`content-permission` | 提供栏目骨架和权限规则，不承担稿件状态流转 |
| 下游 | `content-search`、`04-Portal` | 输出已发布内容所需的分类与展示语义，不直接负责搜索索引或首页模板 |
| 协同 | `00-Infrastructure` 字典、审计、缓存 | 复用基础能力记录配置变更和刷新缓存 |

## 核心能力

| 能力 | 说明 | 关键约束 |
|------|------|----------|
| 栏目树维护 | 支持新增、调整顺序、停用、恢复与层级校验。 | 同级编码唯一，停用栏目不得挂新稿件 |
| 分类/标签治理 | 维护分类树、标签组、标签合并与专题结构。 | 标签和专题不能替代栏目权限 |
| 权限规则编排 | 为栏目配置组织、岗位、角色范围及显式拒绝规则。 | 栏目权限优先于个体偏好和搜索收藏 |
| 结构影响预览 | 预览栏目调整对稿件挂载和门户导航的影响。 | 一期仅提供影响提示，不自动迁移历史内容 |

## 典型业务流

1. 内容管理员创建或调整栏目树，并为栏目分配分类、标签组和专题骨架。
2. 管理员配置栏目可见范围与显式拒绝规则，形成基础权限边界。
3. `content-lifecycle` 在建稿或发布时引用栏目与分类语义。
4. `content-search` 和 `04-Portal` 消费栏目与专题结构输出检索筛选和展示信息。

## 一期实现约束

- 一期只治理栏目、分类、标签、专题和栏目权限，不引入复杂多语言专题运营。
- 组织结构变化后允许刷新权限缓存，但不回写历史稿件主数据。
- 不保存正文、封面或附件二进制，内容实体仍由 `content-lifecycle` 和 `content-storage` 管理。
