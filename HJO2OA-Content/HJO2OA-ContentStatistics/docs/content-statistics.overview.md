# content-statistics 子模块总览

## 模块定位

`content-statistics` 位于运营统计层，负责阅读、收藏、下载等行为明细追加、热点排行聚合以及发布效果分析输出。

## 核心职责

| 核心职责 | 说明 | 输出结果 |
|----------|------|----------|
| 行为明细采集 | 记录阅读、下载、收藏等内容消费行为。 | 形成可追溯明细日志 |
| 统计快照聚合 | 按内容、栏目、时间窗口聚合热度和转化指标。 | 支撑运营看板和排行榜 |
| 效果分析输出 | 输出阅读趋势、订阅转化和发布效果摘要。 | 为运营人员和门户图卡提供统计视图 |

## 上下游边界

| 方向 | 模块/能力 | 边界说明 |
|------|-----------|----------|
| 上游 | 内容详情、`content-search`、门户阅读入口 | 只消费行为事件和内容基础标识，不持有正文 |
| 上游 | `content-lifecycle` | 只统计已发布或历史已发布内容，不驱动状态变更 |
| 下游 | 运营看板、`07-DataServices/report` | 输出聚合快照，不直接暴露原始明细给普通页面 |
| 协同 | `06-Messaging` | 热度阈值或订阅转化可形成提醒意图，但不负责通知发送 |

## 核心能力

| 能力 | 说明 | 关键约束 |
|------|------|----------|
| 阅读明细记录 | 接收阅读、下载、收藏等行为并追加写入。 | 明细只追加不覆盖 |
| 热度排行 | 按时间窗生成阅读排行、收藏排行和专题热点。 | 排行为读模型，允许异步延迟 |
| 发布效果分析 | 汇总发布后阅读走势、来源和订阅转化。 | 仅面向已发布内容统计 |

## 典型业务流

1. 内容详情页、搜索结果和门户入口上报阅读、下载、收藏等行为。
2. 模块按时间窗口聚合行为明细，生成热点排行和效果快照。
3. 运营看板和数据服务消费聚合结果展示趋势和转化指标。
4. 内容下线或归档后保留历史统计，但不再参与公开热度榜。

## 一期实现约束

- 行为明细采用追加式写入，聚合快照允许异步刷新。
- 统计结果不得反向修改内容状态或搜索排序规则。
- 一期不做复杂画像分析，只提供基础阅读、收藏、订阅与发布效果指标。
