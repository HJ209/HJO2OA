# 03-内容与知识-模块功能

## 模块定位

该模块用于复刻 O2OA 的内容管理平台能力，负责信息发布、栏目管理、知识库、制度文档、新闻公告、专题内容、文档版本、分类标签、订阅收藏与全文检索等内容型场景。它承担企业信息沉淀、知识共享和制度传达的重要职责。

## 业务目标

- 支持新闻、公告、制度、知识文档、专题内容、经验沉淀等内容类型
- 支持栏目、分类、标签、目录、权限、版本、审核、发布与归档
- 支持内容编辑、附件管理、封面配置、推荐位和置顶能力
- 支持全文检索、收藏、订阅、热点排行和相关推荐
- 支持 PC 与移动端阅读体验统一

## 边界约束

- 本模块拥有内容主模型、内容权限、内容检索、内容版本和订阅收藏等底座能力。
- `05-协同办公应用` 中涉及公告、制度、知识展示、网盘与文件协同等内容型场景时，应优先复用本模块与 `00-附件中心` 的底座能力。
- `04-门户与工作台` 可以聚合本模块输出的读模型数据，但不负责定义内容主规则与内容生命周期。

## 子模块划分

### 3.1 栏目与分类管理

负责内容栏目、分类目录和标签体系的维护，是内容的组织骨架。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 栏目管理 | 栏目 CRUD、栏目层级、发布流程绑定、模板策略 | 栏目列表页、栏目新增/编辑弹窗 |
| 分类目录管理 | 分类 CRUD、分类层级、分类与栏目关联 | 分类管理页、分类树浏览 |
| 标签体系管理 | 标签 CRUD、标签分组、标签合并 | 标签管理页、标签选择器 |
| 专题归属 | 专题 CRUD、专题与内容关联、专题排序 | 专题管理页、专题内容配置 |
| 栏目权限配置 | 按组织/角色/岗位/身份/用户配置可见范围 | 栏目权限配置页、范围选择器 |

### 3.2 内容生命周期管理

负责内容从草稿到归档的完整状态流转。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 草稿编辑 | 草稿创建、草稿保存、草稿恢复 | 内容编辑页、草稿保存按钮 |
| 送审与审核 | 送审操作、审核流程接入、审核意见记录 | 审核处理页、审核意见区 |
| 发布管理 | 发布、定时发布、撤回、下线 | 发布确认弹窗、定时发布配置 |
| 归档管理 | 归档、归档数据查询、归档恢复 | 归档列表页、归档详情查看 |
| 生命周期记录 | 每阶段操作人/时间/变更说明记录 | 变更历史时间线 |

### 3.3 内容正文与附件管理

负责内容正文、附件和版本信息的存储与索引。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 正文编辑与存储 | 富文本正文存储、摘要提取、正文与主数据分离 | 富文本编辑器、摘要编辑区 |
| 附件管理 | 附件上传/删除/排序、附件与内容关联 | 附件上传区、附件列表、预览入口 |
| 封面配置 | 封面图片上传/裁剪/替换 | 封面上传区、裁剪工具 |
| 版本管理 | 版本号自增、版本回溯、版本对比 | 版本历史页、版本对比视图 |
| 引用关系 | 内容间引用、关联推荐数据 | 关联内容选择器、引用链查看 |

### 3.4 权限与可见范围控制

负责内容级别的访问控制。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 发布范围配置 | 全部人员/指定组织/指定角色/指定知识群体 | 发布范围选择器、组织/角色选择器 |
| 可见范围校验 | 查询时按组织/角色/岗位/身份/用户过滤 | 不可见内容隐藏、权限提示 |
| 内容级权限 | 单篇内容的编辑/查看/下载/评论权限 | 内容权限配置弹窗 |

### 3.5 搜索与推荐

负责全文检索和内容推荐能力。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 全文检索 | 全文索引、关键词搜索、搜索结果排序 | 统一搜索框、搜索结果页 |
| 分类与标签筛选 | 按栏目/分类/标签/专题筛选 | 筛选面板、标签云 |
| 热门排行 | 阅读量排行、收藏量排行、下载量排行 | 热门排行图卡、排行列表 |
| 相关推荐 | 基于标签/分类/关键词的关联内容推荐 | 推荐内容列表 |
| 订阅推送 | 栏目订阅、标签订阅、更新通知 | 订阅配置页、更新提醒 |

### 3.6 内容审计与统计

负责内容运营数据记录和分析。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 阅读与互动统计 | 阅读量/收藏量/评论量/订阅量/下载量统计 | 统计图卡、阅读量展示 |
| 发布与审核统计 | 发布人统计、审核人统计、修改历史 | 运营统计页、发布效果分析 |
| 收藏与订阅管理 | 用户收藏 CRUD、栏目/标签订阅 CRUD | 收藏列表页、订阅管理页 |

## 优化方向

- 将信息发布和知识库建设在同一内容底座上统一建模
- 建立更强的全文搜索、标签体系和订阅体系
- 加强版本追踪和审核留痕能力
- 强化移动端阅读和附件预览体验
- 将内容管理从传统公告系统升级为企业知识资产平台

## 典型业务场景

- 行政部门发布公司制度并按组织范围可见
- 人事部门发布公告并推送到消息中心和工作台
- 专家团队沉淀项目经验文档供全公司检索复用
- 员工订阅某个专题栏目，内容更新后自动接收提醒
- 管理者查看知识库热点内容和阅读排行，优化内容运营

## 交付边界

该模块负责内容型信息的全生命周期管理，是公告、制度、知识、专题等内容型场景的统一底座，不承载审批事务引擎本身，但可以与流程模块协同完成内容审核、发布审批和制度确认等业务流程。业务模块不得绕过本模块自建内容主模型和统一检索模型。
