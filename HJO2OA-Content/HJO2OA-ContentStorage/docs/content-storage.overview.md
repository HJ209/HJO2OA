# content-storage 子模块总览

## 模块定位

`content-storage` 位于内容主模型与生命周期层，负责正文快照、封面引用、附件绑定、版本关系和内容关联，是内容版本资产的管理中心。

## 核心职责

| 核心职责 | 说明 | 输出结果 |
|----------|------|----------|
| 正文与版本快照 | 管理正文内容、版本草稿和发布快照。 | 为发布提供可追溯版本实体 |
| 封面与附件绑定 | 绑定封面、正文附件和附属资源引用。 | 输出稳定的资源引用关系 |
| 内容关联治理 | 维护相关文章、引用关系和版本间继承。 | 支撑内容回溯和推荐关系 |

## 上下游边界

| 方向 | 模块/能力 | 边界说明 |
|------|-----------|----------|
| 上游 | `00-Infrastructure` 附件中心 | 只持有附件引用与绑定关系，不存二进制文件 |
| 上游 | `content-lifecycle` | 由生命周期模块决定哪个版本进入审核和发布 |
| 下游 | `content-lifecycle`、内容详情页 | 提供可选版本、正文渲染源和资源绑定信息 |
| 协同 | `content-search` | 提供可索引正文与标签引用，不负责索引构建 |

## 核心能力

| 能力 | 说明 | 关键约束 |
|------|------|----------|
| 版本管理 | 创建草稿版本、复制历史版本、查看版本差异。 | 已发布版本不可原地改写 |
| 资源绑定 | 绑定封面、附件和正文内资源引用。 | 资源引用必须来自 `00` 的合法对象 |
| 关系管理 | 建立相关文章、上下位内容和专题引用关系。 | 关系绑定到版本维度，确保历史可回溯 |

## 典型业务流

1. 编辑创建或复制正文版本，并保存结构化正文与摘要。
2. 模块绑定封面、附件和内容引用关系，形成完整版本资产。
3. `content-lifecycle` 选择某个版本进入审核和发布。
4. 发布后该版本转为只读，后续修改需基于新版本继续编辑。

## 一期实现约束

- 发布对象是具体版本，已发布版本和其资源绑定不可原地改写。
- 正文附件、封面、嵌入资源都必须通过 `00` 附件中心引用，不直接存储二进制。
- 一期支持版本复制、附件绑定和关系管理，不引入复杂多人实时协同编辑。
