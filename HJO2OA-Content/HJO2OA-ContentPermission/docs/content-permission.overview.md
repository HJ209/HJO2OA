# content-permission 子模块总览

## 模块定位

`content-permission` 位于发布权限与检索投影层，负责发布范围建模、主体匹配、显式允许/拒绝裁决和运行时可见性判断。

## 核心职责

| 核心职责 | 说明 | 输出结果 |
|----------|------|----------|
| 发布范围建模 | 为已发布内容配置组织、岗位、角色、人员等范围。 | 形成可落库的发布范围规则 |
| 运行时可见性裁决 | 将栏目权限、发布范围和显式拒绝规则合并计算。 | 输出用户可见/不可见结论 |
| 可见性缓存与预览 | 为检索、门户和详情页提供缓存命中与预览能力。 | 缩短运行时判断链路 |

## 上下游边界

| 方向 | 模块/能力 | 边界说明 |
|------|-----------|----------|
| 上游 | `category-management` | 继承栏目级权限规则，不重定义栏目可见性 |
| 上游 | `content-lifecycle` | 仅对已发布内容生成与维护发布范围 |
| 上游 | `01-OrgPerm` 身份上下文 | 复用主体解析和组织岗位关系，不持有身份主数据 |
| 下游 | `content-search`、`04-Portal`、内容详情查询 | 输出最终可见性结论，不负责搜索排序和页面渲染 |

## 核心能力

| 能力 | 说明 | 关键约束 |
|------|------|----------|
| 发布范围配置 | 配置组织、岗位、角色、人员和租户级范围。 | 只面向已发布内容和发布批次 |
| 可见性计算 | 综合栏目规则、发布范围、显式允许/拒绝规则计算命中结果。 | 显式拒绝优先，栏目权限必须先通过 |
| 预览与诊断 | 查看某主体为何能看/不能看某条内容。 | 预览不落库，仅输出命中链路 |

## 典型业务流

1. `content-lifecycle` 发布内容后创建或激活对应发布范围。
2. 管理员按组织或角色配置附加可见范围与显式拒绝规则。
3. 搜索、门户和详情读取时传入身份上下文，模块执行可见性裁决。
4. 裁决结果以短周期缓存供下游复用，并在组织或权限变化时刷新。

## 一期实现约束

- 只有已发布且处于有效期内的内容才参与可见性计算。
- 可见性至少同时经过栏目权限和发布范围两层校验，不能被收藏、订阅绕过。
- 一期不做复杂 AB 测试型可见策略，只支持显式范围和拒绝规则。
