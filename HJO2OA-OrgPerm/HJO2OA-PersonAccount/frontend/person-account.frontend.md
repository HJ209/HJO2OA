# person-account 前端契约

## 页面清单

| 页面/组件 | 端别 | 页面职责 | 备注 |
|-----------|------|----------|------|
| 人员列表页 | 管理端 | 查询人员、状态筛选、查看主属组织和主岗摘要 | 支持批量启停用入口 |
| 人员详情页 | 管理端 | 展示人员档案、账号区、身份总览和审计摘要 | 任职信息只读聚合展示 |
| 人员新增/编辑弹窗 | 管理端 | 维护档案、组织部门归属和基础信息 | 不直接维护任职关系 |
| 账号列表页 | 管理端 | 查询账号、认证类型、主账号标记和安全状态 | 支持锁定状态筛选 |
| 账号详情/编辑抽屉 | 管理端 | 查看或维护账号状态、外部认证绑定、主账号关系 | 凭据变更走专用安全操作 |
| 密码重置与锁定解锁弹窗 | 管理端 | 承载高风险安全操作确认 | 必须显示审计原因输入 |
| 导入预校验弹窗 | 管理端 | 上传人员模板、查看错误行、确认导入 | 与组织导入风格保持一致 |

## 页面职责

| 页面/组件 | 关键职责 | 依赖接口 |
|-----------|----------|----------|
| 人员列表页 | 展示人员列表、筛选条件、批量状态切换和详情入口 | `GET /persons`、`POST /persons/status:batch` |
| 人员详情页 | 展示档案、账号列表、身份总览、状态流转入口 | `GET /persons/{personId}`、`GET /persons/{personId}/identity-summary` |
| 人员新增/编辑弹窗 | 创建或编辑人员基础信息 | `POST /persons`、`PUT /persons/{personId}` |
| 账号列表页 | 展示账号列表、锁定状态和最近登录情况 | `GET /accounts` |
| 账号详情/编辑抽屉 | 查看账号详情、切换主账号、修改状态 | `GET /accounts/{accountId}`、`PUT /accounts/{accountId}`、`POST /persons/{personId}/accounts/{accountId}/primary` |
| 密码重置与锁定解锁弹窗 | 处理锁定、解锁、密码重置 | `POST /accounts/{accountId}/lock`、`POST /accounts/{accountId}/unlock`、`POST /accounts/{accountId}/password-reset` |
| 导入预校验弹窗 | 执行人员导入预校验与确认 | `POST /person-import/preview`、`POST /person-import/{jobId}/confirm` |

## 关键交互

- 人员列表应把人员状态、主属组织、主岗摘要和账号数量放在同一视图，方便快速治理。
- 人员详情页中的“身份总览”需明确标记“只读聚合结果”，并提供跳转到任职模块的入口。
- 账号详情页应将“主账号”“锁定状态”“最近登录时间”“认证类型”集中展示，减少跨页切换。
- 密码重置、锁定、解锁都必须使用二次确认弹窗，并要求填写或选择操作原因。
- 导入页必须支持错误行定位和错误清单导出，避免管理员只能重复整包上传排错。

## 权限与状态约束

- 页面权限至少区分 `person.read`、`person.write`、`person.disable`、`person.import`、`account.read`、`account.write`、`account.lock`、`account.unlock`、`account.reset-password`。
- 离职人员详情可查看但默认只读，不允许直接编辑档案或启用其账号。
- 被锁定账号在列表中应突出显示，但管理员仍可查看详情和审计轨迹。
- 没有 `account.reset-password` 权限的用户可以查看账号安全状态，但不能打开重置弹窗。

## 身份切换、移动端与管理端差异

- 本模块主要是管理端能力，不提供移动端人员与账号维护页面。
- 门户端或移动端可消费账号安全结果，但不承载人员档案编辑。
- 当前管理员的身份切换只影响其可见人员范围和可执行动作，不影响人员详情中的主岗/兼岗事实展示。
