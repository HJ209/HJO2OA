# 1.3 人员与账号管理总览

## 模块定位

`person-account` 持有 `Person` 与 `Account` 两个聚合根，负责人员档案、登录账号、主账号规则和账号安全状态。它明确区分“人员主数据”和“登录身份”，保证人员变化不直接破坏账号模型，也不把任职关系、当前身份上下文或角色集合固化到本模块。

## 核心职责

- 维护人员档案基础信息、主属组织、主属部门、状态和入离转相关事实。
- 维护账号、主账号、一人多号、认证类型和锁定状态。
- 与 `00-安全工具` 协作完成密码策略、凭据加密、登录失败锁定和密码重置。
- 提供人员身份总览视图，聚合显示任职、主岗/兼岗和账号绑定摘要，但不持有任职真相源。
- 发布人员与账号领域事件，为身份上下文、同步治理和审计模块提供事实输入。

## 上下游边界

| 类型 | 对象 | 边界说明 |
|------|------|----------|
| 上游输入 | `org-structure` | 人员主属组织、主属部门必须引用有效组织/部门。 |
| 上游输入 | `00-安全工具` | 密码复杂度、凭据加密、锁定阈值、脱敏规则由基础安全能力提供。 |
| 上游输入 | `org-sync-audit` | 外部 HR/目录同步通过本模块写接口创建或更新人员与账号。 |
| 下游输出 | `position-assignment` | 人员停用或离职会驱动任职转历史或失效校验。 |
| 下游输出 | `identity-context` | 当前身份上下文建立在有效人员与有效账号之上。 |
| 下游输出 | `role-resource-auth`、`data-permission` | 仅消费 `personId` 作为授权主体，不维护账号事实。 |
| 排除范围 | 任职关系真相源 | 主岗、兼岗、任职起止时间的写入归属 `position-assignment`。 |
| 排除范围 | 对外登录入口 | 一期认证入口由统一安全能力承接，本模块负责账号状态和登录结果回写。 |

## 核心能力表

| 能力域 | 能力说明 | 核心规则 | 主要输出 |
|--------|----------|----------|----------|
| 人员档案管理 | 人员新增、编辑、停用、离职 | 工号租户内唯一；主属组织必填；主属部门若存在必须属于主属组织 | 人员列表、人员详情、人员事件 |
| 账号管理 | 账号创建、编辑、启停用、锁定/解锁 | 同一人员可多账号，但同一认证类型最多一个，且仅一个主账号 | 账号列表、账号详情、账号事件 |
| 人员与账号绑定 | 管理主账号和多账号关系 | 账号必须绑定人员后才可启用；主账号切换必须保证唯一性 | 人员-账号关系、主账号切换结果 |
| 认证安全协作 | 登录成功/失败回写、密码重置、锁定策略接入 | 登录入口不在本模块；锁定与重置进入统一审计 | 账号安全状态、登录结果回写 |
| 批量治理 | 人员导入、批量启停用、结果反馈 | 一期批量治理聚焦人员与账号，不承担批量调岗真相写入 | 导入结果、状态变更结果、审计记录 |

## 典型业务流

### 1. 新员工入职建档并创建主账号

1. 管理员先创建人员档案，填入工号、姓名、主属组织、主属部门和基本联系方式。
2. 系统校验工号唯一、组织有效、部门归属合法。
3. 管理员为该人员创建密码账号或外部认证账号，并标记主账号。
4. 写入成功后发布 `org.person.created` 和 `org.account.created`，由任职、身份上下文和审计模块继续消费。

### 2. 人员停用或离职

1. 管理员将人员状态改为 `DISABLED` 或 `RESIGNED`，并填写原因。
2. 系统同步锁定该人员全部有效账号，保留档案和历史任职引用。
3. 分别发布 `org.person.disabled` 或 `org.person.resigned`，以及必要的账号锁定事件。

### 3. 账号锁定、解锁与密码重置

1. 登录失败达到阈值或管理员手动操作时，账号状态切换为锁定。
2. 管理员可在账号详情中解锁或发起密码重置。
3. 登录成功/失败和锁定/解锁写回账号安全状态，并进入统一审计。
4. 密码重置一期不单独作为领域事件真相源，而是以账号更新审计和安全日志体现。

## 一期实现约束

- 人员档案与账号必须分层建模，不允许在人员表中直接嵌入账号凭据字段。
- 账号启停用、锁定和密码重置只能改变账号状态，不得绕过人员状态约束重新激活离职人员。
- 批量治理一期只覆盖人员与账号导入、启停用，不把批量调岗作为本模块真相写入能力。
- 认证入口、密码策略和登录风险控制依赖 `00-安全工具`，本模块负责状态回写和审计归档。
- 人员详情中的任职、主岗和兼岗信息只读展示，真实维护入口归属 `position-assignment`。
