# 1.6 身份切换与会话上下文总览

## 模块定位

`identity-context` 是 `01-组织与权限` 的运行时会话层，负责把人员、账号、任职和角色继承事实组装成统一 `IdentityContext`。它是所有业务模块访问控制的唯一入口，不持有主数据真相源，也不把“当前身份”回写到人员、岗位或任职表。

## 核心职责

- 登录后基于主岗计算默认身份上下文。
- 输出当前身份、可切换身份列表、当前组织/部门/岗位和继承角色集合。
- 支持用户在主岗与兼岗之间切换，并保证切换后菜单、待办、消息、门户卡片和数据权限同步刷新。
- 监听任职、角色、人员、账号状态变化，异步重算会话缓存。
- 发布身份切换事件，供业务模块和缓存治理消费。

## 上下游边界

| 类型 | 对象 | 边界说明 |
|------|------|----------|
| 上游输入 | `person-account` | 当前账号、人员状态必须有效，锁定或离职会终止当前上下文可用性。 |
| 上游输入 | `position-assignment` | 主岗、兼岗和任职有效期决定可切换身份集合。 |
| 上游输入 | `role-resource-auth` | 继承角色集合来源于岗位绑定角色和人员直授角色。 |
| 下游输出 | 全部业务模块 | 所有资源权限、数据权限、菜单和待办计算都必须以本模块输出为准。 |
| 下游输出 | `data-permission` | 输出组织/部门/岗位/人员/角色集合，用于行级和字段级决策。 |
| 下游输出 | 门户、消息、流程等模块 | 切换身份后统一触发前端和缓存刷新。 |
| 排除范围 | 主数据写入 | 不创建人员、岗位、任职、角色记录。 |
| 排除范围 | 多身份并行会话 | 一期一个会话只允许一个当前身份，上下文切换覆盖旧值。 |

## 核心能力表

| 能力域 | 能力说明 | 核心规则 | 主要输出 |
|--------|----------|----------|----------|
| 默认上下文计算 | 登录后计算主岗上下文 | 主岗必选且唯一；若缺失主岗则登录后上下文不成立 | 当前身份上下文、默认角色集合 |
| 可切换身份列表 | 返回主岗和兼岗候选 | 仅返回当前有效任职；已停用、已到期、已失效岗位不可切换 | 候选身份列表 |
| 身份切换 | 主岗与兼岗切换 | 切换结果仅存在于会话/缓存，不回写任职主数据 | 新上下文、切换事件、刷新信号 |
| 上下文刷新 | 任职、角色、人员、账号变化后的重算 | 后台事件触发应支持异步补偿和幂等重算 | 刷新后的上下文、失效缓存 |
| 终端协同 | 门户端、移动端切换一致性 | 一期不支持跨端独立身份栈，同一会话只有一个当前身份 | 门户和移动端上下文刷新结果 |

## 典型业务流

### 1. 登录后生成默认身份

1. 账号认证通过后，系统获取当前人员和其有效任职集合。
2. 以 `PRIMARY` 任职计算默认上下文，组装当前组织、部门、岗位和继承角色。
3. 将结果写入会话缓存，并返回给门户或业务模块。

### 2. 从主岗切换到兼岗

1. 用户在门户或移动端打开身份切换入口，查看可切换岗位列表。
2. 选择目标兼岗后，系统校验该任职仍有效、岗位未停用、人员和账号状态正常。
3. 切换成功后发布 `org.identity.switched`，并统一刷新菜单、待办、消息和数据权限缓存。

### 3. 任职或授权后台变化后的上下文重算

1. 任职新增、主岗变更、角色直授变化或账号锁定后，事件进入上下文重算流程。
2. 系统按人员维度刷新上下文缓存，必要时将当前兼岗回退到主岗或终止当前会话。
3. 若重算失败，应进入补偿重试，不允许留下半刷新的上下文。

## 一期实现约束

- `IdentityContext` 只存在于运行时会话或缓存，不落成独立主数据表。
- 一期仅支持主岗和兼岗切换，不支持多身份并行激活。
- 切换失败时必须保持旧上下文继续有效，不能出现菜单已切换而数据权限未切换的半成功状态。
- 所有业务模块都必须消费本模块输出，禁止自行拼装当前组织、岗位和角色集合。
- 移动端与门户端共用同一套上下文口径，不维护独立身份规则。
