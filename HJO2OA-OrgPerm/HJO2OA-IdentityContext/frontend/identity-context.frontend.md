# identity-context 前端契约

## 页面清单

| 页面/组件 | 端别 | 页面职责 | 备注 |
|-----------|------|----------|------|
| 门户端身份切换入口 | 门户端 | 展示当前身份并打开可切换身份列表 | 通常位于全局头部或工作台顶部 |
| 当前身份展示区 | 门户端/管理端 | 展示当前组织、部门、岗位和岗位类型标签 | 管理端以只读展示为主 |
| 切换确认弹层 | 门户端/移动端 | 确认兼岗切换或切回主岗，并提示全局刷新 | 高风险操作需要二次确认 |
| 移动端身份切换页 | 移动端 | 简化展示身份列表和切换结果反馈 | 适配窄屏和弱网场景 |

## 页面职责

| 页面/组件 | 关键职责 | 依赖接口 |
|-----------|----------|----------|
| 门户端身份切换入口 | 展示当前岗位与身份标签，打开候选身份列表 | `GET /identity-context/current`、`GET /identity-context/available` |
| 当前身份展示区 | 在门户头部、管理端头部或工作台中展示当前上下文 | `GET /identity-context/current` |
| 切换确认弹层 | 发起切换并处理切换后的全局刷新 | `POST /identity-context/switch`、`POST /identity-context/reset-primary` |
| 移动端身份切换页 | 展示可切换身份、刷新结果和失败提示 | 同上 |

## 关键交互

- 页面初始化时先查询当前上下文，再加载候选身份列表，避免出现标签和候选项不一致。
- 切换到兼岗后必须统一刷新菜单、待办、消息、门户卡片和数据权限相关页面。
- 若切换失败，前端必须保留旧身份显示，不得先行切换 UI 再等待接口结果。
- 移动端在弱网场景下要明确展示“切换中”“切换成功”“切换失败”，并支持手动重试。

## 权限与状态约束

- 用户只能切换自己的有效兼岗，不存在管理端代切换能力。
- 已停用、已到期、已失效的岗位应在候选列表中隐藏或显示为禁用，并标注原因。
- 当前账号被锁定、人员已停用或离职时，应直接禁止切换入口并展示明确提示。
- 管理端只读展示当前上下文，不承载批量身份管理功能。

## 身份切换、移动端与管理端差异

- 门户端是身份切换主入口，强调全局刷新和当前身份感知。
- 移动端提供同等语义的切换能力，但交互更轻量，优先保障切换反馈和缓存刷新。
- 管理端默认只展示当前身份上下文，除非业务页面显式需要，否则不提供切换入口，避免误把配置后台当成门户工作台。
