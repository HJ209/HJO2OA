# 1.2 岗位与任职管理总览

## 模块定位

`position-assignment` 持有 `Position` 与 `Assignment` 两个聚合根，负责岗位定义、人员任职关系、唯一主岗约束和兼岗维护。它是 `identity-context` 计算默认身份的基础，也是 `role-resource-auth` 做岗位继承授权的锚点，但不持有岗位-角色绑定真相源。

## 核心职责

- 维护岗位基础信息、组织归属、部门归属、岗位类别、岗位等级和启停用状态。
- 维护人员任职关系，保证“唯一主岗 + 多兼岗 + 同人同岗不重复”。
- 在任职变更时发布标准领域事件，驱动身份上下文和权限缓存重算。
- 提供岗位来源、任职列表和有效角色来源的查询视图，支撑管理端配置和排错。
- 对人员停用、离职、组织/部门停用等上游事实做约束响应，但不越权改写其它模块主数据。

## 上下游边界

| 类型 | 对象 | 边界说明 |
|------|------|----------|
| 上游输入 | `org-structure` | 岗位必须绑定有效组织，可选绑定有效部门；组织/部门停用会影响新增和调岗校验。 |
| 上游输入 | `person-account` | 任职对象必须是有效人员；人员停用或离职后，本模块将任职关系转为历史状态。 |
| 上游输入 | `role-resource-auth` | 岗位继承角色的真相源在 `org_position_role`，本模块只读取继承结果，不负责绑定写入。 |
| 下游输出 | `identity-context` | 任职事实是默认身份、兼岗切换和上下文重算的直接输入。 |
| 下游输出 | `role-resource-auth` | 岗位和任职事件触发角色继承计算、缓存失效和授权重算。 |
| 下游输出 | `org-sync-audit` | 同步治理通过本模块写接口落库岗位和任职事实，并消费事件生成审计。 |
| 排除范围 | 身份会话状态 | 主岗/兼岗切换结果不持久化在 `Assignment` 中，只输出事实事件。 |
| 排除范围 | 岗位角色绑定写入 | 岗位关联角色的新增、删除和模板应用由 `role-resource-auth` 负责。 |

## 核心能力表

| 能力域 | 能力说明 | 核心规则 | 主要输出 |
|--------|----------|----------|----------|
| 岗位生命周期管理 | 岗位创建、编辑、停用、归属维护 | 岗位必须归属组织；部门可为空但若存在必须与组织一致；编码租户内唯一 | 岗位列表、岗位详情、岗位事件 |
| 任职关系管理 | 新增、编辑、移除任职关系 | 同一人员同一岗位不可重复；任职可设置起止时间；状态可独立转历史 | 任职列表、任职事件、身份基础事实 |
| 主岗兼岗约束 | 保证主岗唯一且必选 | 删除主岗前必须先指定新主岗；兼岗不限数量但不可替代主岗必选规则 | `org.assignment.primary-changed` 事件、约束校验结果 |
| 角色来源查看 | 只读查看岗位继承和人员最终角色来源 | 岗位角色绑定不在本模块写入；仅展示来自岗位和例外直授的来源摘要 | 角色来源视图、继承链排错信息 |
| 变更影响预览 | 停用岗位、调整归属、切换主岗前返回影响摘要 | 至少提示受影响任职数量、身份上下文重算范围和缓存刷新范围 | 影响预览、确认写入依据 |

## 典型业务流

### 1. 新增岗位并建立组织归属

1. 管理员创建岗位，填写编码、名称、组织、部门、岗位类别和岗位等级。
2. 系统校验岗位编码唯一、所属组织有效、所属部门与组织一致。
3. 写入成功后发布 `org.position.created`，供角色继承和审计模块消费。

### 2. 为人员配置主岗与兼岗

1. 在人员任职页先选择主岗，再按需补充多个兼岗。
2. 系统校验人员有效、岗位有效、同人同岗不可重复，并保证主岗唯一。
3. 任职成功后发布 `org.assignment.created`；若切换主岗，则额外发布 `org.assignment.primary-changed`。
4. `identity-context` 与 `role-resource-auth` 依据事件做上下文与权限重算。

### 3. 停用岗位或结束任职

1. 管理员先查看岗位停用或任职移除的影响预览。
2. 岗位停用后，不删除历史任职，但禁止再被新任职引用。
3. 任职结束或到期时，将状态置为 `INACTIVE`，并发布 `org.assignment.removed` 或 `org.assignment.expired`。
4. 下游模块据此刷新身份上下文、角色继承结果和审计记录。

## 一期实现约束

- 一期岗位模型不建立岗位树，`level` 仅表示岗位等级或排序辅助信息，不表示父子层级。
- 任职写入必须显式区分“新增兼岗”“切换主岗”“移除任职”，避免一个通用接口覆盖多种语义。
- 岗位角色绑定只读不写；岗位来源查看可以展示继承结果，但配置入口归属 `role-resource-auth`。
- 人员离职或停用后，其任职关系保留历史记录，但不得继续作为当前身份使用。
- 不支持跨租户任职，也不支持把停用岗位设置为主岗或兼岗。
