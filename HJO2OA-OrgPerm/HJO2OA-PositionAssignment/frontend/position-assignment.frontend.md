# position-assignment 前端契约

## 页面清单

| 页面/组件 | 端别 | 页面职责 | 备注 |
|-----------|------|----------|------|
| 岗位列表页 | 管理端 | 查询岗位、查看归属、状态、任职人数和快捷操作 | 支持组织/部门联动筛选 |
| 岗位新增/编辑弹窗 | 管理端 | 维护岗位基础信息、归属、类别和等级 | 不承载任职编辑 |
| 任职关系维护区 | 管理端 | 在人员详情或独立页内维护主岗、兼岗、任职时间和状态 | 主岗切换需单独确认 |
| 主岗切换确认弹窗 | 管理端 | 展示旧主岗、新主岗和影响摘要 | 高风险操作单独弹窗确认 |
| 角色来源查看面板 | 管理端 | 只读查看岗位继承角色与人员最终角色来源 | 可跳转到角色授权模块配置页 |
| 岗位影响预览弹窗 | 管理端 | 预览岗位停用或归属变更后的影响范围 | 需要展示受影响任职数和上下文刷新范围 |

## 页面职责

| 页面/组件 | 关键职责 | 依赖接口 |
|-----------|----------|----------|
| 岗位列表页 | 展示岗位列表、过滤条件、状态操作和详情入口 | `GET /positions`、`GET /positions/{id}` |
| 岗位新增/编辑弹窗 | 创建或编辑岗位基础字段 | `POST /positions`、`PUT /positions/{id}` |
| 任职关系维护区 | 展示人员当前主岗和兼岗，新增或移除任职 | `GET /persons/{personId}/assignments`、`POST /persons/{personId}/assignments`、`DELETE /persons/{personId}/assignments/{assignmentId}` |
| 主岗切换确认弹窗 | 在切换前展示新旧主岗及影响提示 | `POST /persons/{personId}/assignments/primary` |
| 角色来源查看面板 | 只读查看角色来源和继承链 | `GET /persons/{personId}/effective-role-sources` |
| 岗位影响预览弹窗 | 在停用或调整归属前先看影响摘要 | `POST /positions/{positionId}/impact-preview` |

## 关键交互

- 岗位列表必须支持按组织、部门、状态、类别和关键字组合筛选，并保留最后一次筛选状态。
- 任职维护区新增兼岗时，应实时校验是否与现有任职重复，并明确主岗与兼岗的视觉区分。
- 切换主岗必须先选定目标任职，再展示影响摘要，确认后刷新人员身份概览。
- 角色来源查看面板只负责解释“角色从哪里来”，不在本页直接修改岗位绑定角色。
- 岗位停用和归属调整前，都应先展示影响预览，避免管理员在未知影响下直接提交。

## 权限与状态约束

- 页面权限至少区分 `position.read`、`position.write`、`position.disable`、`assignment.read`、`assignment.write`、`assignment.primary-switch`。
- 停用岗位在列表中仍可查看详情和历史任职，但新增任职、切换主岗到该岗位等操作必须禁用。
- 已离职或停用人员的任职区默认只读，页面应明确展示“历史任职”标签。
- 角色来源面板不授予写权限；没有角色配置权限的用户只能查看来源摘要。

## 身份切换、移动端与管理端差异

- 本模块是一套管理端能力，不提供门户端身份切换入口。
- 页面会展示“当前默认主岗”事实，但不会修改运行时会话身份，上下文切换仍由 `identity-context` 负责。
- 一期不提供移动端岗位与任职维护页面；移动端仅消费最终身份上下文结果。
