# org-sync-audit 前端契约

## 页面清单

| 页面/组件 | 端别 | 页面职责 | 备注 |
|-----------|------|----------|------|
| 同步源配置页 | 管理端 | 配置外部源、连接信息、同步范围和启停状态 | 面向实施或运维 |
| 同步任务列表页 | 管理端 | 查看任务状态、结果摘要、失败重试入口 | 支持全量/增量筛选 |
| 同步任务详情页 | 管理端 | 展示处理明细、错误摘要和关联差异 | 便于排障 |
| 差异对比页 | 管理端 | 对比源端值与本地值，执行处理动作 | 支持人工修正入口 |
| 审计日志查询页 | 管理端 | 按类型、实体、操作者、时间检索变更日志 | 统一时间线入口 |
| 审计详情页/时间线 | 管理端 | 展示前后快照、关联任务号和原始事件摘要 | 支持回溯排查 |

## 页面职责

| 页面/组件 | 关键职责 | 依赖接口 |
|-----------|----------|----------|
| 同步源配置页 | 查询、创建、编辑、启停用同步源 | `GET /sync-sources`、`POST /sync-sources`、`PUT /sync-sources/{sourceId}`、`POST /sync-sources/{sourceId}/status` |
| 同步任务列表页 | 展示任务状态、结果摘要和重试入口 | `GET /sync-tasks`、`POST /sync-tasks/run`、`POST /sync-tasks/{taskId}/retry` |
| 同步任务详情页 | 查看任务处理明细、失败原因和关联差异 | `GET /sync-tasks/{taskId}` |
| 差异对比页 | 展示差异详情并执行接受源端、保留本地或人工修正 | `GET /sync-diffs`、`GET /sync-diffs/{diffId}`、`POST /sync-diffs/{diffId}/resolve` |
| 审计日志查询页 | 检索组织、授权、账号等审计日志 | `GET /audit-logs` |
| 审计详情页/时间线 | 展示快照、操作人、关联任务和原始事件 | `GET /audit-logs/{auditLogId}` |

## 关键交互

- 同步源配置页应支持“保存前连通性校验”，避免无效配置直接进入任务调度。
- 任务列表页需要突出显示运行中、失败、部分失败和可重试状态，并支持按源和时间过滤。
- 差异对比页必须把源端值、本地值、推荐处理动作和处理结果并排展示，避免只给文本提示。
- 人工修正动作必须明确提示“最终会调用真相源模块接口落库”，不能让用户误以为只改差异记录。
- 审计详情页应提供关联任务号和原始事件摘要，支持从审计反查同步任务。

## 权限与状态约束

- 页面权限至少区分 `sync-source.read`、`sync-source.write`、`sync-task.run`、`sync-diff.resolve`、`audit-log.read`。
- 没有 `sync-task.run` 权限的用户只能查看任务，不可手工触发或重试。
- 运行中的任务不允许重复点击“再次执行”，按钮应禁用并展示原因。
- 差异已解决后页面应锁定编辑能力，只保留查看处理结果。

## 身份切换、移动端与管理端差异

- 本模块只提供管理端治理能力，不提供门户端和移动端维护入口。
- 身份切换不会改变页面结构，但会影响当前管理员可见的同步源、任务和审计范围。
- 门户和移动端仅消费同步后的组织与授权结果，不直接展示同步治理和审计页面。
