# 1.7 组织同步与审计总览

## 模块定位

`org-sync-audit` 是 `01-组织与权限` 的同步与治理层，负责外部目录源接入、同步任务编排、差异处理、异常补偿和变更审计。它不拥有组织、岗位、人员、角色等主数据真相源，只负责通过各自模块的公开接口驱动落库并完整记录治理轨迹。

## 核心职责

- 管理 LDAP、AD、企业微信、钉钉、HR 等外部组织源配置。
- 执行全量、增量、失败重试等同步任务，并输出任务结果与差异清单。
- 对同步冲突提供差异对账、人工修正和补偿重试入口。
- 汇总组织、任职、授权、账号等变更事实，形成统一审计日志和时间线。
- 发布同步任务和审计事件，供监控、通知和运维使用。

## 上下游边界

| 类型 | 对象 | 边界说明 |
|------|------|----------|
| 上游输入 | 外部目录/HR 源 | 提供组织、部门、岗位、人员、角色关系等原始数据。 |
| 下游写入 | `org-structure`、`position-assignment`、`person-account`、`role-resource-auth` | 必须通过各模块公开写接口落库，不允许绕过所有权直接改表。 |
| 下游消费 | 运维/实施/审计人员 | 查询同步任务、差异、失败原因和变更时间线。 |
| 消费输入 | 全部子模块领域事件 | 消费组织、任职、人员、账号、角色、数据权限事件生成审计和对账。 |
| 排除范围 | 主数据真相源 | 本模块不拥有组织、岗位、人员、角色等业务主数据。 |
| 排除范围 | 静默覆盖 | 同步冲突不得直接覆盖本地关键数据，必须进入差异治理或人工确认。 |

## 核心能力表

| 能力域 | 能力说明 | 核心规则 | 主要输出 |
|--------|----------|----------|----------|
| 同步源管理 | 维护外部源连接、认证、同步范围和启停状态 | 源配置变更需审计；敏感凭据加密存储 | 同步源清单、连接状态 |
| 同步任务编排 | 触发全量、增量、重试和定时任务 | 同一源同一时间仅允许一个活跃任务；失败可重试 | 任务列表、任务详情、任务事件 |
| 差异与冲突治理 | 对比源数据与本地数据，提供人工修正 | 冲突不得静默覆盖；人工修正必须走目标模块接口 | 差异清单、修正结果、补偿记录 |
| 审计留痕 | 记录组织、授权、账号等变更轨迹 | 审计日志不可篡改；需支持按实体、时间、操作者检索 | 审计时间线、详情快照 |
| 异常补偿 | 失败任务重试、单条补偿、重放审计 | 补偿动作必须可追踪、可回放 | 补偿记录、重试结果 |

## 典型业务流

### 1. 配置外部组织源并执行首次全量同步

1. 管理员配置外部源连接信息、认证信息、同步范围和调度方式。
2. 系统校验连接可用后，创建全量同步任务。
3. 任务逐步调用各主数据模块接口落库，并记录成功、失败和跳过明细。
4. 完成后发布 `org.sync.completed` 或 `org.sync.failed`。

### 2. 处理同步差异与人工修正

1. 任务执行后产生差异记录，例如编码冲突、归属不一致、无效上级等。
2. 管理员在差异页查看源端值、本地值和推荐处理建议。
3. 选择“接受源端”“保留本地”或“人工修正”后，由本模块调用对应真相源模块接口执行。
4. 修正动作与结果都进入审计时间线。

### 3. 查询授权或账号变更审计

1. 审计人员按实体、类型、时间范围和操作人筛选日志。
2. 系统返回组织变更、授权变更、账号变更等时间线，并可查看前后快照。
3. 如需排查异常，可从审计记录反查对应同步任务和原始事件。

## 一期实现约束

- 一期先完成同步任务框架、差异治理和审计链路，具体外部源适配器按项目落地。
- 同步任务只能调用各真相源模块的公开接口，不允许直接写业务表。
- 冲突处理必须显式选择处理策略，不允许后台静默覆盖本地关键主数据。
- 审计日志必须保留前后快照、操作者、触发来源和关联任务号。
- 一期不做跨租户同步编排，也不做跨项目模板分发。
