# 1.5 数据权限管理总览

## 模块定位

`data-permission` 是 `01-组织与权限` 的 ABAC/策略化数据裁剪层，持有 `DataPermission` 与 `FieldPermission` 真相源，负责行级和字段级权限策略、权限预览以及统一决策输出。它不负责菜单、按钮、接口等资源访问控制，也不维护运行时身份事实。

## 核心职责

- 维护组织、部门、岗位、人员、角色五类授权主体的行级数据权限策略。
- 维护按业务对象、使用场景、字段、动作建模的字段级数据权限策略。
- 基于 `identity-context` 输出的当前身份，计算行范围条件和字段动作矩阵。
- 提供权限预览和统一决策接口，供业务模块接入。
- 维护常用权限模板与复用能力，提升实施与配置效率。

## 上下游边界

| 类型 | 对象 | 边界说明 |
|------|------|----------|
| 上游输入 | `identity-context` | 决策必须以当前身份上下文、组织/部门/岗位/人员/角色集合为准。 |
| 上游输入 | `role-resource-auth` | 角色是数据权限授权主体之一，但资源权限结果不在本模块计算。 |
| 上游输入 | 业务模块 | 业务模块必须提供 `businessObject`、`usageScenario` 和待判定字段/数据对象。 |
| 下游输出 | 业务模块 | 输出行范围条件、字段动作矩阵和解释信息。 |
| 协同模块 | `org-sync-audit` | 数据权限变更需被审计和回放。 |
| 排除范围 | 资源权限 | 菜单、按钮、接口、页面、资源动作的可访问性归属 `role-resource-auth`。 |
| 排除范围 | 业务数据读模型 | 本模块输出裁剪规则，不承担业务对象数据查询和聚合。 |

## 核心能力表

| 能力域 | 能力说明 | 核心规则 | 主要输出 |
|--------|----------|----------|----------|
| 行级权限策略 | 为业务对象配置数据范围 | 一期聚焦 `ALL`、`ORG_AND_CHILDREN`、`DEPT_AND_CHILDREN`、`SELF`、`CONDITION`；`CUSTOM` 预留扩展 | 行范围策略、条件表达式、变更事件 |
| 字段级权限策略 | 为业务对象场景配置字段动作 | 一期覆盖 `VISIBLE`、`EDITABLE`、`EXPORTABLE`、`DESENSITIZED`、`HIDDEN` | 字段动作矩阵、变更事件 |
| 冲突处理 | 统一解释多主体、多策略冲突 | `DENY` 优先于 `ALLOW`；更具体主体优先于更宽泛主体；同级按 `priority` 排序 | 最终裁剪结果、解释链 |
| 权限预览 | 指定人员或当前身份预览最终权限 | 预览只读，不产生新权限事实 | 行范围预览、字段矩阵预览、解释明细 |
| 模板复用 | 复用常见权限组合 | 模板是配置快照，应用后复制成实体策略 | 模板清单、应用结果、审计记录 |

## 典型业务流

### 1. 为角色配置行级数据权限

1. 管理员选择业务对象和授权主体角色。
2. 配置数据范围为全量、本组织、本部门、本人或条件表达式。
3. 保存后发布 `org.data-permission.row-changed`，业务模块后续判定时按最新策略执行。

### 2. 为业务表单配置字段级权限

1. 管理员按业务对象和使用场景选择字段集合。
2. 为每个字段配置查看、编辑、导出、隐藏、脱敏等动作。
3. 保存后发布 `org.data-permission.field-changed`，下游表单或导出组件据此裁剪。

### 3. 预览指定人员最终数据权限

1. 管理员在预览页选择人员、业务对象和使用场景。
2. 系统先获取该人员当前身份上下文和角色集合，再合并行级与字段级策略。
3. 页面展示最终行范围、字段矩阵和冲突解释链，供配置校验和排错。

## 一期实现约束

- 一期混合权限模型固定为“资源权限由 RBAC 负责，数据权限由本模块负责”，不得混合配置。
- `CUSTOM` 范围类型保留给后续内置策略扩展，一期前端统一落到显式 `CONDITION` 表达式。
- 统一决策输出的是裁剪规则与解释信息，不负责帮业务模块查出最终数据。
- 权限预览是只读能力，不得在预览流程中偷偷生成配置或缓存副本。
- 所有显式拒绝、脱敏规则和高优先级覆盖都必须保留可审计解释链。
