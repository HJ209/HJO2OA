# 1.4 角色与资源授权管理总览

## 模块定位

`role-resource-auth` 是 `01-组织与权限` 的 RBAC 骨架层，持有 `Role`、`PositionRole`、`PersonRole` 和 `ResourcePermission` 真相源。它负责角色定义、岗位继承关系、例外直授和资源权限配置，但不负责行级或字段级数据权限，也不持有运行时身份上下文。

## 核心职责

- 维护角色生命周期、类别、作用域、状态和说明。
- 维护岗位-角色绑定关系，保证“岗位继承优先于逐人直授”。
- 维护人员直授角色作为例外机制，并记录原因、有效期和撤销轨迹。
- 维护菜单、按钮、接口、页面、资源动作等资源权限的角色配置。
- 提供角色模板和授权复用能力，但模板应用后仍以角色和资源权限真相源为准。

## 上下游边界

| 类型 | 对象 | 边界说明 |
|------|------|----------|
| 上游输入 | `position-assignment` | 岗位-角色绑定必须引用有效岗位；任职变更驱动最终权限继承重算。 |
| 上游输入 | `person-account` | 人员直授角色必须引用有效人员；人员停用/离职只影响生效性，不抹掉历史授权轨迹。 |
| 下游输出 | `identity-context` | 输出当前身份应继承的角色集合，用于上下文计算。 |
| 下游输出 | 业务模块 | 输出资源权限判定所需角色和资源动作结果。 |
| 协同模块 | `data-permission` | 本模块只负责资源权限；数据裁剪交由 `data-permission`。 |
| 协同模块 | `org-sync-audit` | 同步治理和审计依赖本模块发布的角色与授权事件。 |
| 排除范围 | 行级/字段级数据权限 | 不在本模块维护数据访问范围或字段矩阵。 |
| 排除范围 | 身份切换会话态 | 不持久化“当前角色集合”，最终角色集由 `identity-context` 运行时计算。 |

## 核心能力表

| 能力域 | 能力说明 | 核心规则 | 主要输出 |
|--------|----------|----------|----------|
| 角色生命周期管理 | 角色创建、编辑、停用、分类和作用域管理 | 角色编码租户内唯一；作用域影响可分配范围；停用后权限立即失效 | 角色列表、角色详情、角色事件 |
| 岗位继承授权 | 维护岗位-角色绑定 | 角色优先通过岗位继承；绑定岗位必须有效；解绑后需触发权限重算 | `org.role.position-bound/unbound`、岗位继承结果 |
| 人员直授例外 | 维护人员-角色直授 | 直授必须填写原因，可选有效期；仅作为岗位继承的补充 | `org.role.person-granted/revoked`、例外授权审计 |
| 资源权限配置 | 配置角色对菜单、按钮、接口、页面、资源动作的访问能力 | 只回答资源能否访问，不回答数据范围；显式拒绝优先 | 资源权限集合、判定结果、变更事件 |
| 模板复用 | 角色模板定义、导入导出和应用 | 模板只复制配置，不建立运行时依赖 | 模板快照、应用结果、审计记录 |

## 典型业务流

### 1. 创建角色并配置资源权限

1. 管理员创建角色，定义编码、名称、类别、作用域和描述。
2. 为角色配置资源权限树，覆盖菜单、按钮、接口、页面和资源动作。
3. 提交后写入 `Role` 与 `ResourcePermission`，并发布 `org.role.created`、`org.resource-permission.changed`。

### 2. 通过岗位继承角色

1. 管理员把角色绑定到岗位，而不是直接给个人。
2. 系统校验岗位有效、角色作用域合法、绑定关系未重复。
3. 写入成功后发布 `org.role.position-bound`，`identity-context` 和权限缓存据此重算。

### 3. 例外直授与撤销

1. 当岗位继承不能覆盖特殊场景时，管理员可对人员直授角色。
2. 系统要求填写直授原因，可选失效时间，并记录审计轨迹。
3. 直授到期或被撤销时发布 `org.role.person-revoked`，最终权限集合即时更新。

## 一期实现约束

- 一期资源权限只覆盖菜单、按钮、接口、页面和资源动作，不扩展 ReBAC/FGA 关系权限。
- 角色继承顺序固定为“岗位继承优先，再叠加人员直授”，人员直授不得替代岗位体系。
- 资源权限配置允许显式拒绝，但冲突判定由统一权限决策链执行，不能在前端私自解释。
- 角色模板只做复制与复用，不支持跨租户共享或跨项目分发。
- 停用角色必须保留历史授权记录和审计轨迹，不物理删除绑定关系。
