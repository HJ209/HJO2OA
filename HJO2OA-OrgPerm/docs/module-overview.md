# 01-组织与权限-模块功能

## 模块定位

该模块是整个平台的基础底座，负责承载组织结构、人员档案、账号身份、角色岗位、权限分配、数据授权和身份切换能力。它既决定系统是否能支撑复杂企业组织，也直接影响流程、内容、门户、业务应用等所有上层模块的访问控制与数据隔离。

## 业务目标

- 支持组织、部门、岗位、人员、角色五个核心维度
- 支持机构树形层级，机构可以归属上级机构
- 支持部门归属机构并形成部门层级，岗位和人员都要同时具备机构归属与部门归属
- 支持人员岗位区分主岗和兼岗，主岗只能有一个且必选，兼岗可以配置多个
- 支持角色与岗位关联，人员优先通过岗位继承角色，并允许少量直接角色授权作为补充
- 支持经典 RBAC 的菜单、按钮、接口和资源动作权限管理
- 支持行级数据权限以组织、部门、岗位、人员、角色作为授权主体
- 支持字段级数据权限按业务对象、使用场景、字段、授权主体、权限动作建模
- 支持管理员、流程管理员、内容管理员、应用管理员等平台级角色管理
- 支持与 LDAP、AD、企业微信、钉钉、HR 系统的组织同步

## 边界约束

- 租户是系统隔离边界，不参与组织层级、岗位层级和授权主体计算。
- 组织、部门、岗位、人员、角色属于业务管理与授权结构，统一在本模块建模，不得与租户层级混用。
- 所有业务模块在进行资源权限、行级权限和字段级权限判定时，都必须以本模块输出的身份上下文为准。

## 推荐权限设计模式

- 资源权限层推荐采用 RBAC，负责菜单、按钮、接口、页面和资源动作权限
- 数据权限层推荐采用 ABAC 加策略化授权，负责行级、列级和条件化数据访问控制
- 共享协作类对象如知识文档、共享文件夹、项目空间，后续可按需引入 ReBAC 或 Fine-Grained Authorization 思路
- 对于本项目的第一阶段，不建议直接以纯 ABAC 或纯 ReBAC 作为全系统基础模型，而应采用 RBAC 负责稳定授权骨架，ABAC 负责细粒度数据裁剪的混合方案

## 子模块划分

### 1.1 组织与部门管理

负责机构树和部门树的维护，是整个组织模型的骨架。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 机构树管理 | 组织 CRUD、树状归属、层级排序、启停用、排序号维护 | 组织树浏览页、组织新增/编辑弹窗、拖拽排序、批量启停用 |
| 部门管理 | 部门 CRUD、所属机构绑定、部门层级、负责人设置 | 部门树浏览页、部门新增/编辑弹窗、部门人员一览 |
| 组织层级调整 | 机构归属变更、部门归属变更、层级重算 | 拖拽调整层级、变更影响预览、确认弹窗 |
| 组织主数据导入 | 批量导入组织/部门、模板下载、校验反馈 | 批量导入弹窗、校验结果展示、错误行定位 |
| 组织状态管理 | 组织/部门启停用、停用影响范围计算 | 状态切换、影响提示、批量操作 |

### 1.2 岗位与任职管理

负责岗位定义和人员任职关系，是角色继承和身份上下文的基础。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 岗位管理 | 岗位 CRUD、所属机构与部门绑定、岗位层级、岗位编码 | 岗位列表页、岗位新增/编辑弹窗、岗位归属选择 |
| 主岗兼岗管理 | 任职关系 CRUD、主岗唯一约束、兼岗多选、默认身份计算 | 人员任职编辑区、主岗必选提示、兼岗增删、岗位来源查看 |
| 岗位角色继承 | 岗位与角色关联、人员通过岗位继承角色、继承链计算 | 岗位关联角色页、角色继承链可视化、继承结果查看 |
| 岗位变更影响 | 岗位变更后角色重算、任职关系校验、权限刷新 | 岗位变更影响预览、确认弹窗 |

### 1.3 人员与账号管理

负责人员档案和用户账号，两者适度分离。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 人员档案管理 | 人员 CRUD、所属机构与部门绑定、人员状态、基本信息 | 人员列表页、人员详情页、人员新增/编辑弹窗 |
| 账号管理 | 账号 CRUD、密码策略、锁定策略、启停用、外部认证绑定 | 账号列表页、账号详情页、密码重置、锁定/解锁操作 |
| 人员与账号关联 | 人员与账号绑定、一人多号约束、主账号设置 | 人员详情中的账号区、账号绑定/解绑操作 |
| 人员批量操作 | 批量导入、批量调岗、批量启停用 | 批量导入弹窗、批量操作确认、结果反馈 |

### 1.4 角色与资源授权管理

负责角色定义和资源权限分配，是 RBAC 的核心。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 角色管理 | 角色 CRUD、角色分类、角色层级、启停用 | 角色列表页、角色新增/编辑弹窗、角色分类筛选 |
| 角色与岗位关联 | 角色-岗位关联 CRUD、关联后继承计算 | 岗位关联角色页、角色关联岗位页、双向查看 |
| 资源权限配置 | 菜单权限、按钮权限、接口权限、资源动作权限的 CRUD | 资源授权树形配置页、勾选式授权、批量授权 |
| 人员直授角色 | 人员-角色直接关联、例外授权标记、有效期控制 | 人员直授角色页、例外授权提示、有效期设置 |
| 角色模板 | 角色模板定义、模板复用、模板导入导出 | 角色模板管理页、模板应用弹窗 |

### 1.5 数据权限管理

负责行级和字段级数据权限，是 ABAC 的核心。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 行级数据权限 | 授权主体（组织/部门/岗位/人员/角色）、策略（全量/本组织/本部门/本岗位/本人/自定义）、条件表达式 | 行级权限配置页、策略选择、条件编辑器、预览结果 |
| 字段级数据权限 | 业务对象+场景+字段+授权主体+动作建模、动作（查看/编辑/导出/脱敏/隐藏） | 字段级权限配置页、字段选择器、动作矩阵、场景切换 |
| 数据权限预览 | 指定用户的数据权限计算结果、行范围和字段范围预览 | 权限预览页、用户选择、结果展示 |
| 数据权限模板 | 常用权限组合模板、模板复用 | 权限模板管理页、模板应用弹窗 |

### 1.6 身份切换与会话上下文

负责主岗/兼岗身份切换和上下文传递。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 身份切换 | 主岗/兼岗切换、上下文刷新、切换后权限重算 | 门户端身份切换入口、切换后全局刷新 |
| 会话上下文 | 当前身份、组织、部门、岗位的上下文传递 | 上下文展示区、当前身份标签 |
| 移动端身份切换 | 移动端切换接口、缓存刷新 | 移动端身份切换入口、刷新反馈 |

身份切换后，菜单、待办、消息、数据范围、门户卡片和组织通讯录上下文都必须同步刷新。

### 1.7 组织同步与审计

负责外部组织源同步和变更审计。

| 功能点 | 后端能力 | 前端页面/能力 |
|--------|----------|---------------|
| 同步任务管理 | 同步源配置、定时/增量同步、同步任务调度 | 同步配置页、同步任务列表、任务启停 |
| 同步结果处理 | 差异对比、冲突检测、自动校正、人工修正 | 差异对比页、冲突提示、人工修正入口 |
| 变更审计 | 组织变更日志、授权变更日志、账号变更日志 | 审计日志查询页、变更详情、时间线展示 |

## 优化方向

- 人员、账号、任职、角色四层对象明确分离，减少历史 OA 常见的模型混乱问题
- 主岗与兼岗规则内建到模型中，避免岗位多任职时上下文错乱
- 角色通过岗位关联，降低逐人授权带来的维护成本
- 采用 RBAC 管资源权限、ABAC 管数据权限的混合模式，兼顾稳定性与灵活性
- 数据权限统一抽象，避免每个业务模块各自实现
- 支持组织模板、角色模板和授权模板，提升实施效率
- 支持组织变更留痕和授权回溯，满足审计要求

## 典型业务场景

- 新员工入职后同步创建账号、挂接机构、部门、主岗和角色
- 员工兼任多个岗位时以主岗作为默认身份，并可切换兼岗处理流程和查看数据
- 角色与岗位绑定后，新增岗位任职即可自动继承对应资源权限与数据权限
- 不同岗位对同一业务对象仅能查看自己被授权的字段和数据范围
- 集团总部管理员查看全局数据，分公司管理员仅查看本组织数据
- 外部目录同步异常时人工修正岗位归属、主兼岗关系和角色关系

## 交付边界

该模块不直接承载具体业务审批逻辑，但必须为流程、内容、门户、文件、消息及各类业务应用提供统一、稳定、可审计的身份与授权底座。
