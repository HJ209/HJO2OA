# 01-组织与权限-开发任务

## 目标

完成组织、部门、岗位、人员、角色五维组织模型，以及主岗/兼岗、角色岗位关联、资源权限、行级数据权限、字段级数据权限底座建设，为后续所有业务模块提供统一身份与授权能力。

## 子模块开发任务

### 1.1 org-structure 组织与部门管理

**后端**
- 设计组织、部门核心数据模型
- 明确组织树模型，支持机构归属机构
- 明确部门归属机构与部门层级规则
- 建立租户、业务域或组织隔离规则
- 完成组织/部门增删改查与状态管理
- 发布 `org.organization.*`、`org.department.*` 领域事件

**前端**
- 完成组织树浏览页、组织新增/编辑弹窗、拖拽排序
- 完成部门管理页、批量启停用
- 支持组织筛选、状态筛选和快速搜索

### 1.2 position-assignment 岗位与任职管理

**后端**
- 设计岗位数据模型，明确岗位归属机构与部门的约束规则
- 建立人员任职关系模型，支持一个主岗和多个兼岗
- 建立主岗必选、唯一，兼岗可多选的约束校验机制
- 完成主岗默认身份上下文与兼岗切换上下文的计算逻辑
- 发布 `org.position.*`、`org.assignment.*` 领域事件

**前端**
- 完成岗位列表页、岗位编辑弹窗
- 完成任职关系维护页，支持主岗选择、兼岗维护
- 支持岗位归属查看和角色来源查看

### 1.3 person-account 人员与账号管理

**后端**
- 设计人员、账号核心数据模型
- 明确人员归属机构与部门的约束规则
- 完成用户与账号分层模型，支持人员档案变化不影响主账号
- 完成账号登录、密码策略、锁定策略、重置策略和安全日志能力
- 接入外部认证能力，预留 LDAP、AD、企业微信、钉钉等接入点
- 发布 `org.person.*`、`org.account.*` 领域事件

**前端**
- 完成人员列表页、人员详情页
- 完成账号管理页，支持账号列表、账号详情
- 支持查看一个用户的所有身份、机构、部门、主岗、兼岗

### 1.4 role-resource-auth 角色与资源授权管理

**后端**
- 建立角色与岗位关联模型
- 明确人员通过岗位继承角色的计算规则
- 支持人员直接授权角色作为例外处理机制
- 支持角色停用、继承失效和岗位变更后的权限重算
- 完成经典 RBAC 的资源权限配置能力
- 完成按组织、部门、岗位、人员、角色分配权限的能力
- 完成最终权限计算、缓存和失效刷新策略
- 发布 `org.role.*`、`org.resourcePermission.*` 领域事件

**前端**
- 完成角色管理页、授权管理页面
- 支持角色与岗位关联配置
- 支持资源权限树和授权结果可视化

### 1.5 data-permission 数据权限管理

**后端**
- 建立行级数据权限模型，授权主体支持组织、部门、岗位、人员、角色
- 建立行级数据权限策略，支持全量、本组织、本部门、本岗位、本人和自定义条件范围
- 建立字段级数据权限模型，按业务对象、使用场景、字段、授权主体、权限动作建模
- 建立字段级权限动作体系，至少覆盖查看、编辑、导出、隐藏和脱敏
- 完成统一权限决策接口，支持资源权限、行级权限、字段级权限组合判定
- 输出适用于本项目的混合权限模型设计，明确 RBAC 负责资源权限，ABAC 负责数据权限
- 明确权限策略优先级、显式拒绝、冲突处理和审计落库规则
- 发布 `org.dataPermission.*` 领域事件

**前端**
- 完成行级权限配置页、字段级权限配置页
- 完成权限预览功能
- 支持资源权限、行级数据权限、字段级数据权限配置和授权结果可视化

### 1.6 identity-context 身份切换与会话上下文

**后端**
- 完成身份切换、组织切换、岗位切换的会话上下文机制
- 为流程、内容、消息、业务模块输出统一鉴权接口
- 发布 `org.identity.switched` 领域事件
- 为任职失效、岗位停用、账号锁定等被动变化发布 `org.identity-context.invalidated` 领域事件

**前端**
- 在门户端实现身份切换入口
- 默认展示主岗身份，并允许切换兼岗身份
- 切换后刷新菜单、工作台、待办、消息和通讯录上下文
- 支持移动端身份切换与缓存刷新

### 1.7 org-sync-audit 组织同步与审计

**后端**
- 完成组织同步任务框架
- 支持同步组织、部门、岗位、人员、角色关联关系
- 支持同步后自动校验主岗、兼岗和岗位角色继承关系
- 完成同步结果审计、差异对账和异常补偿能力
- 建立组织与授权变更日志查询能力
- 发布 `org.sync.*`、`org.audit.*` 领域事件

**前端**
- 完成组织同步配置、同步记录、差异查看和失败重试页面
- 提供授权变更记录、岗位角色继承结果和字段权限配置记录查看页面

## 联调与协作任务

- 统一前后端权限数据结构与返回协议
- 明确组织、部门、岗位、人员、角色五维主数据的字段口径
- 明确主岗、兼岗、岗位角色继承和例外授权的计算顺序
- 明确身份切换后的上下文刷新机制
- 明确菜单权限、按钮权限、行级权限、字段级权限的职责边界
- 输出权限异常提示规范与越权反馈规范

## 验收标准

- 可以完整维护组织、部门、岗位、人员和角色关系
- 一个用户必须具备唯一主岗，并可安全切换多个兼岗身份
- 角色与岗位关联后，人员可正确继承资源权限和数据权限
- 所有核心业务接口都可接入统一的资源权限、行级权限和字段级权限校验
- 权限变更具备可追溯审计记录
- 外部组织同步可执行、可回滚、可排查
