# 1.1 组织与部门管理总览

## 模块定位

`org-structure` 是 `01-组织与权限` 的组织主数据真相源，负责维护 `Organization` 与 `Department` 两个聚合根，输出稳定的组织树、部门树和层级变更事实。它只持有组织与部门主数据，不承担岗位、人员、角色、数据权限或运行时身份上下文的所有权。

## 核心职责

- 维护组织与部门的新增、编辑、启停用、排序和层级调整。
- 维护 `path`、`level`、`sortOrder` 等树形结构字段，确保无环、可追溯、可快速查询后代节点。
- 对岗位、人员、角色授权等下游模块输出稳定的组织与部门引用事实。
- 提供批量导入、预校验、确认落库和影响范围预览能力。
- 发布组织与部门的领域事件，供审计、同步治理和下游投影消费。

## 上下游边界

| 类型 | 对象 | 边界说明 |
|------|------|----------|
| 上游输入 | `org-sync-audit` | 同步任务只能通过本模块公开写接口或应用服务落库，不直接改写组织表与部门表。 |
| 上游输入 | 管理端组织维护页面 | 页面可发起树调整、状态切换、导入确认，但必须先经过预校验与影响预览。 |
| 下游输出 | `position-assignment` | 岗位必须绑定有效组织，可选绑定有效部门；层级变更会影响岗位归属合法性校验。 |
| 下游输出 | `person-account` | 人员主属组织与主属部门必须引用本模块有效节点。 |
| 下游输出 | `role-resource-auth`、`data-permission` | 仅消费组织/部门标识作为授权主体，不维护树结构。 |
| 下游输出 | `identity-context` | 只提供静态组织与部门事实，不参与运行时上下文持久化。 |
| 排除范围 | 租户模型 | `tenantId` 只用于隔离，不参与组织层级和业务语义计算。 |
| 排除范围 | 岗位/人员生命周期 | 本模块不负责岗位、任职、人员、账号和角色状态联动写入，只发布事实事件。 |

## 核心能力表

| 能力域 | 能力说明 | 核心规则 | 主要输出 |
|--------|----------|----------|----------|
| 组织生命周期管理 | 维护组织创建、编辑、启停用、排序 | 组织编码租户内唯一；禁止形成父子环；停用后禁止新增下级与新绑定关系 | 组织树、组织详情、组织变更事件 |
| 部门生命周期管理 | 维护部门创建、编辑、负责人、启停用 | 部门必须归属组织；部门负责人必须属于同组织；禁止跨组织挂接上级部门 | 部门树、部门详情、部门变更事件 |
| 层级调整 | 支持组织树、部门树移动与排序重算 | 调整前必须做影响预览；调整后同步更新 `path`、`level`、后代路径 | 层级变更结果、影响预览、层级变更事件 |
| 主数据导入 | 模板下载、预校验、确认导入、结果回执 | 一期先支持“预校验 + 确认落库”，不支持无校验覆盖导入 | 导入任务结果、错误行清单、导入审计 |
| 状态影响控制 | 组织/部门启停用与影响提示 | 停用不物理删除历史引用；新建与调整动作必须校验上级节点状态 | 状态切换结果、影响摘要、审计记录 |

## 典型业务流

### 1. 新增组织与顶级部门

1. 管理员先创建组织，写入编码、名称、类型、上级组织和排序号。
2. 系统校验编码唯一、上级组织状态和租户隔离条件，生成组织 `path` 与 `level`。
3. 管理员在该组织下新增顶级部门或下级部门，系统校验所属组织一致性与部门负责人合法性。
4. 写入成功后分别发布 `org.organization.created` 与 `org.department.created`。

### 2. 组织或部门层级调整

1. 管理员在树上拖拽节点或通过“调整归属”入口发起移动。
2. 系统先返回影响预览，至少包含旧路径、新路径、后代节点数量和受影响的岗位/人员绑定数量摘要。
3. 管理员确认后执行移动，系统同步重算当前节点和所有后代节点的 `path`、`level`、`sortOrder`。
4. 调整完成后发布 `org.organization.hierarchy-changed` 或 `org.department.hierarchy-changed`，供审计和缓存刷新使用。

### 3. 组织主数据导入与停用治理

1. 管理员下载模板并上传组织/部门导入文件，系统先返回逐行校验结果。
2. 仅当校验通过时允许确认落库；失败行需明确重复编码、无效上级、跨组织挂接等原因。
3. 对组织或部门执行停用时，系统返回影响范围摘要，但不删除历史引用。
4. 停用成功后发布对应 `disabled` 事件，由下游模块自行决定缓存失效和读模型刷新。

## 一期实现约束

- 组织树与部门树必须分开建模，禁止把部门直接混入组织树。
- 一期仅支持逻辑停用，不支持物理删除已被引用的组织与部门。
- 层级调整必须走显式接口，不允许通过通用编辑接口偷偷修改 `parentId`。
- 导入流程必须至少分为“预校验”和“确认落库”两步，不支持直接覆盖。
- 不支持跨租户组织共享，也不支持跨组织挂接部门。
- 本模块不承担岗位、人员、角色的联动写入，只提供影响摘要和事件输出。
