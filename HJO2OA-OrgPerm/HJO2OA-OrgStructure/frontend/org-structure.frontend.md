# org-structure 前端契约

## 页面清单

| 页面/组件 | 端别 | 页面职责 | 备注 |
|-----------|------|----------|------|
| 组织树浏览页 | 管理端 | 展示组织树、组织详情摘要、状态和快捷操作 | 主入口，支持左树右详情布局 |
| 组织新增/编辑弹窗 | 管理端 | 维护组织基础信息、上级组织、排序号和状态 | 层级调整不在该弹窗内完成 |
| 部门树与列表页 | 管理端 | 按组织查看部门树、部门表格、负责人和状态 | 支持树表联动 |
| 部门新增/编辑弹窗 | 管理端 | 维护部门基本信息、负责人、所属组织和上级部门 | 需显示组织归属路径 |
| 层级调整预览弹窗 | 管理端 | 展示拖拽或移动后的影响范围和确认动作 | 必须先预览后确认 |
| 导入预校验页/弹窗 | 管理端 | 上传模板、查看逐行校验结果、确认导入 | 支持错误行定位与导出 |

## 页面职责

| 页面/组件 | 关键职责 | 依赖接口 |
|-----------|----------|----------|
| 组织树浏览页 | 提供组织树筛选、节点展开、状态过滤、组织详情摘要和操作入口 | `GET /organizations/tree`、`GET /organizations/{id}` |
| 组织新增/编辑弹窗 | 仅维护基础字段，不承担层级移动 | `POST /organizations`、`PUT /organizations/{id}` |
| 部门树与列表页 | 支持按组织切换部门树和分页表格，查看负责人和绑定摘要 | `GET /departments/tree`、`GET /departments`、`GET /departments/{id}` |
| 部门新增/编辑弹窗 | 对部门做新增、编辑、负责人选择和状态切换 | `POST /departments`、`PUT /departments/{id}`、`POST /departments/{id}/status` |
| 层级调整预览弹窗 | 提前展示新旧路径、受影响节点数、关联岗位/人员摘要 | `POST /organizations/{id}/move-preview`、`POST /departments/{id}/move-preview` |
| 导入预校验页/弹窗 | 承接文件上传、错误清单查看和确认导入 | `POST /organization-import/preview`、`POST /organization-import/{jobId}/confirm` |

## 关键交互

- 组织树与部门树都应支持 `keyword + status + 根节点裁剪` 的组合筛选。
- 拖拽调整层级时，前端先调用预览接口，再在确认后调用真正的移动接口。
- 组织详情与部门详情需要展示“受影响摘要”，而不是直接加载下游模块全量数据。
- 导入页必须把错误行、错误码、错误原因和修复建议并排展示，避免只提示“导入失败”。
- 状态切换前必须展示影响提示，至少说明对子级新增绑定和下游校验的影响。

## 权限与状态约束

- 管理端页面至少区分 `read`、`write`、`move`、`disable`、`import` 五类动作权限。
- 没有 `move` 权限的用户可以查看树结构，但不能拖拽、调整层级或打开移动确认弹窗。
- 节点为 `DISABLED` 时，前端仍允许查看详情和历史路径，但禁止新增子节点和重新挂接。
- 部门负责人选择器只能返回当前组织范围内的人员，跨组织候选项必须在前端禁用并在后端兜底校验。

## 身份切换、移动端与管理端差异

- 本模块是一套管理端能力，不提供门户端和移动端维护入口。
- 身份切换不改变页面结构，但会影响当前管理员可见的组织范围和可操作动作集合。
- 若当前身份仅具备组织级管理员权限，页面应默认限制在其可管理的组织根节点下，不展示超范围节点。
