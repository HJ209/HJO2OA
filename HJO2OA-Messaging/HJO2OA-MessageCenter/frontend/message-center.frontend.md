# message-center 前端契约

## 1. 页面清单

| 页面 | 角色 | 主要内容 |
|------|------|----------|
| 消息列表页 | 用户侧 | 分类 Tab、状态筛选、来源筛选、关键字搜索、未读标识 |
| 消息详情页/抽屉 | 用户侧 | 标题、摘要、正文快照、来源信息、深链跳转、状态标签 |
| 门户消息卡片 | 门户侧 | 未读总数、最近消息、快捷进入列表 |
| 管理撤回入口 | 管理侧 | 仅管理员可见，用于撤回或补充失效原因 |

## 2. 页面职责

### 2.1 消息列表页

- 展示当前用户可见的消息集合。
- 支持按 `category / inboxStatus / sourceModule / priority / createdAt` 筛选。
- 提供单条已读、批量已读、归档快捷动作。

### 2.2 消息详情页

- 展示消息快照，不回查上游业务主数据作为主展示源。
- 对 `REVOKED`、`EXPIRED` 消息显示占位说明和原因。
- 深链跳转前校验当前登录态与身份上下文。

### 2.3 门户消息卡片

- 只展示未读汇总和最近消息摘要。
- 卡片点击统一进入消息列表页，不直接承担复杂筛选逻辑。

## 3. 关键交互

| 交互 | 行为约束 |
|------|----------|
| 打开详情 | 若消息为 `UNREAD`，允许在详情加载成功后自动回写已读 |
| 批量已读 | 仅处理当前页已选消息，失败项单独提示 |
| 归档 | 归档后从默认“收件箱”视图移除，但可在“已归档”视图查回 |
| 深链跳转 | 深链失败时保留消息详情页，不丢失当前列表上下文 |
| 身份切换 | 切换后必须刷新列表、未读数和筛选结果，不能沿用旧身份缓存 |

## 4. 站内与外部渠道差异

- 本模块页面只以站内消息为主视图，不提供外部渠道模板、端点或配置编辑能力。
- 如需展示外部送达状态，只能作为只读摘要，不暴露重试、补偿或渠道配置入口。
- 外部渠道打开 App/门户后，最终阅读行为仍回归站内消息事实。

## 5. 移动端弱网与登录态约束

- 移动端消息页优先通过 `mobile-support` 聚合接口获取数据，不直接复用 PC 管理接口。
- 弱网时允许展示最近一次缓存的列表与未读数，但状态动作必须在会话有效且服务端确认后才更新最终 UI。
- 登录态失效、身份切换或会话冻结后，前端必须清空本地消息缓存并重新拉取。

## 6. 管理端配置约束

- 管理端仅允许撤回、过期查看和审计追踪，不允许编辑消息正文、模板或渠道。
- 撤回操作必须要求填写原因，并在界面上回显操作者与时间。
- 发送配置、订阅配置、移动接入配置、生态配置全部跳转到对应子模块，不在本模块承载。
