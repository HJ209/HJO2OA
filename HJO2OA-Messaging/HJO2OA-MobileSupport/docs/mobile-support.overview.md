# mobile-support 概览

## 1. 模块定位

`mobile-support` 是 `06-消息移动与生态` 中负责移动端设备接入、移动会话和高频轻量接口的子模块，持有 `DeviceBinding` 与 `MobileSession`。

该模块面向移动端首页、消息、待办入口、阅读预览等高频场景提供轻量聚合能力，并处理设备绑定、登录态续期与弱网容错；它不拥有流程、内容或组织主数据。

## 2. 核心职责

| 职责 | 说明 |
|------|------|
| 设备接入 | 维护 `deviceId`、`pushToken`、平台类型、绑定状态和风险级别 |
| 移动会话 | 维护移动端会话续期、冻结、失效和当前身份快照 |
| 高频聚合接口 | 为首页、消息页、待办入口提供轻量接口，减少移动端直连多个服务 |
| 弱网容错 | 提供缓存兜底、重试提示、请求降级和会话失效处理 |

## 3. 上下游边界

### 3.1 上游输入

- `01-identity-context` 提供人员、账号、任职、岗位真相与身份切换结果。
- `message-center` 提供消息列表、未读数和消息详情。
- `02-todo-center`、`05-业务协同与执行` 提供待办、审批、阅读、预览等业务数据。

### 3.2 下游输出

- 为移动端 App/H5 提供设备绑定、会话续期、首页聚合和消息轻量查询。
- 为 Push 链路提供设备与 token 主数据。

### 3.3 明确不负责

- 不持有业务主数据，不直接承载审批、流程、内容实体的最终事实。
- 不负责模板、发送策略、订阅规则或生态配置。
- 不发布稳定跨模块领域事件，以 API 与缓存刷新为主。

## 4. 核心能力表

| 能力 | 输入 | 输出 | 关键规则 | 一期约束 |
|------|------|------|----------|----------|
| 设备绑定 | 账号、设备指纹、平台、push token | `DeviceBinding` | 同一租户同一 `deviceId` 只允许一个有效绑定 | 支持 `IOS / ANDROID / H5 / PWA` |
| 会话管理 | 已认证身份、设备绑定、身份切换结果 | `MobileSession` | 会话里的任职是快照，不替代 `01` 真相 | 支持续期、冻结、失效 |
| 首页聚合 | 当前人、当前身份、未读与待办摘要 | 轻量首页数据 | 聚合只返回高频摘要，不返回大对象 | 聚焦消息、待办、审批、阅读、预览入口 |
| 弱网容错 | 缓存、最近成功快照、会话状态 | 降级结果与重试提示 | 弱网不伪造成功写操作 | 仅保证核心页可恢复 |

## 5. 典型业务流

### 5.1 首次移动接入

1. 用户完成账号认证。
2. 移动端向本模块登记 `deviceId`、平台类型、指纹和 `pushToken`。
3. 本模块创建或更新 `DeviceBinding`，签发 `MobileSession`。
4. 移动端调用首页聚合接口获取未读数、待办摘要和常用入口。

### 5.2 身份切换与弱网恢复

1. 用户切换当前任职身份。
2. 本模块刷新 `MobileSession.currentAssignmentId/currentPositionId` 快照。
3. 首页、消息和待办摘要重新按新身份拉取。
4. 若处于弱网状态，先展示最近成功缓存，恢复在线后再回写最新结果。

## 6. 一期实现约束

- 一期只覆盖消息、待办、审批、阅读、预览等高频场景，不复制完整 PC 工作台。
- `MobileSession` 中的任职和岗位只作为移动端快速判定快照，不替代 `01` 的实时身份计算。
- 设备丢失、撤销或高风险冻结时，必须立即使 `pushToken` 与移动会话失效。
- 本模块不发布独立领域事件，缓存刷新可消费 `msg.message.*` 和 `org.identity.switched`。
- 不实现复杂离线命令队列，只保证核心页面的读取降级与安全重登。
