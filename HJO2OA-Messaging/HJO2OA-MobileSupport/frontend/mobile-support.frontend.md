# mobile-support 前端契约

## 1. 页面清单

| 页面 | 角色 | 主要内容 |
|------|------|----------|
| 移动登录/激活页 | 用户侧 | 设备激活、登录态校验、安全提示 |
| 移动首页 | 用户侧 | 未读消息、待办摘要、审批入口、阅读/预览快捷入口 |
| 移动消息页 | 用户侧 | 轻量消息列表、筛选、跳转 |
| 设备与会话页 | 用户侧/安全侧 | 当前设备、最近登录、退出或设备撤销提示 |

## 2. 页面职责

### 2.1 移动登录/激活页

- 完成登录态接入后的设备绑定和移动会话建立。
- 展示设备风险、会话失效和重登引导。

### 2.2 移动首页

- 聚焦消息、待办、审批和阅读等高频入口。
- 只展示摘要和快捷操作，不承载复杂管理功能。

### 2.3 移动消息页

- 展示轻量消息列表和未读标记。
- 详情跳转优先走深链或聚合详情页，不在页面内承载复杂正文排版。

### 2.4 设备与会话页

- 展示当前设备状态和最近活跃时间。
- 支持退出登录或触发设备撤销流程。

## 3. 关键交互

| 交互 | 行为约束 |
|------|----------|
| 首次激活 | 必须先绑定设备，再创建移动会话 |
| 会话续期 | 前台无感续期失败时应提示重新登录 |
| 身份切换 | 切换后首页、消息和待办摘要必须一起刷新 |
| Push 打开消息 | 先校验会话，再落到消息详情或对应业务深链 |
| 设备撤销 | 撤销后立即清空本地 token 和缓存 |

## 4. 站内与外部渠道差异

- 移动端主视图仍以站内消息和统一深链为主，不直接暴露外部渠道配置。
- 外部 Push 只负责把用户唤起到 App/H5，真正的阅读和已读事实仍落在站内消息。
- 对外部渠道异常只做“通知可能延迟”提示，不在移动端提供投递运维入口。

## 5. 移动端弱网与登录态约束

- 弱网时首页和消息页允许显示最近一次成功缓存，并标识“数据可能不是最新”。
- 写操作只允许在服务端确认后更新最终 UI；离线状态下不伪造成功。
- 登录态失效、会话过期、风险冻结时必须清空本地缓存并重新认证。
- 本模块页面必须考虑小屏和弱网，不允许把 PC 配置页直接搬到移动端。

## 6. 管理端配置约束

- 移动端不承载模板、路由、订阅、生态等管理配置页面。
- 设备撤销和风险处理若在管理端触发，移动端必须在下次心跳或请求时立即感知并退出。
- 管理配置统一回到对应子模块 PC 页面，移动端只负责消费结果。
