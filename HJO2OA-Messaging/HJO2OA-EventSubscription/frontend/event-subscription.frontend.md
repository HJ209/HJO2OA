# event-subscription 前端契约

## 1. 页面清单

| 页面 | 角色 | 主要内容 |
|------|------|----------|
| 订阅规则页 | 管理端 | 规则列表、事件类型、模板编码、启停状态 |
| 订阅规则详情页 | 管理端 | 目标解析、优先级、去重窗口、启停操作 |
| 用户偏好页 | 用户侧 | 允许渠道、静默时段、升级开关 |
| 执行审计页 | 管理端/审计端 | 事件命中、去重、静默、失败记录 |

## 2. 页面职责

### 2.1 订阅规则页

- 展示一期已支持的事件类型与对应规则。
- 管理员可启停规则，但不能从页面注册新的通用事件源。

### 2.2 用户偏好页

- 用户按消息类别维护自己的静默与渠道允许范围。
- 页面需要明确区分“用户允许”和“系统强制”。

### 2.3 执行审计页

- 查询规则命中结果、去重、静默、失败原因。
- 用于验证提醒未生成是因为规则未命中还是被偏好/静默拦截。

## 3. 关键交互

| 交互 | 行为约束 |
|------|----------|
| 启停规则 | 需要填写原因，停用后只影响后续新事件 |
| 编辑规则 | 只能引用已注册事件类型和已发布模板 |
| 修改偏好 | 提交前校验强制消息不能被完全关闭 |
| 查看审计 | 支持按事件类型、规则编码、结果过滤 |

## 4. 站内与外部渠道差异

- 本模块页面只表达“是否允许某类渠道触达”，不维护具体外部渠道配置。
- 站内 `INBOX` 是消息留存底座，强制消息即使用户关闭外部渠道也必须保留站内提醒。
- 外部渠道是否可用由 `channel-sender` 与 `ecosystem` 决定，本页面不直接判断健康状态。

## 5. 移动端弱网与登录态约束

- 用户偏好页可在移动端提供简化版，仅允许修改本人偏好。
- 管理端规则与审计页面默认仅支持 PC，不承诺移动端完整编辑体验。
- 弱网下偏好页可展示最近一次配置缓存，但保存操作必须在服务端确认后生效。

## 6. 管理端配置约束

- 规则配置不得越过父模块已定义的事件边界，不得创建通用接入点或 Webhook。
- 规则配置必须引用已发布模板，不能引用草稿模板。
- 强制消息策略、优先级升级和去重窗口变更都必须保留审计记录。
