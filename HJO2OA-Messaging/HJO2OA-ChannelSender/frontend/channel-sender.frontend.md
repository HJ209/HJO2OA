# channel-sender 前端契约

## 1. 页面清单

| 页面 | 角色 | 主要内容 |
|------|------|----------|
| 模板管理页 | 管理端 | 模板列表、版本、预览、发布 |
| 渠道端点页 | 管理端 | 端点状态、配额、限流、测试发送 |
| 路由策略页 | 管理端 | 按类别和优先级配置渠道顺序、降级链路 |
| 投递追踪页 | 运维端 | 投递任务列表、尝试记录、错误详情、人工重试 |

## 2. 页面职责

### 2.1 模板管理页

- 维护多渠道模板编码、语言和版本。
- 发布前必须支持预览，不允许直接编辑已发布版本。

### 2.2 渠道端点页

- 查看端点状态、提供方类型、健康状态、限流和配额。
- 发起测试发送，但不暴露明文配置。

### 2.3 路由策略页

- 按消息类别和优先级配置渠道顺序。
- 显式展示 `INBOX` 必达规则和外部渠道降级链路。

### 2.4 投递追踪页

- 展示 `DeliveryTask` 与 `DeliveryAttempt`。
- 提供人工重试入口和失败原因定位。

## 3. 关键交互

| 交互 | 行为约束 |
|------|----------|
| 模板预览 | 必须输入示例变量，渲染失败时阻止发布 |
| 模板发布 | 发布后当前版本只读，后续修改需生成新版本 |
| 端点启停 | 禁用端点前提示影响范围，但不得影响 `INBOX` |
| 测试发送 | 仅对指定测试目标生效，不写入正式投递统计 |
| 人工重试 | 仅对 `FAILED / GAVE_UP` 任务开放，需填写原因 |

## 4. 站内与外部渠道差异

- `INBOX` 作为平台内建渠道，只展示状态，不允许编辑凭证或删除。
- 邮件、短信、企业微信、钉钉、App Push 等外部渠道只显示 `configRef` 与健康状态，具体连接配置在 `ecosystem` 维护。
- 外部渠道投递失败可降级，但不能阻断站内消息入箱。

## 5. 移动端弱网与登录态约束

- 本模块页面以管理端和运维端为主，默认只支持 PC，不承诺移动端完整配置能力。
- 若在平板或移动端访问，最多提供投递结果只读浏览，不开放模板发布、端点变更和人工重试。
- 登录态失效后必须重新获取管理权限，不允许本地缓存继续提交配置更改。

## 6. 管理端配置约束

- 任何模板、端点、策略变更都必须带审计留痕。
- 界面不得保存或回显明文密钥、密码、令牌。
- 配置页只负责消息投递侧治理；订阅偏好、移动接入、生态接入配置需跳转到对应子模块。
