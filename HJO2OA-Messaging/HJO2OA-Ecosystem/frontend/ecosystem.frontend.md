# ecosystem 前端契约

## 1. 页面清单

| 页面 | 角色 | 主要内容 |
|------|------|----------|
| 接入配置页 | 管理端/安全端 | 集成列表、类型、状态、认证方式、回调地址 |
| 接入详情页 | 管理端/安全端 | 配置引用、签名算法、连接测试、启停操作 |
| 健康监控页 | 运维端 | 健康状态、最近检查时间、错误摘要 |
| 回调审计页 | 管理端/审计端 | 回调校验结果、失败原因、摘要查看 |

## 2. 页面职责

### 2.1 接入配置页

- 统一维护企业微信、钉钉、短信、邮件、SSO、LDAP 等接入对象。
- 只展示 `configRef` 和配置摘要，不展示明文凭证。

### 2.2 接入详情页

- 维护签名算法、回调地址、启停状态。
- 提供连接测试入口和最近错误摘要。

### 2.3 健康监控页

- 以治理视角展示接入是否健康、是否降级、是否不可达。
- 用于判断路由链路是否应继续使用某类外部渠道。

### 2.4 回调审计页

- 查看回调是否验签通过、是否重复、失败原因是什么。
- 支持按接入类型、校验结果、时间范围过滤。

## 3. 关键交互

| 交互 | 行为约束 |
|------|----------|
| 新建接入 | 必须选择父模块已定义的接入类型 |
| 启停接入 | 停用前提示影响渠道或登录入口范围 |
| 连接测试 | 执行后即时刷新健康状态和错误摘要 |
| 查看回调详情 | 只显示摘要，不直接暴露完整敏感载荷 |

## 4. 站内与外部渠道差异

- 本模块只管理外部渠道和外部登录接入；站内 `INBOX` 不在此处配置。
- 外部消息渠道的配置在这里维护，但模板、路由和投递任务仍在 `channel-sender` 管理。
- `SSO / LDAP` 与消息渠道共享治理页面，但前端需明确区分“登录接入”和“发送渠道”。

## 5. 移动端弱网与登录态约束

- 本模块页面是管理与安全治理页面，默认只支持 PC，不提供移动端完整编辑能力。
- 若在移动端访问，最多允许查看健康状态和审计摘要，不开放配置编辑和连接测试。
- 进入该模块必须使用高权限登录态；登录态失效后立即关闭配置页并重新认证。

## 6. 管理端配置约束

- 页面不得保存或展示明文密钥、密码、证书内容。
- 接入配置变更、连接测试、启停操作必须产生审计记录。
- 通用开放接口、连接器治理和跨系统数据同步入口不得出现在本模块页面中。
