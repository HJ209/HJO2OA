# 数据开放分层文档

对应架构决策：D17（数据开放分层）、D04（`06` 与 `07` 模块边界）、D05（`04` 与 `07` 数据边界）。

## 1. 文档目的

定义 `07-数据服务与集成` 内部的六层能力边界、职责划分和协作规则，明确各层与 `04-门户与工作台`、`06-消息移动与生态` 的边界。

## 2. 能力分层总览

```
┌─────────────────────────────────────────────┐
│  7.6 治理监控层                               │  版本管理、健康检查、告警、链路追踪
├─────────────────────────────────────────────┤
│  7.5 报表分析层                               │  统计口径、聚合查询、趋势排行
├─────────────────────────────────────────────┤
│  7.4 同步交换层                               │  增量/全量同步、回写、差异补偿
├─────────────────────────────────────────────┤
│  7.3 连接器层                                 │  HTTP、数据库、文件、MQ、SaaS 连接器
├─────────────────────────────────────────────┤
│  7.2 开放接口层                               │  开放 API、凭证、配额、审计、限流
├─────────────────────────────────────────────┤
│  7.1 数据服务层                               │  服务定义、查询、提交、转换映射
└─────────────────────────────────────────────┘
```

## 3. 各层职责与边界

### 3.1 数据服务层（7.1）

**职责**：可管理的数据查询、提交、转换和映射能力。

| 能力 | 说明 |
|------|------|
| 数据服务定义 | 定义数据服务的参数模型、字段映射和权限模型 |
| 数据查询与提交 | 统一查询/提交接口，读写分离，缓存策略 |
| 数据转换与映射 | 字段映射配置，数据格式转换 |

**边界**：
- 数据服务层不直接暴露给第三方，仅作为开放接口层和内部模块的数据源。
- 数据服务层不持有业务主数据，仅提供查询和转换能力。

### 3.2 开放接口层（7.2）

**职责**：标准开放 API 和调用鉴权。

| 能力 | 说明 |
|------|------|
| 开放 API 管理 | 接口注册、接口文档、版本管理 |
| 应用凭证管理 | 凭证 CRUD、调用鉴权、访问配额 |
| 调用审计 | 调用日志、错误追踪、性能监控 |
| 限流与告警 | 限流策略、异常率告警 |

**边界**：
- 开放接口层面向第三方系统，不面向前端用户。
- 开放接口层通过数据服务层获取数据，不直接访问业务模块数据库。
- 开放接口层的鉴权与 `01-组织与权限` 的用户鉴权独立，使用应用凭证（API Key / OAuth2 Client Credentials）。

### 3.3 连接器层（7.3）

**职责**：各类连接器的统一管理。

| 连接器类型 | 说明 |
|-----------|------|
| HTTP 连接器 | 调用外部 HTTP/REST 接口 |
| 数据库连接器 | 连接外部数据库，执行查询/写入 |
| 文件交换连接器 | 文件上传/下载/同步 |
| 消息队列连接器 | 对接外部 MQ（如 Kafka、RocketMQ） |
| SaaS 连接器 | 对接企业微信、钉钉、飞书等 SaaS 平台 |

**边界**：
- 连接器层提供连接能力，不定义业务逻辑。
- 连接器参数配置由管理员维护，运行时由同步交换层调用。
- `06-消息移动与生态` 的消息渠道（企业微信/钉钉消息推送）使用 `06` 自身的渠道发送器，不走 `07` 连接器层。

### 3.4 同步交换层（7.4）

**职责**：跨系统数据同步和交换。

| 能力 | 说明 |
|------|------|
| 同步任务管理 | 组织/字典/主数据/业务单据同步任务 CRUD |
| 同步策略 | 增量/全量同步、失败重试、对账机制 |
| 结果回写与事件推送 | 回写规则、事件推送配置 |
| 同步差异与补偿 | 差异检测、人工补偿 |

**边界**：
- 同步交换层使用连接器层的能力与外部系统交互。
- 同步结果通过事件总线通知业务模块。
- `01-组织与权限` 的 `org-sync-audit` 子模块是同步交换层在组织域的特化实现，使用同步交换层的基础框架。

### 3.5 报表分析层（7.5）

**职责**：平台级报表和统计分析。

| 能力 | 说明 |
|------|------|
| 统计口径定义 | 流程/内容/会议/任务/考勤/消息统计口径 |
| 聚合查询 | 按组织/时间/业务类型/角色维度聚合 |
| 趋势与排行 | 趋势分析、排行分析 |
| 门户图卡嵌入 | 统计结果输出为图卡数据源 |

**边界**：
- 报表分析层消费业务事件更新统计缓存，不发布领域事件。
- 报表分析层的图卡数据源可被 `04-门户与工作台` 的聚合接口消费，但 `04` 的聚合接口不依赖报表分析层（`04` 有自己的轻量聚合逻辑）。
- 报表分析层提供深度分析页面，与 `04` 的首页卡片是不同场景。

### 3.6 治理监控层（7.6）

**职责**：开放接口的治理和监控。

| 能力 | 说明 |
|------|------|
| 接口版本管理 | 版本注册、发布管理、兼容策略 |
| 健康检查 | 接口健康状态、性能指标 |
| 告警与异常 | 告警规则、异常链路追踪 |

**边界**：
- 治理监控层仅管理 `07` 内部的开放接口，不管理业务模块的内部接口。
- 业务模块的健康检查由 `00-平台基础设施` 的监控能力承担。

## 4. 与其他模块的边界

### 4.1 与 `04-门户与工作台` 的边界

| 维度 | 04-门户与工作台 | 07-数据服务与集成 |
|------|----------------|------------------|
| 服务对象 | 前端用户（首页/办公中心） | 第三方系统 / 管理员 |
| 数据范围 | 当前用户身份可见数据 | 按凭证授权的数据范围 |
| 缓存策略 | 用户级缓存，身份切换失效 | 接口级缓存，按凭证隔离 |
| 实现方式 | 聚合读模型 + 轻量缓存 | 数据服务 + 开放 API + 治理 |

### 4.2 与 `06-消息移动与生态` 的边界

| 维度 | 06-消息移动与生态 | 07-数据服务与集成 |
|------|------------------|------------------|
| 消息渠道 | 06 自有渠道发送器 | 07 连接器层不参与用户消息推送 |
| 生态登录 | 06 负责 SSO/扫码登录 | 07 连接器层可对接 SaaS 但不处理登录 |
| 数据交换 | 06 不做跨系统数据同步 | 07 负责所有跨系统数据交换 |

### 4.3 与 `01-组织与权限` 的边界

- `01` 的 `org-sync-audit` 子模块使用 `07` 同步交换层的基础框架。
- `01` 的组织数据通过 `07` 开放接口层对外提供（如 HR 系统查询组织架构）。
- `01` 的权限判定不依赖 `07`，`07` 的开放接口鉴权使用应用凭证而非用户权限。

## 5. 一期简化策略

- 若一期项目存在组织同步刚需，仅提前启用同步交换层基础框架和 HTTP 连接器供 `01-org-sync-audit` 复用。
- 一期不实现开放接口层、报表分析层和治理监控层。
- 一期的组织同步如需落地，由 `01-org-sync-audit` 直接使用 `07-data-sync` 的基础框架实现，不作为平台最小闭环验收前提。
- 一期不对外提供开放 API，第三方集成通过定制化开发实现。
