# 一期实施切片文档

对应架构决策：D19（工程启动条件）、D20（一期实施范围）。

## 1. 一期目标

一期不是做完整 OA，而是形成**最小平台闭环**：用户可登录、可切换身份、可在身份上下文失效后完成被动回退、可发起/审批流程、可在工作台看到待办和消息、管理员可维护组织和权限。

### 2.2 暂缓模块

| 模块 | 暂缓原因 |
|------|----------|
| 03-内容与知识 | 一期无内容管理场景，二期按需接入 |
| 05-协同办公应用 | 一期聚焦底座闭环，业务应用后续接入 |
| 07-数据服务与集成 | 一期仅用内置同步，通用开放接口二期建设 |

### 2.3 本期不作为最小闭环验收前提的子模块

| 模块 | 子模块 | 处理策略 |
|------|--------|----------|
| 01-组织与权限 | org-sync-audit | 按项目接入需求后置，非最小闭环必做 |
| 02-流程与表单 | process-monitor | 作为二期读模型增强能力后置 |
| 04-门户与工作台 | portal-model / personalization / portal-designer | 一期采用轻量配置，不落完整门户建模与设计器 |
| 06-消息移动与生态 | ecosystem | 生态登录与深度集成按项目需求增量接入 |

## 3. 一期交付里程碑

### M0：工程骨架与基础设施（2 周）

- 模块化单体工程骨架搭建（Spring Boot 3.3.x + Java 17）
- MyBatis Plus 多租户拦截器、审计字段填充、乐观锁插件
- RabbitMQ + Outbox 事件总线框架
- Redis 缓存与幂等键管理
- 统一异常处理与错误码体系
- 统一接口契约落地（请求头、响应体、分页、幂等）

### M1：组织与权限闭环（3 周）

- 组织、部门、岗位、人员、账号、角色六维模型
- 主岗/兼岗任职关系与身份上下文
- RBAC 资源权限 + 行级数据权限 + 字段级数据权限
- 登录、密码策略、锁定策略
- 身份切换、被动失效与权限重算
- `org.identity-context.invalidated` 事件驱动的上下文刷新与强制重新登录回退

### M2：流程与表单闭环（4 周）

- Flowable 集成与流程定义管理
- 流程实例启动、流转、会签/或签、条件分支
- 表单元数据定义与渲染
- 待办中心（待办/已办/我发起/抄送/草稿）
- 待办投影与聚合视图

### M3：门户与工作台（2 周）

- 工作台卡片配置与聚合接口
- 门户首页与办公中心布局
- 待办、消息、公告、常用流程等基础卡片接入
- 轻量门户配置能力（不含完整设计器与个性化模板体系）

### M4：消息与移动端（2 周）

- 统一消息中心与站内信/邮件通知
- 待办事件订阅与提醒策略
- 企业微信/钉钉等外部渠道接入
- 移动端 H5/PWA 支持

### M5：联调与验收（2 周）

- 端到端联调（登录 → 身份切换 → 发起流程 → 审批 → 待办 → 消息 → 门户 → 身份上下文失效回退）
- 性能基线测试
- 安全审计检查
- 文档闭环验收

## 4. 一期验收标准

- [ ] 用户可登录并切换身份上下文，切换后菜单/待办/权限同步刷新
- [ ] 收到 `org.identity-context.invalidated` 后，系统可完成身份上下文重刷；若上下文不可恢复，则可强制重新登录并清理本地缓存
- [ ] 管理员可维护组织、部门、岗位、人员、角色和权限
- [ ] 用户可发起流程、填写表单、审批任务，流程可正常流转至完成
- [ ] 待办中心可查看待办/已办/我发起/抄送/草稿，状态与流程一致
- [ ] 门户首页可展示待办、消息、公告、常用流程等基础卡片
- [ ] 消息可通过站内信和至少一个外部渠道（邮件/企微/钉钉）送达
- [ ] 移动端可处理高频审批和待办
- [ ] 所有写操作具备幂等保障
- [ ] 所有跨模块业务联动通过事件总线完成
- [ ] 多租户数据隔离验证通过

## 5. 一期技术约束

- 模块化单体部署，不拆微服务
- 工程基线统一使用 Spring Boot 3.3.x、Java 17、Maven 3.9+、UTF-8 与 `-parameters` 编译参数
- `HJO2OA-Bootstrap` 默认装配一期最小闭环子模块；一期后置与二期预留模块通过 Maven Profile 显式接入
- 搜索使用 PostgreSQL 全文检索（pg_jieba），不引入 Elasticsearch
- 流程引擎使用 Flowable 社区版，按 ADR-006 策略自主封装领域模型
- ORM 使用 MyBatis Plus，多租户拦截器自动注入 tenantId
- MQ 使用 RabbitMQ，事件总线基于 Spring AMQP 封装
- 对象存储使用 MinIO 或兼容 S3 的服务
