# 设计文档完善路线图

## 1. 文档目的

本文档用于盘点 HJO2OA 当前设计文档完成度，并将后续文档完善工作整理为可持续推进的路线图。当前重点已经从“补齐父模块缺失文档”转向“统一子模块契约文档深度、索引治理和跨文档一致性回归”。

本文档延续当前已收敛的项目基线：

- 采用 **模块化单体 + 独立基础设施服务** 的一期形态
- 采用 **参考设计、自主实现** 的分析与落地策略
- 工程骨架当前以 **Java 17 / Spring Boot 3.3.x / Maven 3.9+** 为真实基线
- 模块命名、目录命名和工程命名遵循当前 `HJO2OA-*` 约定

## 2. 当前文档现状盘点

### 2.1 项目级文档现状

当前项目级文档已形成稳定骨架，覆盖以下层次：

- `architecture/`：前后端总体架构规范
- `contracts/`：统一 API、统一事件、身份上下文协议
- `decisions/`：架构收敛决策、关键技术 ADR
- `implementation/`：一期切片、最终一致性、门户聚合、数据开放分层
- `engineering/`：命名总表、M0 工程骨架方案、设计路线图

结论：**项目级架构收敛已基本完成，当前重点应放在模块级与子模块级文档深度统一。**

### 2.3 当前缺口的本质

当前缺的已不再是父模块级设计文档，也不再是 `01-07` 业务域子模块的基础模板，而主要是以下三类工作：

- **顶层索引与导航治理**
  - 需让根 `README.md`、`docs/README.md`、模块 `docs/` 之间形成稳定可检索链路
- **统一契约与模块文档的口径回归**
  - 需持续检查事件命名、阶段范围、模块边界、工程基线是否一致
- **历史残留文档清理**
  - 主要集中在 `00-HJO2OA-Infrastructure` 与少量引用旧事件前缀、旧阶段说法的高影响文档

## 3. 当前阶段目标

当前阶段的目标不是继续新增父模块文档，而是让现有文档体系具备以下特征：

- 父模块文档完整，可直接作为工程与建模的上层约束
- 子模块文档结构统一，可直接作为 API、事件、前端分工和实现切片输入
- 顶层索引、模块索引、子模块契约之间引用清晰、命名统一、阶段口径一致

## 4. 下一阶段重点

### 4.1 顶层索引治理

建议将以下文件作为统一导航入口持续维护：

- 根 `README.md`
- `docs/README.md`
- 各父模块 `docs/` 目录
- 需要长期保留的路线图与工程命名文档

治理目标：

- 项目级文档能快速导航到父模块文档
- 父模块文档能快速导航到子模块契约
- 当前完成状态和下一步工作可从顶层索引直接获知

### 4.2 顶层一致性回归

持续检查以下口径：

- 工程基线：Java 17、Spring Boot 3.3.x、Maven 3.9+
- 基础设施：PostgreSQL、Redis、RabbitMQ、MinIO / S3 兼容对象存储
- 边界口径：
  - `03` 与 `05`
  - `04` 与 `07`
  - `06` 与 `07`
  - 租户与组织权限分离

### 4.3 统一事件契约扩充

当前统一事件契约已落后于模块级与子模块级文档，需要重点回归：

- `03-内容与知识` 的发布、下线、可见性与检索相关事件
- `05-协同办公应用` 的会议、任务、考勤、合同、空间等业务事件
- `07-数据服务与集成` 的服务启用、开放接口、报表刷新和治理告警事件

原则：

- 先统一跨模块事件命名，再回写各模块说明文档
- 允许模块级文档只列“核心事件”，但不得出现旧前缀或旧动作命名
- 高影响索引文档不得继续保留 `app.*` 等历史示例

## 6. 阶段验收标准

### 6.1 文档完整性

- 所有父模块均具备 `module-overview.md / development-tasks.md / domain-model.md / module-design.md`
- `Shared` 与 `Bootstrap` 至少具备模块概览和模块设计文档
- 所有子模块均具备 `api/docs/events/frontend` 四类契约文件

### 6.2 文档一致性

- 根 `README.md`、`docs/README.md` 与各模块文档索引保持同步
- 工程基线、数据库、中间件、一期范围、Profile 口径一致
- 模块前缀、事件命名和目录命名一致

### 6.3 可落地性

- 父模块文档可直接指导表设计、接口设计和工程分层
- 子模块文档可直接指导控制器、应用服务、事件和前端页面分工
- 文档引用路径可直接导航到真实文件，不依赖口头约定

## 7. 当前建议的下一步

基于当前完成度，最值得继续推进的是：

1. 对根 `README.md`、`docs/README.md`、父模块索引做一次全量导航治理
2. 对统一事件契约和高影响模块文档做一次旧前缀/旧动作命名清理
3. 对 `00-HJO2OA-Infrastructure` 和项目级契约文档做最后一轮全仓一致性回归

## 8. 结论

HJO2OA 当前已经完成父模块级文档闭环，以及 `01-07` 业务域子模块契约模板统一。文档工作重心应正式切换到：

- 顶层索引与导航治理
- 统一事件契约与模块文档口径回归
- `00-HJO2OA-Infrastructure` 及历史残留文档清理

后续文档工作应尽量遵循“少补新文件，多提高已有文件质量”的原则，让文档体系直接服务后续工程实现，而不是继续堆叠孤立说明文件。
