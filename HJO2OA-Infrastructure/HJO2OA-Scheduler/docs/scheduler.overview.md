# scheduler 概览

## 模块定位

`HJO2OA-Scheduler` 是 `00-平台基础设施` 中面向执行与运行层的统一调度子模块，负责平台级任务定义、触发调度、执行留痕、失败重试与治理操作，为配置刷新、异步补偿入口、租户初始化编排、定期巡检等场景提供统一底座。模块只负责“何时触发、如何执行、如何留痕”，不拥有业务补偿语义，也不替代业务模块自己的回滚或 Saga 编排。

| 维度 | 说明 |
|------|------|
| 领域归属 | 执行与运行层 |
| 真相源 | `ScheduledJob`、`JobExecutionRecord` |
| 主要服务对象 | `config`、`event-bus`、`tenant`、`security` 及需要周期作业的业务模块 |
| 一期定位 | 基础 Cron、手工触发、失败重试、执行记录 |
| 明确不负责 | 复杂 DAG 图形编排、业务补偿闭环、强制改写业务状态 |

## 核心职责

| 职责 | 说明 | 关键输出 |
|------|------|----------|
| 任务定义管理 | 维护任务编码、触发方式、时区、并发策略、超时策略和处理器引用 | 可审计的任务定义快照 |
| 任务调度执行 | 按 Cron 或人工指令触发执行，控制并发、超时和租户上下文 | 标准化运行请求 |
| 执行记录沉淀 | 记录执行批次、结果、耗时、错误码、重试轨迹和操作人 | `JobExecutionRecord` |
| 失败重试治理 | 依据重试策略自动或手动重试失败执行，保留操作审计 | 可追踪重试链路 |
| 运维治理入口 | 提供启停、手动触发、失败重试和配置回看能力 | 运维动作审计与二次确认 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `config` | 提供调度开关、默认超时、重试阈值等运行参数；scheduler 不拥有这些配置真相源。 |
| 上游 | `timezone` | 提供系统/租户默认时区解释规则；scheduler 只负责按时区计算触发时刻。 |
| 上游 | `tenant` | 提供租户状态、初始化完成状态和配额约束；scheduler 不定义租户隔离语义。 |
| 上游 | `security` | 提供任务执行所需的签名、密钥引用和限频策略；scheduler 不保存密钥明文。 |
| 下游 | 业务模块任务处理器 | scheduler 只触发处理器并记录结果，不拥有业务处理的领域状态。 |
| 下游 | `audit` / 运维台 | 对治理动作、执行结果、失败原因做统一留痕与查询。 |
| 边界外 | 业务补偿与人工回滚 | scheduler 可触发补偿任务，但“是否补偿、补偿什么、何时回滚”由业务模块定义。 |

## 核心能力

| 能力 | 输入 | 输出 | 约束 |
|------|------|------|------|
| 任务定义 | 任务编码、Cron、处理器引用、租户范围、超时与重试策略 | 可启停任务定义 | 任务编码全局唯一，同一时刻只允许一个生效版本。 |
| 时区感知调度 | Cron 表达式、IANA 时区、系统时间 | 下一次触发时间 | 所有时间统一以 UTC 存储，展示时按租户/系统时区转换。 |
| 并发与超时控制 | 并发策略、最大执行时长 | 运行中状态、超时结果 | 禁止通过并发绕过业务幂等要求。 |
| 失败重试 | 自动重试策略、人工重试指令 | 重试记录、下一次重试时间 | 重试只重复调度，不自动声明业务补偿成功。 |
| 手工治理 | 停用、启用、立即执行、失败重试 | 运维操作记录 | 高风险动作必须二次确认、记录原因并提供回看入口。 |

## 典型业务流

1. 运维管理员或平台模块创建任务定义，填写 `jobCode`、Cron、时区、处理器引用、超时与重试策略。
2. scheduler 校验任务编码唯一性、Cron 合法性、租户状态与策略约束后保存定义，并进入可调度状态。
3. 到达触发时刻后，scheduler 生成执行批次并调用对应处理器，写入执行开始记录。
4. 执行成功时沉淀耗时与结果快照，发布 `infra.scheduler.task-succeeded`；执行失败时根据策略进入重试或失败态，并发布失败/重试事件。
5. 运维人员可在治理台查看执行链路，对失败记录执行人工重试；该动作必须带原因、二次确认并进入统一审计。

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 调度能力收敛 | 一期仅支持基础 Cron、手工触发、失败重试和执行记录，不做复杂 DAG 图形编排。 |
| 业务语义边界 | scheduler 不直接声明业务补偿完成，也不允许将业务回滚逻辑固化为平台通用规则。 |
| 租户隔离方式 | 一期以共享库 + 租户字段隔离为主，租户级任务仅作为可选作用域，不做自动独立库迁移。 |
| 治理动作要求 | 停用、立即执行、人工重试等动作必须有二次确认、审计记录和失败重试入口。 |
| 观测模型 | 执行记录是运行真相源，图表/列表等管理视图均视为可重建投影。 |
