# scheduler 前端

## 页面清单

| 页面 | 适用端 | 权限 | 页面职责 |
|------|--------|------|----------|
| 任务清单页 | 管理端 Web | `infra.scheduler.read` | 展示任务编码、状态、下一次触发时间、最近执行结果与租户范围。 |
| 任务编辑抽屉 | 管理端 Web | `infra.scheduler.manage` | 维护 Cron、时区、处理器引用、超时与重试策略，并展示最近配置快照。 |
| 执行记录页 | 管理端 Web | `infra.scheduler.read` | 查询执行批次、失败原因、耗时趋势和重试轨迹。 |
| 执行详情侧栏 | 管理端 Web | `infra.scheduler.read` | 查看单次执行输入、输出摘要、错误码、审计链路。 |
| 治理操作对话框 | 管理端 Web | `infra.scheduler.operate` | 承载启停、立即执行、失败重试的二次确认和原因填写。 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 治理要求 |
|-----------|----------|----------|
| 任务清单页 | 按任务编码、状态、租户、模块、下一次触发时间筛选；进入详情或编辑 | 列表默认只读，不允许在列表直接无确认启停。 |
| 任务编辑抽屉 | 编辑 Cron、时区、并发策略、超时、重试次数；保存前展示差异摘要 | 保存必须携带版本号，避免覆盖并发修改。 |
| 执行记录页 | 查看失败列表、跳转执行详情、按执行状态筛选 | 失败记录必须保留“重试入口”但不能提供“强制标记成功”。 |
| 执行详情侧栏 | 查看重试链路、错误上下文、关联事件与审计记录 | 敏感参数和密钥引用必须脱敏展示。 |
| 治理操作对话框 | 启停任务、立即执行、重试失败执行 | 必须先展示影响面、要求二次确认、填写操作原因并回写审计。 |

## 权限与状态约束

| 维度 | 约束 |
|------|------|
| 权限 | 查询、定义维护、治理操作分离授权；只有运维或平台管理员可执行高风险动作。 |
| 任务状态 | 页面至少区分启用、停用、运行中、最近成功、最近失败、重试中等状态。 |
| 执行状态 | 单次执行需明确展示等待、运行中、成功、失败、重试中、已放弃等运行态。 |
| 审计 | 每个高风险按钮都必须能跳转到对应审计记录或执行详情。 |

## 终端与治理约束

- 一期仅提供管理端 Web 页面，不提供移动端写操作入口。
- 所有治理动作都必须经过二次确认，且展示“这不会替代业务补偿”的明确提示。
- 页面必须保留失败重试入口与最近一次配置快照回看能力，但不提供跨模块业务状态回滚按钮。
