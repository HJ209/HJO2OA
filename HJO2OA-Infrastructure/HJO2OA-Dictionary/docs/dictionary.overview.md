# 数据字典概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块角色 | `HJO2OA-Dictionary` 是 `00-平台基础设施` 契约与策略层子模块，统一承载字典类型与字典项真相源，向流程、内容、协同办公等模块提供可复用选项集。 |
| 策略对象 | `DictionaryType`、`DictionaryItem` 负责定义字典编码、层级关系、排序、启停和多语言展示规则。 |
| 运行对象 | 业务单据中的已选字典值、前端本地下拉缓存、聚合读模型快照属于运行对象，不由本模块持有。 |
| 边界强调 | 本模块管理“可引用的选项定义”，不管理业务状态机，不承接业务对象上的实际取值生命周期。 |

## 核心职责

| 职责 | 说明 |
|------|------|
| 字典类型治理 | 维护全局/租户级字典类型、分类、层级开关与缓存属性。 |
| 字典项治理 | 维护字典项编码、展示值、父子关系、排序、启停和多语言展示快照。 |
| 字典引用输出 | 向业务模块提供稳定的按编码查询、级联展开和启用态过滤能力。 |
| 变更治理 | 所有新增、调整、启停、排序、导入导出和人工修复动作都进入统一审计，并触发下游缓存刷新。 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `01-组织与权限` | 只引用操作者、组织、角色等主体 ID 做权限判断和审计留痕，不复制主体主数据。 |
| 上游 | `HJO2OA-Tenant` | 可基于租户初始化结果装载默认字典，但租户生命周期不归本模块管理。 |
| 下游 | `02-流程与表单` | 表单字段、流程条件、待办筛选可引用字典编码；本模块不存储表单值。 |
| 下游 | `03-内容与知识`、`05-协同办公应用` | 分类、标签、业务选项可引用字典项；业务模块自行决定字段必填、状态流转和历史留存。 |
| 下游 | `04-门户与工作台` | 门户聚合页可直接读取字典显示值；前端缓存失效由事件或查询版本控制触发。 |
| 边界外 | 业务枚举快照 | 不允许业务模块将字典复制成新的“本地真相源”；允许做只读投影和前端缓存，但必须可刷新。 |

## 核心能力

| 能力 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| 字典类型管理 | `code`、`name`、`category`、`hierarchical`、`tenantId` | 可被引用的字典类型定义 | `code` 在全局或租户范围内唯一。 |
| 字典项管理 | `itemCode`、`displayName`、`parentItemId`、`sortOrder`、`enabled` | 可查询的字典项树/列表 | 层级字典不能成环；停用不删除历史引用。 |
| 引用查询与级联解析 | 字典编码、父项编码、语言、启用态 | 扁平列表或树形选项集 | 默认仅返回启用项；级联查询按排序稳定返回。 |
| 多语言展示 | `multiLangValue` 或回退语言 | 展示值快照 | 一期仅维护字典项内的多语言展示快照，不替代独立翻译工作流。 |
| 导入导出与治理 | 模板文件、批量操作请求 | 变更结果、失败明细、审计记录 | 批量写入必须幂等，且失败项可回查。 |

## 典型业务流

1. 管理员创建字典类型，声明编码、分类、是否层级和租户适用范围。
2. 在字典类型下新增或导入字典项，配置编码、展示值、排序、父子关系和启停状态。
3. 业务模块通过字典编码查询可用项，并在表单、筛选器、门户聚合中以只读方式引用。
4. 字典项发生新增、启停、排序或展示值调整后，本模块发布 `infra.dictionary.item-changed` 与汇总事件 `infra.dictionary.updated`。
5. 下游缓存和投影收到事件后刷新本地快照；若刷新失败，应重新回源查询字典真相源而非手工修补缓存值。

## 一期实现约束

- 一期仅支持全局和租户级字典，不单独持有组织级或角色级字典主数据。
- 一期支持树形级联和扁平字典两种模型，不提供版本化发布链，变更以启停和排序调整为主。
- 一期的多语言展示采用字典项快照字段，不建设独立字典翻译审批流。
- 导入导出以字典类型为边界，不支持跨租户批量合并或跨环境自动同步。
- 任何治理动作，包括批量导入、手工修复、启停、排序和删除尝试，都必须记录统一审计。
