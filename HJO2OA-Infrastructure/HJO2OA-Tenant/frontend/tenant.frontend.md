# tenant 前端

## 页面清单

| 页面 | 适用端 | 权限 | 页面职责 |
|------|--------|------|----------|
| 租户清单页 | 管理端 Web | `infra.tenant.read` | 展示租户状态、初始化状态、套餐、隔离模式与预警摘要。 |
| 租户详情/编辑抽屉 | 管理端 Web | `infra.tenant.manage` | 维护租户基本信息、默认语言/时区、套餐和隔离模式。 |
| 初始化中心 | 管理端 Web | `infra.tenant.operate` | 展示初始化步骤、进度、失败原因和重试入口。 |
| 配额治理页 | 管理端 Web | `infra.tenant.govern` | 展示各类配额用量、预警趋势、调整与回滚入口。 |
| 状态治理对话框 | 管理端 Web | `infra.tenant.govern` | 停用租户前展示影响摘要、二次确认和审计原因填写。 |
| 变更审计抽屉 | 管理端 Web | `infra.tenant.read` | 查看租户档案、初始化与配额治理的历史记录。 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 治理要求 |
|-----------|----------|----------|
| 租户清单页 | 按编码、名称、状态、初始化状态、套餐、隔离模式筛选；进入详情或初始化中心 | 列表默认只读，不提供无确认的停用快捷操作。 |
| 租户详情/编辑抽屉 | 创建或编辑租户档案；展示默认语言、时区、套餐和隔离模式 | 保存前必须显示差异摘要，避免误改平台级隔离信息。 |
| 初始化中心 | 触发初始化、查看步骤日志、重试失败步骤、跳转下游模块回执 | 发起与重试都必须二次确认，并附带操作原因。 |
| 配额治理页 | 查看用量、设置阈值、调整上限、回滚最近一次调整 | 变更必须保留前值、后值和回滚入口，不允许静默覆盖。 |
| 状态治理对话框 | 停用租户、查看影响对象数量、填写停用原因 | 必须展示“不会自动删除业务数据”的提示，并强制二次确认。 |
| 变更审计抽屉 | 查看创建、更新、初始化、调额、停用链路 | 每个治理操作都应可追到操作人和结果。 |

## 权限与状态约束

| 维度 | 约束 |
|------|------|
| 权限 | 查询、档案维护、初始化、治理分层授权；只有平台管理员可执行停用和调额。 |
| 状态 | 页面至少区分草稿/启用/停用，以及未初始化/初始化中/初始化成功/初始化失败两个维度。 |
| 边界提示 | 前端不得展示“租户组织树”“租户角色管理”等越界入口，避免混淆组织权限语义。 |
| 审计 | 初始化、停用、配额调整和回滚都必须能跳到审计记录。 |

## 终端与治理约束

- 一期仅提供管理端 Web 页面，不提供移动端写操作。
- 高风险动作必须经过二次确认，且在提交前展示影响范围、原因输入和后续重试或回滚入口。
- 页面允许查看租户初始化涉及的下游模块回执，但不得直接在 tenant 前端修改这些模块的业务主数据。
