# I18n 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-I18n` |
| 模块定位 | `00-平台基础设施` 的共享语义资源中心，统一管理语言目录、资源包、回退链和多语言消息解析 |
| 真相源边界 | 拥有 `LocaleBundle`、`LocaleResourceEntry`、默认语言与回退策略等技术真相源；不拥有业务字段翻译值、不拥有业务模板内容、不承担前端页面运行态缓存真相源 |
| 一期目标 | 建立语言管理、资源包维护、后端消息国际化、用户语言偏好和热更新能力，支撑全平台统一文案口径 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 语言目录与默认策略 | 管理启用语言、默认语言、租户可见语言和回退链 | 支持新增、启停、默认值维护和回退校验 |
| 资源包与资源项管理 | 按模块维护资源包、资源键、占位符规则和版本信息 | 支持编辑、导入导出、发布和废弃 |
| 统一消息解析 | 为前端文案、后端提示、错误消息提供统一解析入口 | 支持按 `locale`、模块、资源键解析并回退 |
| 语言偏好管理 | 管理个人或作用域级语言偏好，驱动前后端刷新 | 支持个人偏好与管理员代设 |
| 热更新与治理审计 | 资源包发布、回滚、批量导入和热更新必须可审计 | 支持审计查询和手工刷新 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `HJO2OA-Config` | 提供默认语言、格式化策略和环境级配置，不在本模块复制配置真相源 |
| 上游 | `HJO2OA-ErrorCode` | 错误码模块复用资源键，不在错误码模块重复维护多语言文本 |
| 下游 | `01-07` 所有业务域与前端壳层 | 统一消费共享资源键和语言偏好，不各自维护多语言消息真相源 |
| 下游 | `HJO2OA-DataI18n` | `data-i18n` 复用语言目录与回退规则，但业务字段翻译值不归本模块 |
| 非本模块 | 业务字段翻译 | 业务数据的 `entityId + field + locale` 翻译值归 `HJO2OA-DataI18n` 所有 |
| 非本模块 | 页面运行态缓存 | 浏览器资源缓存、CDN 缓存、前端内存字典是运行投影，不是本模块真相源 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 语言目录 | 语言清单与可用性策略 | 平台可用语言集合 | 复杂语言市场和自动机器翻译 |
| 资源包管理 | `LocaleBundle`、`LocaleResourceEntry` | 模块级资源键值真相源 | 富文本模板 CMS 化管理 |
| 回退解析 | 语言回退链和占位符规则 | 统一消息解析结果 | 任意循环回退或隐式覆盖 |
| 偏好切换 | 作用域语言偏好与刷新触点 | 前后端语言切换输入 | 独立身份体系 |
| 热更新治理 | 发布、回滚、刷新和校验 | 资源包版本治理和审计 | 绕过审计的在线改库 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 管理员维护语言目录、默认语言和回退链，校验回退链无循环 | 生效语言策略 |
| 2 | 资源维护人按模块导入或编辑资源项，完成占位符与重复键校验 | 待发布资源包 |
| 3 | 资源包发布后触发热更新，前后端按 `locale + module + key` 获取解析结果 | 已生效资源版本 |
| 4 | 用户或管理员调整语言偏好，系统发布语言切换事件并刷新界面上下文 | 语言偏好变更记录 |
| 5 | 资源包回滚、发布、导入等治理动作统一进入审计 | 审计留痕与治理历史 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| `00` 只拥有共享语义真相源 | 资源包和语言策略归 `i18n`，业务字段翻译值不归本模块 |
| 与 `data-i18n` 严格分层 | `i18n` 管资源键与共享消息，`data-i18n` 管业务字段翻译值，禁止混放 |
| 回退链必须可验证 | 回退语言链不能成环，默认语言必须在启用语言集合内 |
| 热更新只刷新投影 | 客户端缓存、服务端缓存和前端字典都是运行投影，刷新失败不得篡改资源包真相源 |
| 资源键要稳定 | 资源键发布后应保持兼容，破坏性调整必须显式治理和审计 |
| 治理动作必须审计 | 批量导入、发布、回滚、强制刷新都必须记录操作人、批次号和原因 |
