# DataI18n 前端说明

## 页面清单

| 页面/区域 | 页面职责 | 权限要求 | 状态要点 | 移动端约束 |
|-----------|----------|----------|----------|------------|
| 翻译工作台 | 查看待翻译、已翻译、已审核和已完成对象清单 | 翻译员、审核人、翻译管理员 | 待翻译、翻译中、待审核、已完成 | 移动端仅保留待办查看和基础编辑 |
| 翻译项对照编辑页 | 对照原值编辑目标语言译文，查看术语和历史版本 | 翻译员、审核人 | 未开始、已保存、校验失败、待审核 | 移动端支持单条编辑，不支持复杂批量操作 |
| 缺失翻译治理页 | 查看缺失项、完成度、重开对象和忽略记录 | 翻译管理员、审核人 | 正常、缺失、已忽略、待重建 | 移动端只看摘要，不支持治理操作 |
| 导入导出页 | 管理批量导入导出、冲突确认、错误报告和批次历史 | 翻译管理员、审计员 | 导入中、待确认、已完成、已失败 | 移动端不支持导入导出 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 说明 |
|-----------|----------|------|
| 翻译工作台 | 按模块/对象/语言筛选、领取任务、跳转编辑、查看完成度 | 列表需突出原值、目标语言、审核状态和最近更新时间 |
| 翻译项对照编辑页 | 填写译文、切换语言 Tab、提交审核、查看历史版本 | 页面必须同时展示原值与当前译文，避免误覆盖已审核内容 |
| 缺失翻译治理页 | 标记完成、忽略缺失、重建候选、查看影响对象 | 治理动作只影响翻译工作流，不得直接修改业务对象状态 |
| 导入导出页 | 上传批量文件、查看冲突、确认覆盖、下载错误报告 | 覆盖确认前必须明确展示将覆盖的键集合和审核状态 |

## 前端约束

| 项 | 说明 |
|----|------|
| 权限 | 必须区分翻译编辑、翻译审核、批量治理和审计查看权限 |
| 状态 | 前端需清晰区分翻译状态与业务对象状态，禁止把“翻译完成”渲染成“业务发布成功” |
| 终端 | 一期移动端仅保留待办查看和单条译文编辑，不支持导入导出、覆盖确认和缺失项治理 |
| 治理 | 覆盖确认、完成标记、重开对象、忽略缺失都必须填写原因并进入审计 |
| 边界 | 前端只服务业务字段翻译治理，不承担共享资源包编辑，该职责归 `HJO2OA-I18n` |
