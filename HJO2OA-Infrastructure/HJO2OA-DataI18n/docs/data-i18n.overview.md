# DataI18n 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-DataI18n` |
| 模块定位 | `00-平台基础设施` 共享资源层中的业务字段翻译真相源，统一管理多语言翻译值、翻译状态和 locale 感知查询 |
| 真相源边界 | 拥有 `TranslationEntry` 及其状态、导入导出批次和完成度口径；不拥有业务对象主数据、不拥有共享资源包文本、不承担业务生命周期推进 |
| 一期目标 | 建立基础翻译值存储、按 locale 解析、翻译工作台、批量导入导出和完成事件输出能力 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 翻译值存储 | 以 `entityType + entityId + fieldName + locale` 维护翻译值和状态 | 支持新增、更新、审核、唯一键校验 |
| locale 感知查询 | 对外提供当前语言优先、默认语言回退、原值兜底的查询能力 | 支持单条与批量解析 |
| 翻译工作台 | 提供待翻译、已翻译、已审核、缺失项等工作视图 | 支持人工翻译、审核和完成标记 |
| 批量导入导出 | 维护翻译批次、导入校验、导出复核和覆盖策略 | 支持 Excel/JSON 等基础批量能力 |
| 治理与审计 | 对批量覆盖、完成确认、缺失项忽略、重建候选等动作留痕 | 支持统一审计与治理备注 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `01-07` 业务域 | 业务域提供 `entityType`、`entityId`、原始字段值和可翻译字段范围，本模块不拥有业务对象生命周期 |
| 上游 | `HJO2OA-I18n` | 复用语言目录和回退规则，不在本模块重复维护共享语义资源包 |
| 下游 | `03/04/05/07` 以及前端壳层 | 向读模型、列表、详情和导出场景提供 locale 感知的翻译结果 |
| 下游 | 治理与审计模块 | 输出翻译完成、翻译更新、缺失项治理等运行状态 |
| 非本模块 | 共享文案与系统提示 | 按钮文案、错误消息、系统提示仍归 `HJO2OA-I18n` |
| 非本模块 | 业务对象主状态 | 译文完成或缺失不应直接推进业务对象状态机，由业务域自行决定如何消费 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 翻译值真相源 | `TranslationEntry` | 业务字段多语言值与状态 | 机器翻译引擎内置化 |
| 解析与回退 | locale 解析链 | 当前语言视图值和回退结果 | 复杂语言变体自动推断 |
| 批量导入导出 | 导入批次与校验报告 | 批量维护与复核结果 | 自动语料库训练 |
| 完成度治理 | 缺失项、完成度看板、审核状态 | 翻译治理输入与完成事件 | 替代业务域的任务中心 |
| 事件输出 | 翻译变更与完成事件 | 刷新读模型和下游投影 | 直接回写业务主数据 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 业务域通过 API 或接入事件声明某对象字段需要翻译，并传入原始值上下文 | 待翻译候选项 |
| 2 | 翻译员在工作台填写目标语言译文，审核人校验术语和格式 | 已翻译或已审核条目 |
| 3 | 查询场景按 `locale` 解析翻译值；若缺失则回退默认语言或原始值 | locale 感知视图值 |
| 4 | 批量导入导出用于集中维护译文，覆盖和冲突必须显式确认 | 导入批次与冲突报告 |
| 5 | 当对象所需语言集合满足完成口径时发布完成事件，供下游刷新投影 | 完成事件与治理记录 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| `00` 只拥有翻译值真相源 | `data-i18n` 只持有译文和值状态，不拥有业务主数据和业务状态真相源 |
| 翻译状态与业务状态解耦 | `TRANSLATED/REVIEWED/COMPLETED` 只表示翻译治理状态，不代表业务对象发布、归档或生效 |
| 唯一键必须稳定 | `entityType + entityId + fieldName + locale` 是基础唯一键，禁止同键多值并存 |
| 回退结果是运行视图 | 默认语言回退和原值兜底是查询投影，不得把回退结果反写为翻译真相源 |
| 批量覆盖必须显式治理 | 导入覆盖、审核驳回、缺失项忽略都必须带批次号、操作人和原因 |
| 候选项可由事件驱动生成 | 业务对象变更事件只用于生成待翻译候选和刷新投影，不转移业务对象所有权 |
