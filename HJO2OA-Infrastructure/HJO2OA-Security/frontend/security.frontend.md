# security 前端

## 页面清单

| 页面 | 适用端 | 权限 | 页面职责 |
|------|--------|------|----------|
| 安全策略清单页 | 管理端 Web | `infra.security.read` | 展示策略编码、类型、作用域、状态、最近版本和变更时间。 |
| 策略详情/编辑页 | 管理端 Web | `infra.security.manage` | 维护脱敏、签名、密码、限频等策略快照，并查看版本差异。 |
| 策略发布与回滚对话框 | 管理端 Web | `infra.security.operate` | 承载策略生效、回滚的二次确认与影响面提示。 |
| 密钥引用与轮换台 | 管理端 Web | `infra.security.secret.rotate` | 查看 `keyRef` 状态、算法、轮换历史和当前引用范围。 |
| 脱敏/签名测试台 | 管理端 Web | `infra.security.use` | 做脱敏预览、签名和验签测试，不暴露真实敏感数据。 |
| 异常中心与敏感操作页 | 管理端 Web | `infra.security.read` / `infra.security.operate` | 查看异常、处置风险、检索敏感操作审计。 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 治理要求 |
|-----------|----------|----------|
| 安全策略清单页 | 按策略类型、状态、租户、更新时间筛选；进入详情或治理对话框 | 清单默认只读，不允许无确认直接发布。 |
| 策略详情/编辑页 | 编辑规则表达式、作用域、阈值和版本说明；对比历史版本差异 | 保存必须带版本号，防止覆盖并发修改。 |
| 策略发布与回滚对话框 | 发布策略、回滚到历史版本、填写操作原因 | 必须展示影响范围、二次确认和审计号。 |
| 密钥引用与轮换台 | 发起轮换、查看当前引用范围、查看最近轮换结果 | 轮换前必须校验引用影响，且不能展示密钥明文。 |
| 脱敏/签名测试台 | 输入样例进行脱敏预览、签名、验签 | 测试输入应即时脱敏；调用结果进入安全审计。 |
| 异常中心与敏感操作页 | 过滤异常、查看证据、确认处置、回看敏感操作 | 处置动作必须保留处理意见并支持回跳到相关策略版本。 |

## 权限与状态约束

| 维度 | 约束 |
|------|------|
| 权限 | 查询、管理、发布/回滚、密钥轮换、服务调用分层授权。 |
| 策略状态 | 页面至少区分草稿、生效、停用、废弃等状态，并显示当前版本号。 |
| 异常状态 | 异常中心需区分待研判、已确认、已处置，禁止前端本地伪造状态流转。 |
| 数据保护 | 所有页面都不得展示明文密钥、原始密码或完整敏感字段。 |

## 终端与治理约束

- 一期仅提供管理端 Web 页面，不提供移动端写操作入口。
- 发布、回滚、密钥轮换和异常处置都必须经过二次确认，并要求填写操作原因。
- 页面必须保留审计跳转、策略版本回看和最近一次治理动作结果查看入口，但不得提供账号主模型编辑能力。
