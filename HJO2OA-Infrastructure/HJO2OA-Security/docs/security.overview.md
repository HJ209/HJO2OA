# security 概览

## 模块定位

`HJO2OA-Security` 是 `00-平台基础设施` 中负责平台级安全策略与治理的子模块，维护密钥引用、脱敏、签名、密码、限频与异常检测等安全规则，为网关、开放接口和各业务模块提供统一安全底座。该模块不拥有账号主模型，也不替代认证、会话和角色授权主流程。

| 维度 | 说明 |
|------|------|
| 领域归属 | 契约与策略层 + 运维与治理层 |
| 真相源 | `SecurityPolicy`、`SecretKeyMaterial`、`MaskingRule`、`RateLimitRule` |
| 核心目标 | 策略统一、密钥引用治理、异常检测、敏感操作留痕 |
| 一期定位 | 基础密钥引用、脱敏、签名、密码规则、限频、规则型异常检测 |
| 明确不负责 | 账号主数据、认证登录主流程、角色授权模型、复杂自适应风控 |

## 核心职责

| 职责 | 说明 | 关键输出 |
|------|------|----------|
| 安全策略管理 | 维护安全策略编码、类型、作用域、版本和状态 | `SecurityPolicy` |
| 密钥引用与轮换元数据 | 管理密钥引用、算法、轮换状态和版本，不存业务明文密钥 | `SecretKeyMaterial` |
| 脱敏与签名规则 | 提供字段脱敏规则、签名/验签策略和调用约束 | 可复用安全规则 |
| 密码与限频策略 | 维护密码复杂度、历史密码策略、限频窗口与对象类型 | 密码与限频规则 |
| 异常检测与敏感操作审计 | 识别异常登录、暴力破解、策略越权修改等风险，并统一留痕 | 异常事件与敏感操作审计 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `01-组织与权限` | 提供账号、人员、登录成功/失败等身份上下文；security 不拥有账号主模型。 |
| 上游 | `tenant` | 提供租户作用域和停用状态；security 不负责租户隔离主规则。 |
| 上游 | `config` | 提供全局阈值、白名单或策略开关；security 不拥有配置真相源。 |
| 下游 | API 网关、开放接口、业务模块 | 统一消费安全策略、签名规则、限频规则和脱敏规则。 |
| 下游 | 运维治理台、审计模块、消息模块 | 消费异常事件、敏感操作事件和策略变更结果。 |
| 边界外 | 认证、会话、权限授予 | 认证流程、会话生命周期、角色授权仍归 `01-组织与权限`。 |
| 边界外 | 明文密钥存储 | security 只保存密钥引用与元数据，密钥明文不进入业务表和页面输出。 |

## 核心能力

| 能力 | 输入 | 输出 | 约束 |
|------|------|------|------|
| 策略定义 | 策略编码、类型、作用域、状态、配置快照 | 生效中的安全策略 | 策略更新必须保留版本与差异快照。 |
| 密钥引用治理 | `keyRef`、算法、轮换状态、使用范围 | 密钥元数据与轮换记录 | 不向调用方返回密钥明文。 |
| 脱敏规则 | 数据类型、规则表达式、展示场景 | 统一脱敏展示结果 | 仅定义规则，不拥有业务字段主数据。 |
| 签名/验签 | 签名策略、算法、载荷摘要 | 签名结果、验签结论 | 失败结果必须可审计，且不得泄露敏感摘要。 |
| 密码与限频 | 密码复杂度、窗口、对象类型、阈值 | 平台统一安全约束 | 只定义规则，不拥有账号状态机。 |
| 异常检测 | 登录结果、敏感操作、策略变更 | 异常事件与告警 | 一期基于规则检测，不做自学习画像。 |

## 典型业务流

1. 安全管理员新建或调整安全策略，维护作用域、策略快照和生效状态。
2. security 校验规则表达式、密钥引用和作用域后保存策略快照，并在生效时发布 `infra.security.policy-updated`。
3. 网关、开放接口和业务模块按策略执行签名、脱敏、限频和密码校验。
4. 登录失败、签名失败、限频命中、敏感策略变更等信号进入安全检测链路，security 记录敏感操作并在必要时发布 `infra.security.sensitive-operation` 或 `infra.security.anomaly-detected`。
5. 运维或安全管理员在治理台查看异常详情，对风险事件进行确认、处置或回滚策略版本，所有动作进入统一审计。

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 能力范围 | 一期仅覆盖基础密钥引用、脱敏、签名、密码规则、限频与规则型异常检测。 |
| 账号边界 | security 不拥有账号、会话、角色和权限模型，只引用身份上下文。 |
| 密钥治理 | 一期只管理密钥引用和轮换元数据，不在业务表或页面展示明文密钥。 |
| 风控能力 | 一期不实现复杂自适应风控、行为画像和机器学习检测。 |
| 治理要求 | 策略发布、回滚、密钥轮换、异常处置必须支持二次确认、审计与回滚或重试入口。 |
