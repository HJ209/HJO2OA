# 错误码体系前端

## 页面清单

| 页面/区域 | 适用端 | 页面职责 |
|-----------|--------|----------|
| 错误码管理页 | 管理端 | 展示错误码目录、模块归属、严重级别、HTTP 状态和废弃状态。 |
| 错误码详情/编辑页 | 管理端 | 编辑消息键、分类、可重试属性并查看替代关系。 |
| 错误码文档页 | 管理端、开发端 | 面向开发和联调场景展示错误码目录与说明。 |
| 使用统计页 | 管理端 | 展示错误码使用频次、趋势和最近引用模块。 |
| 响应映射预览面板 | 管理端 | 预览统一错误响应体、消息键和前端展示文案。 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 约束 |
|-----------|----------|------|
| 错误码管理页 | 新建、筛选、按模块分组浏览 | 仅管理定义，不展示业务异常堆栈。 |
| 错误码详情/编辑页 | 修改元数据、废弃、指定替代码 | 已发布错误码需显示兼容风险提示。 |
| 错误码文档页 | 搜索目录、查看接口映射、复制说明 | 文档页只读，不提供治理按钮。 |
| 使用统计页 | 切换时间窗、查看趋势、下钻模块 | 统计仅反映引用趋势，不代表异常根因。 |
| 响应映射预览面板 | 输入语言和 HTTP 场景查看前端展示 | 只预览映射，不生成异常实例。 |

## 权限、状态与终端治理约束

- 权限约束：
  - 错误码管理员可维护定义和替代关系。
  - 普通开发和前端只读查看目录与文档。
- 状态约束：
  - 错误码展示 `deprecated=true/false`。
  - 替代关系应在详情页显式展示，避免误用已废弃错误码。
  - 可重试属性应以标签方式展示，供调度和事件场景参考。
- 终端约束：
  - 一期仅提供管理端和开发端文档页，不提供移动端治理能力。
  - 业务前端通过统一错误响应映射展示文案，不单独维护对外错误码目录。
- 治理约束：
  - 页面必须提示“错误码是统一定义，不是业务异常真相源”。
  - 废弃、替代和批量修订前需展示兼容性影响与审计留痕提示。
