# Attachment 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-Attachment` |
| 模块定位 | `00-平台基础设施` 共享资源层中的附件元数据与绑定关系中心，统一管理文件资产、版本、预览状态和配额占用 |
| 真相源边界 | 拥有 `AttachmentAsset`、`AttachmentVersion`、`AttachmentBinding` 等元数据真相源；不拥有业务生命周期、不拥有业务可见范围真相源，文件二进制内容走对象存储 |
| 一期目标 | 建立上传、下载、预览、版本、业务绑定和配额预警能力，供各业务域统一复用附件基础设施 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 资产元数据管理 | 管理文件名、大小、校验摘要、存储键、预览状态和所属租户 | 支持上传建档、查询、逻辑删除 |
| 版本链管理 | 维护追加式版本链、版本差异和当前有效版本 | 支持新版本创建和历史版本查询 |
| 绑定关系管理 | 维护附件与业务对象的绑定角色和激活状态 | 支持绑定、解绑、绑定摘要查询 |
| 预览与下载编排 | 对接对象存储和预览服务生成预览、下载凭证和状态 | 支持基础预览、下载授权和失败重试 |
| 配额与治理审计 | 统计配额占用、输出预警、治理孤儿文件和删除动作 | 支持预警、清理候选和统一审计 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `02/03/05/07` 业务域 | 提供 `businessType`、`businessId`、绑定角色和访问控制回调，本模块不拥有业务对象生命周期 |
| 上游 | 对象存储、预览转换、病毒扫描等外部能力 | 二进制上传、预览转换和扫描由外部运行能力承担，本模块只保存元数据和状态 |
| 下游 | 各业务域详情页、列表页、下载与预览入口 | 业务域通过 `attachmentId` 引用附件元数据和绑定关系 |
| 下游 | 审计、治理和消息触达 | 配额预警、删除、版本创建等事件提供治理输入 |
| 非本模块 | 业务生命周期 | 附件删除、归档、可见范围最终以业务域规则为准，附件模块不能反向拥有业务状态机 |
| 非本模块 | 文件二进制真相源 | 文件二进制内容、分片、对象版本和存储副本不存放在本模块表内 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 元数据真相源 | `AttachmentAsset` | 统一文件资产记录 | 自建对象存储实现 |
| 版本管理 | `AttachmentVersion` | 追加式版本链和历史版本摘要 | 覆盖式改写旧版本 |
| 绑定关系 | `AttachmentBinding` | 业务对象与附件的绑定真相源 | 由附件模块接管业务生命周期 |
| 预览下载编排 | 预览状态和授权票据 | 预览、下载和状态查询 | 富媒体编辑器本体 |
| 配额治理 | 配额统计、预警、清理候选 | 运维看板和治理输入 | 复杂内容检索平台 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 客户端申请上传会话或分片任务，文件二进制直接上传到对象存储 | 上传会话与对象键 |
| 2 | 上传完成回调创建 `AttachmentAsset` 并按业务上下文写入 `AttachmentBinding` | 附件元数据与绑定关系 |
| 3 | 预览服务异步转换文件并回写 `previewStatus`，前端根据状态展示预览入口 | 预览状态 |
| 4 | 业务场景追加新版本时创建 `AttachmentVersion`，保留历史版本链 | 版本记录与版本事件 |
| 5 | 删除或治理前先校验活跃绑定和配额规则，治理动作进入审计，物理清理异步执行 | 删除记录、清理任务和审计留痕 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 只持有元数据和绑定关系 | 附件模块拥有文件元数据、版本和绑定关系，不拥有业务对象生命周期 |
| 文件二进制走对象存储 | 一期不在业务数据库存放二进制内容，上传、下载、预览均通过对象存储和外部运行能力完成 |
| 版本必须追加式 | 新版本只能追加，不允许覆盖旧版本或篡改历史校验摘要 |
| 删除必须先校验绑定 | 有活跃绑定的附件不得直接物理删除，必须先解除绑定或显式治理确认 |
| 权限默认继承业务 | 一期优先采用 `INHERIT_BUSINESS`，附件不建设独立业务权限真相源 |
| 治理动作必须审计 | 删除、回放预览、配额处理、孤儿清理都必须记录操作人、原因和结果 |
