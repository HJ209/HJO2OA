# EventBus 前端说明

## 页面清单

| 页面/区域 | 页面职责 | 权限要求 | 状态要点 | 移动端约束 |
|-----------|----------|----------|----------|------------|
| 事件契约目录页 | 查看事件定义、版本、发布方、订阅方和兼容状态 | 事件契约管理员、审计员 | 草稿、已生效、已废弃、兼容窗口中 | 移动端只建议查看，不开放编辑 |
| 订阅关系管理页 | 管理消费者订阅、匹配规则、重试和死信策略 | 事件契约管理员、运维管理员 | 启用、停用、待验证、策略冲突 | 移动端不提供订阅编辑 |
| 事件追踪页 | 查看事件流、投递轨迹、关联链路和失败摘要 | 运维管理员、审计员 | 已发布、投递中、部分失败、已完成 | 移动端仅保留摘要和详情查看 |
| 死信与回放页 | 处理死信、发起回放、查看重试历史和处置记录 | 运维管理员 | 待处理、重试中、已关闭、需升级 | 移动端不支持回放和关闭操作 |
| 运行审计与指标页 | 查看吞吐、失败率、积压和治理动作审计 | 运维管理员、审计员 | 正常、告警、积压、恢复中 | 移动端仅看看板摘要，不做复杂筛选 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 说明 |
|-----------|----------|------|
| 事件契约目录页 | 条件筛选、查看 schema、比对版本差异、查看订阅关系 | 页面必须突出 `eventType`、版本、状态和发布模块，不展示业务语义解释 |
| 订阅关系管理页 | 新增订阅、调整重试、启停订阅、查看死信策略 | 策略变更前需展示影响消费者和预计投递变化 |
| 事件追踪页 | 按 `traceId/correlationId/eventType` 检索、查看投递时间线 | 详情页展示信封、投递尝试和失败摘要，不允许在此页直接改写事件内容 |
| 死信与回放页 | 发起回放、填写原因、查看回放结果、关闭异常单 | 回放前必须二次确认，明确“只重驱投递、不重演业务事务” |
| 运行审计与指标页 | 查看失败率趋势、积压分布、治理动作审计 | 运行指标用于定位链路风险，不替代业务模块的运行看板 |

## 前端约束

| 项 | 说明 |
|----|------|
| 权限 | 至少区分契约管理、运维治理、审计查看三类权限，禁止在前端合并成单一超级角色 |
| 状态 | 页面按钮必须以总线运行状态为准，不能把业务领域状态映射成事件总线状态 |
| 终端 | 一期移动端只提供事件追踪和告警摘要查看，不支持订阅治理、死信回放和批量操作 |
| 治理 | 回放、停用订阅、关闭死信等治理动作必须强制填写原因并写入审计，不允许匿名操作 |
| 边界 | 前端只服务契约与运行治理，不承担业务流程监控中心或消息编排中心职能 |
