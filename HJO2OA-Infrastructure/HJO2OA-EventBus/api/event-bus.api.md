# EventBus 接口说明

## 接口分组

| 接口分组 | 关键查询 | 关键写入 | 分页与筛选 | 幂等要求 | 权限要求 | 审计要求 | 错误处理 |
|----------|----------|----------|------------|----------|----------|----------|----------|
| 事件契约管理 | 查询事件定义列表、版本详情、发布方和订阅方关系 | 注册事件定义、启用/废弃定义、更新载荷描述 | 支持按 `eventType`、`modulePrefix`、`eventVersion`、`status`、`publisherCode` 分页筛选 | 按 `eventType + eventVersion` 幂等注册；重复注册返回首个成功结果 | 事件契约管理员可写，审计员只读 | 注册、启停、废弃、兼容窗口调整必须全量审计 | `409` 事件类型版本冲突，`422` 载荷结构非法，`404` 定义不存在 |
| 订阅关系管理 | 查询订阅列表、消费者能力、重试与死信策略 | 新增订阅、调整匹配规则、启停订阅、更新重试策略 | 支持按 `subscriberCode`、`eventType`、`matchMode`、`active`、`deadLetterEnabled` 分页筛选 | 按 `subscriberCode + matchKey + eventVersion` 幂等写入 | 事件契约管理员、运维管理员可写 | 订阅新增、停用、重试策略调整必须记录操作原因 | `409` 订阅冲突，`422` 通配规则非法，`404` 消费者不存在 |
| 事件发布与追踪 | 查询事件列表、详情、投递尝试、关联链路 | 发布事件、批量发布、内部补发入站请求 | 支持按 `eventType`、`source`、`tenantId`、`correlationId`、`publishStatus`、时间区间分页筛选 | 按 `eventId` 或 `Idempotency-Key` 幂等；批量发布按 `batchNo` 幂等 | 内部服务账号可写；运维和审计角色可读 | 手工补发、批量补发、失败重试必须留痕并记录来源 | `400` 信封缺字段，`404` 事件未注册，`409` 重复事件，`422` 版本不兼容 |
| 死信与回放治理 | 查询死信列表、失败统计、回放历史、关闭原因 | 重试死信、发起回放、关闭异常单、补录治理备注 | 支持按 `subscriberCode`、`eventType`、`deadLetteredAt`、`replayStatus`、`alertLevel` 分页筛选 | 按 `deadLetterId + operationNo` 幂等；重复回放不能生成多份治理记录 | 运维管理员可处理，审计员只读 | 每次回放、关闭、备注都必须记录操作人、原因和二次确认结果 | `409` 死信处理中，`422` 消费者已停用，`404` 死信不存在 |
| 运行审计与指标 | 查询投递延迟、失败率、积压量、订阅健康度和审计明细 | 标记异常单优先级、补录处置结论 | 支持按 `subscriberCode`、`status`、`timeRange`、`tenantId`、`traceId` 筛选 | 补录处置结论按 `incidentId` 幂等覆盖 | 运维管理员、审计员可写 | 运行指标调整、处置结论和备注需进入统一审计 | `404` 事件链路不存在，`422` 指标窗口非法 |

## 接口治理约束

| 项 | 约束说明 |
|----|----------|
| 查询口径 | 列表接口统一返回事件定义版本、投递状态、失败摘要和最近治理动作，避免前端拼接多表口径 |
| 写入口径 | 写接口只管理契约、订阅、事件信封和治理动作，不得替代业务模块执行业务补偿 |
| 幂等策略 | 事件注册、事件发布、死信回放、治理结论补录必须支持业务键幂等 |
| 权限模型 | 至少区分契约管理、运维治理、审计查看三类权限，不允许超级写权限混用 |
| 审计留痕 | 契约变更、订阅调整、手工补发、死信回放、关闭异常单必须全量审计 |
| 错误模型 | 统一区分参数非法、契约缺失、版本冲突、订阅冲突、投递失败、治理状态冲突六类错误 |
