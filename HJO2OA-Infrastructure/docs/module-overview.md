# 00-平台基础设施-模块功能

## 模块定位

平台基础设施域是横跨所有业务域的共享能力层，为 01-07 业务域提供统一的事件驱动、国际化、时区、多语言数据、附件、字典、配置、调度、审计、缓存、租户、错误码和安全工具能力。所有业务模块必须优先使用平台基础设施，不得自行重复建设。

其中多租户能力只负责系统隔离、资源隔离、配置隔离和配额隔离，不承担组织层级、岗位层级和业务授权主体建模；组织、部门、岗位、人员、角色等业务管理结构统一由 `01-组织与权限` 负责。

## 事件驱动架构

### 事件模型

```yaml
Event:
  eventId: string
  eventType: string        # 格式：{模块前缀}.{事件主题}.{动作}
  source: string
  timestamp: datetime
  payload: object
  metadata:
    correlationId: string
    userId: string
    traceId: string
```

### 事件类型命名规范

| 模块 | 前缀 | 示例 |
|------|------|------|
| 00-平台基础设施 | `infra.*` | `infra.config.updated`, `infra.tenant.disabled` |
| 01-组织与权限 | `org.*` | `org.person.created`, `org.role.person-granted` |
| 02-流程与表单 | `process.*` | `process.task.completed`, `process.instance.started` |
| 03-内容与知识 | `content.*` | `content.article.published`, `content.visibility.changed` |
| 04-门户与工作台 | `portal.*` | `portal.template.published` |
| 05-协同办公应用 | `biz.*` | `biz.meeting.created`, `biz.document.approved` |
| 06-消息移动与生态 | `msg.*` | `msg.notification.sent`, `msg.notification.delivered`, `msg.notification.read` |
| 07-数据服务与集成 | `data.*` | `data.sync.completed`, `data.api.published` |

### 模块角色

| 模块 | 角色 | 说明 |
|------|------|------|
| 00-平台基础设施 | 基础设施 | 提供事件总线、缓存刷新、审计消费等 |
| 01-组织与权限 | 生产者 | 发布组织/人员/角色变更事件 |
| 02-流程与表单 | 生产者+消费者 | 发布流程/任务事件；订阅组织变更 |
| 03-内容与知识 | 生产者 | 发布内容发布/下线事件 |
| 04-门户与工作台 | 消费者 | 订阅事件刷新卡片数据 |
| 05-协同办公应用 | 生产者+消费者 | 发布公文/会议/考勤事件；订阅流程事件 |
| 06-消息移动与生态 | 消费者 | 订阅跨模块总线事件生成消息和提醒；模块内部可继续细分 `msg.message.*` 与 `msg.channel.*` 领域事件 |
| 07-数据服务与集成 | 消费者 | 订阅事件触发同步和回写 |

## 国际化与时区策略

- 所有时间字段以 UTC 存储，查询时按用户时区转换
- 前端文案、后端消息、错误提示统一走 i18n 资源包
- 业务数据多语言字段按 `{entityId, field, locale}` 存储翻译值
- 无翻译时回退默认语言

## 交付边界

- 事件总线必须在第一阶段完成，作为所有跨模块联动的基础
- i18n/timezone/data-i18n 在第一阶段完成基础能力，后续阶段按需扩展语言包
- 附件中心、字典、配置中心、审计日志、错误码在第一阶段完成
- 缓存管理、定时任务、多租户、安全工具在第二阶段完成
