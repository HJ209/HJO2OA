# Timezone 前端说明

## 页面清单

| 页面/区域 | 页面职责 | 权限要求 | 状态要点 | 移动端约束 |
|-----------|----------|----------|----------|------------|
| 系统/租户时区策略页 | 管理默认时区、未来生效窗口、启停状态和作用域优先级 | 平台管理员、租户管理员 | 已生效、待生效、已停用、冲突待处理 | 移动端不提供策略治理 |
| 个人时区偏好页 | 用户查看当前有效时区并切换个人偏好 | 用户本人、管理员代设场景 | 跟随租户、个人覆盖、切换中、切换失败 | 移动端保留个人切换能力 |
| 时区转换与校验页 | 预览 UTC 与本地时间互转、DST 歧义提示和导入校验 | 已登录用户；批量校验需管理员权限 | 正常、DST 歧义、格式错误、可导出 | 移动端仅保留单值预览 |
| 生效窗口与审计页 | 查看时区变更历史、未来生效记录、冲突和治理备注 | 平台管理员、审计员 | 正常、待确认、已撤销、审计中 | 移动端只看摘要，不支持治理操作 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 说明 |
|-----------|----------|------|
| 系统/租户时区策略页 | 设置默认值、安排未来生效、停用策略、查看优先级链 | 页面需明确显示“用户覆盖 > 租户默认 > 系统默认” |
| 个人时区偏好页 | 切换个人时区、恢复跟随上级作用域、查看当前生效来源 | 切换成功后应刷新当前界面时间展示，但不得改写业务数据 |
| 时区转换与校验页 | 输入 UTC/本地时间、选择目标时区、查看 DST 提示、导出校验结果 | DST 歧义必须显式提示用户选择解释策略 |
| 生效窗口与审计页 | 查看变更记录、撤销未来生效、补录治理备注 | 撤销未来生效前需展示影响范围和二次确认 |

## 前端约束

| 项 | 说明 |
|----|------|
| 权限 | 必须区分平台级策略治理、租户级治理、个人偏好修改和审计查看权限 |
| 状态 | 前端需同时展示“当前有效时区”和“未来生效配置”，避免误导用户 |
| 终端 | 一期移动端只保留个人偏好切换和单值时间预览，不提供策略治理、批量校验和撤销操作 |
| 治理 | 默认时区切换、未来生效、撤销和冲突确认必须要求填写原因并进入审计 |
| 边界 | 前端只管理时间解释策略，不承担业务排班、会议编排、流程 SLA 等业务时间规则配置 |
