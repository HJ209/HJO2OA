# Timezone 模块概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块名称 | `HJO2OA-Timezone` |
| 模块定位 | `00-平台基础设施` 的时间解释策略中心，统一管理系统、租户、用户三级时区设置和 UTC 转换口径 |
| 真相源边界 | 拥有 `TimezoneSetting` 与时区优先级规则等技术真相源；不拥有业务日历语义、不拥有排班或会议时间主数据、不回写业务表存储值 |
| 一期目标 | 建立统一的时区配置、解析、转换和偏好能力，确保全平台时间字段 UTC 存储、按作用域正确展示 |

## 核心职责

| 职责 | 说明 | 一期范围 |
|------|------|----------|
| 时区策略管理 | 管理系统、租户、用户三级默认时区和生效窗口 | 支持新增、启停、设置默认和未来生效 |
| 统一解析链路 | 明确用户优先、租户次之、系统兜底的时区解析规则 | 支持按作用域解析有效时区 |
| UTC 转换服务 | 为写入、查询、导入导出和调度提供统一转换能力 | 支持单值和批量转换预览 |
| 时区偏好治理 | 管理用户自助切换和管理员代设，保留审计 | 支持个人偏好和租户默认管理 |
| 生效与审计 | 对策略切换、回滚、未来生效进行治理记录 | 支持审计查询和异常校验 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `01` 身份与权限 | 提供 `personId`、`tenantId` 等作用域标识，本模块不拥有身份主数据 |
| 上游 | `HJO2OA-Config`、`HJO2OA-Scheduler` | 复用环境配置与调度触发场景，不在本模块复制调度定义 |
| 下游 | `01-07` 各业务域 | 所有业务域都应统一通过本模块获取展示时区和转换结果 |
| 下游 | 前端壳层、门户、协同工作台 | 统一读取解析后的当前时区，用于列表、详情、日历和提醒展示 |
| 非本模块 | 业务日程与时间业务规则 | 会议开始时间、流程截止时间、排班规则等业务语义仍归业务域所有 |
| 非本模块 | 时间字段存储真相源 | 业务表中的时间字段必须继续以 UTC 存储，Timezone 只决定解释与展示方式 |

## 核心能力

| 能力 | 关键对象 | 输出 | 一期不包含 |
|------|----------|------|------------|
| 作用域时区策略 | `TimezoneSetting` | 系统、租户、用户三级有效时区 | 复杂时区模板市场 |
| 优先级解析 | 作用域解析链 | 当前请求的最终时区 | 动态脚本化优先级 |
| 转换与预览 | UTC 转换服务 | 展示时间、导出时间和校验结果 | 自定义日历计算引擎 |
| 调度解释 | 定时任务时区解释输入 | 调度触发时区基线 | 调度编排本身 |
| 生效治理 | 生效窗口与审计轨迹 | 时区切换历史和异常告警输入 | 绕过审计的手工改库 |

## 典型业务流

| 步骤 | 流程说明 | 关键产出 |
|------|----------|----------|
| 1 | 平台或租户管理员设置默认时区并配置生效时间 | 生效中的时区策略 |
| 2 | 用户按需设置个人时区覆盖，系统按优先级解析最终展示时区 | 当前用户有效时区 |
| 3 | 业务模块写入时间时统一转为 UTC 存储，查询时再按解析结果转换 | 一致的 UTC 存储与本地展示 |
| 4 | 调度、提醒、导出等场景统一调用转换服务生成展示值或执行窗口 | 场景化时间解释结果 |
| 5 | 时区策略切换、撤销或未来生效均进入审计，必要时发布配置变更事件 | 审计留痕与同步事件 |

## 一期实现约束

| 约束 | 说明 |
|------|------|
| 统一 UTC 存储 | 所有业务时间字段必须继续以 UTC 存储，禁止因时区切换回写历史业务数据 |
| 只接受标准时区标识 | 一期仅支持 IANA 时区标识，不支持自定义偏移字符串作为真相源 |
| 策略对象与运行结果分离 | `TimezoneSetting` 是策略真相源；转换结果、页面缓存和导出格式都是运行投影 |
| 解析优先级固定 | 用户级优先于租户级，租户级优先于系统级，禁止前端自行发明优先级 |
| 时区切换不改业务语义 | 时区变化只影响解释和展示，不改变会议是否迟到、流程是否超时等业务判定规则 |
| 治理动作必须审计 | 默认时区切换、未来生效、撤销覆盖和人工刷新都必须记录操作人和原因 |
