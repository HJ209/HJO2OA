# 审计日志前端

## 页面清单

| 页面/区域 | 适用端 | 页面职责 |
|-----------|--------|----------|
| 审计日志查询页 | 管理端 | 按模块、对象、操作者、租户、时间范围检索审计记录。 |
| 审计详情页 | 管理端 | 展示审计摘要、字段差异、事件链路和敏感字段脱敏视图。 |
| 归档管理页 | 管理端 | 发起归档任务、查看归档范围、状态和归档统计。 |
| 导出任务页 | 管理端 | 发起导出任务、跟踪执行状态并下载结果。 |
| 合规筛选面板 | 管理端 | 提供高频筛选模板、时间窗和风险动作快速定位。 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 约束 |
|-----------|----------|------|
| 审计日志查询页 | 高级筛选、保存筛选条件、跳转详情 | 查询页只读，不提供任何“修复”按钮。 |
| 审计详情页 | 展开字段差异、查看关联 `traceId`、切换脱敏视图 | 审计详情不允许编辑，不允许删除。 |
| 归档管理页 | 选择归档范围、确认执行、查看任务结果 | 归档前必须展示影响范围和不可逆说明。 |
| 导出任务页 | 提交导出、轮询状态、下载结果 | 大范围导出仅走异步任务。 |
| 合规筛选面板 | 按关键动作快速筛选，如登录、回滚、手工失效缓存 | 只改变查询条件，不改变事实数据。 |

## 权限、状态与终端治理约束

- 权限约束：
  - 审计员可查询与查看脱敏详情。
  - 合规管理员可导出。
  - 归档操作需更高治理权限。
- 状态约束：
  - 记录状态展示 `ACTIVE / ARCHIVED`。
  - 敏感字段默认脱敏，若无权限则不出现原值切换入口。
  - 归档任务应展示 `PENDING / RUNNING / SUCCEEDED / FAILED`。
- 终端约束：
  - 一期仅提供管理端页面，不提供移动端审计治理。
  - 业务端不开放审计写入或审计修复 UI。
- 治理约束：
  - 前端必须持续提示“审计只做留痕，不是业务状态机”。
  - 所有归档、导出等治理动作提交前需展示二次确认与审计写入提示。
