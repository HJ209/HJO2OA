# 缓存管理前端

## 页面清单

| 页面/区域 | 适用端 | 页面职责 |
|-----------|--------|----------|
| 缓存策略配置页 | 管理端 | 展示命名空间、TTL、淘汰策略、失效模式和启用状态。 |
| 缓存失效治理页 | 管理端 | 发起单次/批量失效，查看最近治理动作。 |
| 缓存监控页 | 管理端 | 展示命中率、容量、热点 Key、失效次数等指标。 |
| 命名空间详情抽屉 | 管理端 | 查看单一命名空间的策略、最近失效轨迹和排障提示。 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 约束 |
|-----------|----------|------|
| 缓存策略配置页 | 新建策略、编辑 TTL、启停命名空间 | 仅调整策略，不直接操作缓存值。 |
| 缓存失效治理页 | 输入命名空间/Key、选择原因、执行手工失效 | 高风险批量操作必须二次确认。 |
| 缓存监控页 | 切换时间窗、查看热点、下钻命名空间 | 监控页只读，不允许直接写缓存。 |
| 命名空间详情抽屉 | 查看最近失效事件、关联审计和来源事件 | 只显示运行信息，不显示业务真相源内容。 |

## 权限、状态与终端治理约束

- 权限约束：
  - 缓存管理员和运维管理员可执行手工失效。
  - 普通只读运维角色仅可查看监控与轨迹。
- 状态约束：
  - 策略展示 `active=true/false`。
  - 失效记录展示 `reasonType`、来源事件、执行时间与执行结果。
  - 指标卡片必须明确展示数据窗口，避免将瞬时统计误读为业务事实。
- 终端约束：
  - 一期仅提供管理端页面，不提供业务端或移动端缓存治理入口。
  - 业务端组件不得暴露“清缓存”按钮。
- 治理约束：
  - 前端必须显式提示“缓存不是事实真相源”。
  - 手工失效提交前需展示影响范围和审计留痕提示。
