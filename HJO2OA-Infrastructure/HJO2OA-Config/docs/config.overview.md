# 配置中心概览

## 模块定位

| 项 | 说明 |
|----|------|
| 模块角色 | `HJO2OA-Config` 是 `00-平台基础设施` 契约与策略层 + 运维治理层子模块，负责统一参数、覆盖链、Feature Flag 与回滚治理。 |
| 策略对象 | `ConfigEntry`、`ConfigOverride`、`FeatureRule` 描述默认值、作用域覆盖与开关规则。 |
| 运行对象 | 配置解析快照、热更新通知、运行中订阅者本地缓存属于运行对象，不是配置真相源。 |
| 边界强调 | 本模块管理“配置定义与生效规则”，不管理业务状态本身；回滚也只回滚配置真相源，不回滚业务数据。 |

## 核心职责

| 职责 | 说明 |
|------|------|
| 参数注册 | 维护配置键、类型、默认值、校验规则、是否支持热更新。 |
| 覆盖链管理 | 维护全局、租户、组织、角色、用户等作用域覆盖值，且仅引用 `01` 输出的主体 ID。 |
| Feature Flag 治理 | 维护全局开关、灰度百分比、租户/组织/角色命中规则和回退顺序。 |
| 配置解析与发布 | 以“默认值 -> 覆盖链 -> Feature Rule”顺序解析，并对外输出稳定查询、变更事件和热更新通知。 |
| 审计与回滚 | 所有配置变更、灰度切换和人工修复都进入审计，并支持基于历史版本执行回滚。 |

## 上下游边界

| 方向 | 对象 | 边界说明 |
|------|------|----------|
| 上游 | `01-组织与权限` | 只引用组织、角色、用户 ID 作为覆盖范围，不维护这些主体主数据。 |
| 上游 | `HJO2OA-Tenant` | 可在租户初始化时生成默认覆盖，但租户生命周期不归本模块。 |
| 下游 | `01-07` 全业务域 | 业务模块只消费解析结果和事件，不得各自维护平行配置中心。 |
| 下游 | `HJO2OA-Cache` | 配置变更触发缓存失效和运行快照刷新，但缓存不是配置真相源。 |
| 下游 | `HJO2OA-Audit` | 所有治理动作写入统一审计，审计仅留痕，不反向控制配置状态机。 |
| 边界外 | 业务策略快照 | 门户、消息、流程等业务模块可做只读缓存和本地快照，但必须可回源、可刷新。 |

## 核心能力

| 能力 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| 配置项管理 | `configKey`、`configType`、`defaultValue`、`validationRule` | 配置定义真相源 | `configKey` 全局唯一。 |
| 覆盖链管理 | `scopeType`、`scopeId`、`overrideValue` | 作用域覆盖值 | 作用域主体只引用 ID；覆盖不改变默认值定义。 |
| Feature Flag 规则 | `ruleType`、`ruleValue`、`sortOrder` | 开关命中顺序和灰度策略 | 规则顺序必须显式声明，命中后停止继续计算。 |
| 解析与预览 | 配置键、租户/主体上下文 | 最终生效值和命中链路 | 返回命中链，但不暴露底层实现细节。 |
| 回滚治理 | 历史版本、目标版本 | 回滚后的新配置版本 | 回滚是新的治理动作，不覆盖历史记录。 |

## 典型业务流

1. 管理员创建配置项，声明键名、类型、默认值、校验规则和是否允许运行时热更新。
2. 若业务存在差异化诉求，管理员为租户或主体配置覆盖值，形成可追溯的覆盖链。
3. 对 `FEATURE_FLAG` 类型配置继续维护 `FeatureRule`，声明灰度范围、百分比规则和命中顺序。
4. 下游模块查询解析结果时，本模块按“默认值 -> 覆盖链 -> Feature Rule”输出最终值，并返回命中链路用于排障。
5. 配置发生修改、灰度调整或回滚后，本模块发布 `infra.config.updated` 或 `infra.feature-flag.changed`，驱动缓存刷新和运行时热更新，同时写入统一审计。

## 一期实现约束

- 一期支持 `STRING / NUMBER / BOOLEAN / JSON / FEATURE_FLAG` 五类配置，不支持任意脚本型动态配置。
- 一期覆盖范围支持全局、租户及主体 ID 引用，不在本模块维护主体生命周期。
- 一期灰度策略优先支持全局开关、租户命中、主体命中和百分比命中，不提供复杂规则编排引擎。
- 仅标记 `mutableAtRuntime=true` 的配置允许热更新；其余配置只允许写入并在受控窗口生效。
- 回滚动作必须重新写入配置表和审计表，不允许直接恢复数据库旧值绕过治理流程。
