# 配置中心前端

## 页面清单

| 页面/区域 | 适用端 | 页面职责 |
|-----------|--------|----------|
| 配置项管理页 | 管理端 | 展示配置键、类型、默认值、热更新属性和当前状态。 |
| 覆盖链管理页 | 管理端 | 维护租户/主体级覆盖链，展示命中优先级和作用域。 |
| Feature Flag 管理页 | 管理端 | 管理开关状态、灰度规则、命中顺序和启停。 |
| 配置解析预览页 | 管理端 | 输入上下文预览最终生效值和命中链。 |
| 变更历史与回滚页 | 管理端 | 查看差异、审计关联和执行回滚。 |

## 页面职责与关键交互

| 页面/区域 | 关键交互 | 约束 |
|-----------|----------|------|
| 配置项管理页 | 新建配置、编辑默认值、切换状态 | 非热更新配置修改后应明确提示生效窗口。 |
| 覆盖链管理页 | 按租户/主体新增覆盖、停用覆盖、查看冲突 | 仅允许选择外部主体 ID，不在页面内维护主体数据。 |
| Feature Flag 管理页 | 新增规则、拖动排序、设置灰度比例、启停规则 | 命中顺序必须唯一且显式展示。 |
| 配置解析预览页 | 输入租户、组织、角色、用户上下文并查看命中链 | 预览只读，不得触发写操作。 |
| 变更历史与回滚页 | 查看前后值差异、关联审计、提交回滚 | 回滚前需二次确认并展示影响范围。 |

## 权限、状态与终端治理约束

- 权限约束：
  - 配置管理员可维护配置项和覆盖链。
  - 灰度管理员可调整 Feature Flag 规则。
  - 回滚权限应与普通编辑权限分离。
- 状态约束：
  - 配置项状态展示 `ACTIVE / DISABLED / DEPRECATED`。
  - Feature Rule 展示启用态和命中顺序，禁用规则不参与解析但保留历史。
  - 覆盖链页面需显式提示当前规则是否被上游更高优先级覆盖。
- 终端约束：
  - 一期仅提供管理端治理页面，不提供移动端配置维护能力。
  - 业务端只读取解析结果，不暴露配置编辑 UI。
- 治理约束：
  - 页面必须提示“缓存不是配置真相源”，变更后展示刷新和下游应用状态。
  - 任何回滚、批量停用、灰度比例调整都属于治理动作，前端必须展示审计留痕提示。
